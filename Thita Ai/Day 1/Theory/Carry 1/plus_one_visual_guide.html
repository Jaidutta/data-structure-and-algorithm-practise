Plus One

Handling Carry - The Odometer Pattern

🎯 What is Plus One?

The Plus One (Handling Carry) pattern solves a deceptively simple problem: given a number stored as an array of digits, add 1 to it. Sounds easy, right? But there's a catch — the number might be too large for standard integer types (think 500 digits long), and you need to handle carry propagation manually, just like you learned in elementary school.

📚 The Core Idea

Think of a car odometer rolling from 00999 to 01000.
• The rightmost 9 becomes 0, and triggers a carry
• The next 9 becomes 0, triggers another carry
• This ripples left until you hit a non-9 digit
• If ALL digits are 9, you need a new digit at the front: [1,0,0,0,0]

Time: O(N) Space: O(1) in-place

🚗 The Odometer Analogy

Imagine an old-school car odometer with 4 wheels, each showing a digit. When you drive one more mile, the rightmost wheel advances by 1. But what happens when it's at 9?

✅ Simple Case — No Carry

🚗

Odometer shows: 0 1 2 3
Drive 1 mile → 0 1 2 4

The rightmost wheel just advances. Done! No other wheels need to move.

⚡ Carry Case — Ripple Effect

🚗💨

Odometer shows: 0 1 9 9
Drive 1 mile:
• Rightmost 9 → 0, carry 1
• Next 9 → 0, carry 1
• Next 1 → 2, stop
Result: 0 2 0 0

🔥 Edge Case: All 9s

Odometer at: 9 9 9 9
Drive 1 mile → Need a new wheel!
Result: 1 0 0 0 0 (5 digits now)

This is like when your car odometer adds a new digit after 99,999 miles.

🧠 The Algorithm

🔑 Key Insight

You only need to process digits as long as there's a carry.
If you add 1 to a non-9 digit, you're done immediately — no need to check the rest!

Step-by-Step Process

  1. Start from the rightmost digit (last index)
  2. Add 1 to it
  3. If result < 10: Update digit and return immediately (done!)
  4. If result = 10: Set digit to 0, continue to the next digit left (carry)
  5. Repeat step 3-4 for each digit moving left
  6. If you finish all digits and still have a carry: Create a new array with a leading 1

Decision Tree

for i from (n-1) down to 0:
if digit[i] < 9:
digit[i]++
return array // Done!
else:
digit[i] = 0 // Carry to next
// If we're here, all digits were 9
return [1, 0, 0, ..., 0]

💻 Code Implementation

Java Solution

public int[] plusOne(int[] digits) {
    // Start from the rightmost digit (ones place)
    for (int i = digits.length - 1; i >= 0; i--) {
        
        // If current digit is less than 9
        // We can just increment it and we're done — no carry!
        if (digits[i] < 9) {
            digits[i]++;
            return digits;  // Early exit — no more processing needed
        }
        
        // If current digit is 9, it becomes 0 and we continue
        // (carry propagates to the next left digit)
        digits[i] = 0;
    }
    
    // If we're here, all digits were 9 → we need a new leading 1
    // Example: [9,9,9] becomes [1,0,0,0]
    int[] result = new int[digits.length + 1];
    result[0] = 1;  // First digit is 1, rest are already 0 (default)
    return result;
}

Python Solution

def plusOne(digits):
    # Process from right to left
    for i in range(len(digits) - 1, -1, -1):
        if digits[i] < 9:
            digits[i] += 1
            return digits
        digits[i] = 0
    
    # All digits were 9 — prepend 1
    return [1] + digits

✅ Why This Works

Efficiency: We only touch each digit once in the worst case (all 9s).
Early exit: Most of the time (when rightmost digit < 9), we finish in 1 step.
No extra space: We modify the input array in-place (except for the all-9s case).
Clean logic: The loop naturally handles both the carry and the all-9s edge case.

📊 Complexity Analysis

Metric Best Case Worst Case Explanation
Time O(1) O(N) Best: rightmost digit < 9, increment and return. Worst: all 9s, touch every digit.
Space O(1) O(N) Best: modify in-place. Worst: all 9s, create new array with N+1 elements.

📈 Real-World Performance

Probability of carry propagating:
• 10% chance the last digit is 9 (needs carry)
• 1% chance last two digits are both 9
• 0.1% chance last three digits are all 9

In practice, the algorithm runs in near-constant time for random inputs!

⚠️ Edge Cases & Pitfalls

Common Edge Cases

1. All 9s: [9, 9, 9] → [1, 0, 0, 0]
Solution: Create new array with N+1 length, first element = 1
2. Single digit 9: [9] → [1, 0]
Solution: Same logic applies
3. No carry: [1, 2, 3] → [1, 2, 4]
Solution: Early return after incrementing last digit
4. Carry in middle: [1, 9, 9] → [2, 0, 0]
Solution: Carry stops when we hit a non-9

Common Mistakes to Avoid

  1. ❌ Forgetting to handle the all-9s case (trying to modify array in-place when you need to expand it)
  2. ❌ Off-by-one errors when looping backwards (should be i >= 0, not i > 0)
  3. ❌ Not returning early when no carry is needed (wastes computation)
  4. ❌ Treating this as string manipulation instead of digit manipulation

🎯 When to Use This Pattern

✅ Use Plus One Pattern When:

  • Number is represented as an array/list of digits
  • Need to perform arithmetic operations on very large numbers
  • Input might exceed standard integer limits (e.g., 500 digits)
  • Problem mentions "increment", "add one", or "next number"
  • Dealing with carry propagation in base-10 arithmetic

Real-World Applications

  • 🔐 Cryptography: Operations on very large prime numbers
  • 📊 Scientific computing: High-precision arithmetic
  • 💾 Version control: Incrementing version numbers [1.9.9] → [2.0.0]
  • 🎮 Game development: High score counters beyond int limits
  • 🧮 Big integer libraries: Custom implementations for languages without native BigInt
