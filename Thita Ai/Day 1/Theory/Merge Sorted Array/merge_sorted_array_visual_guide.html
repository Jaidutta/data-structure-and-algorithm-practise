Array/Matrix Manipulation Pattern

Merge Sorted Array
(In-place from End)

Master the art of combining two sorted arrays efficiently by working backward—leveraging existing space to avoid overwrites and achieve O(1) memory complexity.

Complexity Analysis

Time Complexity
O(m + n)

Single pass through both arrays—every element touched exactly once

Space Complexity
O(1)

No extra space needed—merge happens directly in the existing array

Pattern Overview

The Merge Sorted Array (In-place from End) pattern is a technique for combining two sorted arrays into a single sorted array. You start at the end of the destination array and work backward, placing the largest elements from both arrays into the available space.

Why From the End?

If you merged from the start, you'd overwrite elements in the first array before you had a chance to compare them. By starting from the end (where there's empty space), you avoid this problem entirely.

This method ensures that you do not overwrite any values in the first array before they have been processed. Use this pattern when you need to merge sorted data and the first array already has enough capacity to hold the final result.

Core Concepts

⚡ Why Subtract 1?

Short answer: Arrays start at index 0!

Detailed explanation:

If m = 3, nums1 has valid elements at indices: 0, 1, 2

Index:
0 ↓
1 ↓
2 ↓
Value:
1
2
3

So the last valid element is at index m - 1 = 2
NOT at index m = 3 (that would be out of bounds!)

Similarly, if the total capacity is m + n = 6:

Index:
0
1
2
3
4
5

The last position to fill is at index m + n - 1 = 5
NOT at index m + n = 6 (that's beyond the array!)

The Pattern:
• Array of size n has indices from 0 to n-1
• First element: array[0]
• Last element: array[n-1]
• Length n is NOT a valid index!

💡 Pro Tip: When you see -1 in index initialization, it's almost always because of 0-based indexing. This is one of the most common sources of off-by-one errors in programming!

Two Pointers Technique
Imagine two fingers pointing at the end of each sorted array. At each step, you compare the numbers under your fingers and move the larger one to the end of the destination array. This ensures you're always placing the next largest element in its correct position.
Initialize Three Pointers
  • i = m - 1: Last valid element of nums1 (remember: 0-indexed!)
  • j = n - 1: Last element of nums2 (remember: 0-indexed!)
  • k = m + n - 1: Last position in nums1 (remember: 0-indexed!)
Compare and Place
At each step, compare nums1[i] and nums2[j]. Place the larger value at nums1[k]. Move the corresponding pointer(s) backward.
Finish Up
If nums2 still has elements left, copy them into nums1. If nums1 has elements left, they're already in place—no action needed!

void merge(vector<int>& nums1, int m, vector<int>& nums2, int n) {
    // Initialize three pointers
    int i = m - 1;      // Last valid element in nums1
    int j = n - 1;      // Last element in nums2
    int k = m + n - 1; // Last position in nums1
    
    // Merge from end to beginning
    while (i >= 0 && j >= 0) {
        if (nums1[i] > nums2[j]) {
            nums1[k--] = nums1[i--]; // Take from nums1
        } else {
            nums1[k--] = nums2[j--]; // Take from nums2
        }
    }
    
    // Copy remaining elements from nums2 (if any)
    // Note: Remaining nums1 elements are already in place
    while (j >= 0) {
        nums1[k--] = nums2[j--];
    }
}
public void merge(int[] nums1, int m, int[] nums2, int n) {
    // Initialize three pointers
    int i = m - 1;      // Last valid element in nums1
    int j = n - 1;      // Last element in nums2
    int k = m + n - 1; // Last position in nums1
    
    // Merge from end to beginning
    while (i >= 0 && j >= 0) {
        if (nums1[i] > nums2[j]) {
            nums1[k--] = nums1[i--]; // Take from nums1
        } else {
            nums1[k--] = nums2[j--]; // Take from nums2
        }
    }
    
    // Copy remaining elements from nums2 (if any)
    // Note: Remaining nums1 elements are already in place
    while (j >= 0) {
        nums1[k--] = nums2[j--];
    }
}

// Alternative: Using System.arraycopy for remaining elements
public void mergeOptimized(int[] nums1, int m, int[] nums2, int n) {
    int i = m - 1;
    int j = n - 1;
    int k = m + n - 1;
    
    // Merge larger elements first
    while (i >= 0 && j >= 0) {
        nums1[k--] = (nums1[i] > nums2[j]) ? nums1[i--] : nums2[j--];
    }
    
    // Efficiently copy remaining nums2 elements
    if (j >= 0) {
        System.arraycopy(nums2, 0, nums1, 0, j + 1);
    }
}
Key Implementation Points
  • Three pointers track position in both arrays and where to write next
  • Merge from end to prevent overwriting unprocessed values
  • Only copy leftover nums2 elements; nums1 elements already in place
  • Decrement all pointers after each placement using post-decrement
  • Java Bonus: System.arraycopy provides optimized bulk copy for remaining elements

🤔 Why Only Copy Remaining nums2 Elements?

Short Answer: Leftover nums1 elements are already in their correct positions!

🎯 The Key Insight

We're merging INTO nums1. When nums2 runs out, any remaining nums1 elements haven't been moved yet, which means they're still sitting in their original positions—and those positions are already correct for the final sorted array!

Think about it: if we haven't moved them yet, it's because all nums2 elements were larger and got placed after them. So the remaining nums1 elements are already sorted at the front.

⚠️ Why We DON'T Need This:

// ❌ This loop is NOT needed!
while (i >= 0) {
    nums1[k--] = nums1[i--];  // Unnecessary!
}

This would be copying nums1 elements to themselves. If i = 2 and k = 2, you'd do nums1[2] = nums1[2] which is pointless!

💡 Memory Trick

"If it's still in nums1, it's already home!"

nums2 elements need to be moved into nums1.
nums1 elements are already in nums1 (just haven't been touched yet).

Java-Specific Optimizations

  • System.arraycopy: Native method for efficient array copying (faster than loop for large arrays)
  • Ternary operator: More concise than if-else for simple comparisons
  • Array bounds: Java automatically checks bounds (C++ doesn't), so no additional checks needed
  • Pass by reference: Arrays are objects in Java, passed by reference like C++ references

Edge Cases & Debugging

nums2 is Empty
No action needed—nums1 is already sorted and complete. The while loop for copying nums2 won't execute.
nums1 is Empty (apart from buffer)
When m=0, the first while loop never executes. The second loop copies all elements from nums2 into nums1.
All nums2 Elements Larger
nums2 elements get placed at the end. nums1 elements remain at the front. Example: nums1=[1,2,3], nums2=[7,8,9] → [1,2,3,7,8,9]
All nums2 Elements Smaller
All nums1 elements get moved to the back first. Then remaining nums2 elements copied to front. Example: nums1=[7,8,9], nums2=[1,2,3] → [1,2,3,7,8,9]
⚠️ Common Pitfalls
  • Off-by-one errors in pointer initialization (remember: m-1, n-1, m+n-1)
  • Forgetting to handle leftover nums2 elements
  • Not recognizing that leftover nums1 elements don't need moving
  • Comparing indices instead of actual array values

Mathematical Foundation

This pattern leverages a fundamental property of sorted arrays: the largest elements are at the end. By merging from the back, you ensure you never overwrite unprocessed data.

It's Actually Merge Sort's Merge Step!

This is a practical application of the merge step in the classic Merge Sort algorithm, optimized for in-place operations. Instead of creating a temporary array, we use the existing buffer space in nums1.

  • Invariant: At each step, elements from index k+1 to m+n-1 are in their final sorted positions
  • Correctness: We always pick the larger of nums1[i] or nums2[j], maintaining sorted order
  • Termination: Loop ends when we've processed all elements from at least one array
