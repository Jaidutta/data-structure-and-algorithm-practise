1. What is Kadane's Algorithm? 🤔

Simple Definition:

Kadane's Algorithm is a Dynamic Programming pattern for finding the "best" (maximum or minimum) value in a contiguous subarray. It works by calculating the "Best Ending at Index i" and making a local decision at each step.

Key Characteristics:

  • ✓ Works with negative numbers (unlike sliding window)
  • ✓ Contiguous subarrays only
  • ✓ O(n) time, O(1) space - very efficient!
  • ✓ Decision-based approach at each index
  • ✓ Can be adapted for sum, product, frequency
⚠️ Important: Kadane's is NOT about memorizing code - it's about understanding the thought process and decision-making at each index!

2. The Core Technique: "The Party Choice" 🎉

The Fundamental Question at Each Index:

At every index i, ask yourself:
"Should I join the previous party, or start my own party?"

Visual Representation:

       [ Previous History ]      [ Current Element (You) ]
              (i-1)                        (i)
               |                            |
               v                            v
      ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
      |   THE DECISION FORK (Best Ending at i)         |
      ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
               /                                \
              /                                  \
             /                                    \
    OPTION A: JOIN THE PARTY            OPTION B: START NEW PARTY
    ═════════════════════════            ══════════════════════════
    (Previous Best + Current)            (Current Element Alone)
    
    "The history is positive/           "The history is negative/
     helpful. I will extend              burden. I will start a
     the streak."                        fresh streak."
     
     bestEnding + arr[i]                 arr[i]

The Two Choices in Detail:

Choice 1 Join the Previous Party

When: The previous "bestEnding" is helpful (positive for max, negative for min)
Action: bestEnding = bestEnding + arr[i]
Meaning: Extend the existing subarray by including current element
Example: If bestEnding = 5 and arr[i] = 3, join to get 8

Choice 2 Start Your Own Party

When: The previous "bestEnding" is a burden
Action: bestEnding = arr[i]
Meaning: Discard all history and start fresh from current element
Example: If bestEnding = -10 and arr[i] = 5, start new at 5

The Update Step:

After making the local decision at index i, compare the new bestEnding with the global result to see if you've found a new overall maximum/minimum. 

// After calculating bestEnding at index i
result = Math.max(result, bestEnding);  // For maximum problems
result = Math.min(result, bestEnding);  // For minimum problems

3. When to Use Kadane's Algorithm 🎯

Recognition Checklist - ALL Must Be True:

Criterion What to Look For Example
1. Data Structure Input is an Array int[] nums = {-2, 1, -3, 4}
2. Content Type Contains positives, negatives, zeros [-1, 5, -3, 0, 2]
3. Target Find a Contiguous Subarray "Find the subarray..."
4. Objective Find the "Best" value (Max/Min) "Maximum sum", "Minimum product"
5. Operation Involves Sum, Product, or Frequency Sum of elements, product of elements

Common Keywords That Indicate Kadane's:

  • ✓ "Maximum subarray sum"
  • ✓ "Minimum subarray sum"
  • ✓ "Maximum product subarray"
  • ✓ "Largest contiguous sum"
  • ✓ "Best subarray" (with sum/product)
  • ✓ "Contiguous elements" + "maximum/minimum"

Variations to Watch For:

Advanced Variations:

  • 🔸 One deletion allowed
  • 🔸 Circular array (last connects to first)
  • 🔸 Absolute sum (|sum|)
  • 🔸 K deletions allowed
  • 🔸 With constraints on length

4. Kadane vs Sliding Window 🆚

⚠️ Why Sliding Window FAILS with Negatives

Sliding Window loses its decision-making power when negative numbers are present!

The Problem with Sliding Window:

With Positive Numbers ONLY:
━━━━━━━━━━━━━━━━━━━━━━━━━━
Expand window (high++)  → Sum INCREASES  ✅ Predictable
Shrink window (low++)   → Sum DECREASES  ✅ Predictable

With Negative Numbers:
━━━━━━━━━━━━━━━━━━━━━━━━━━
Expand window (high++) → Might add -5  → Sum DECREASES  ❌ Unpredictable!
Shrink window (low++)  → Might remove -5 → Sum INCREASES  ❌ Unpredictable!

Result: You don't know which pointer to move! 💥

Comparison Table:

Feature Sliding Window Kadane's Algorithm
Input Type Works with positive numbers Handles positives, negatives, zeros
Logic Adjust start/end pointers (low/high) Calculate "Best Ending at i"
Decision "Should I grow or shrink?" "Should I join or start new?"
Primary Use Subarrays with specific constraints (sum=k) Best (Max/Min) sum or product
Time Complexity O(n) O(n)

Visual Demonstration of the Failure:

❌ WHY SLIDING WINDOW FAILS WITH NEGATIVES:

Scenario 1: All Positives (Sliding Window Works ✅)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Array: [1, 2, 3, 4, 5]
       ↑─────↑
       low   high   Sum = 1+2+3 = 6

Expand (high++):  Add 4 → Sum = 10 ⬆ INCREASES (Predictable!)
Shrink (low++):   Remove 1 → Sum = 9 ⬇ DECREASES (Predictable!)

Decision: CLEAR! Move pointers based on what you need.

Scenario 2: With Negatives (Sliding Window Breaks ❌)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Array: [5, -3, 2, -1, 4]
       ↑────────↑
       low      high   Sum = 5+(-3)+2 = 4

Expand (high++):  Add -1 → Sum = 3 ⬇ DECREASED! (Unexpected!)
                  What if next is +10? Should we expand?
                  
Shrink (low++):   Remove 5 → Sum = -1 ⬇ DECREASED! (Unexpected!)
                  We removed positive but sum went down!

Decision: CONFUSED! 🤯 Can't predict which pointer to move!

The Problem:
┌───────────────────────────────────────────────────┐
│ Negatives make it IMPOSSIBLE to know if:          │
│ • Expanding will increase or decrease sum         │
│ • Shrinking will increase or decrease sum         │
│ → Lost decision-making power! 💥                  │
└───────────────────────────────────────────────────┘

public int maxSubArray(int[] nums) {
    // Handle empty array
    if (nums.length == 0) return 0;
    
    // Initialize with first element
    int bestEnding = nums[0];  // Best sum ending at current index
    int result = nums[0];      // Global maximum
    
    // Iterate through array starting from index 1
    for (int i = 1; i < nums.length; i++) {
        // THE DECISION: Join previous party OR start new party
        // Option A: Extend previous subarray (bestEnding + nums[i])
        // Option B: Start fresh from current element (nums[i])
        bestEnding = Math.max(bestEnding + nums[i], nums[i]);
        
        // Update global result if we found a better sum
        result = Math.max(result, bestEnding);
    }
    
    return result;
}

// Time Complexity: O(n) - single pass through array
// Space Complexity: O(1) - only use two variables

💡 Key Insights:

  • ✓ At each index, compare: (previous + current) vs (current alone)
  • ✓ If previous sum is negative, it's a burden - start fresh
  • ✓ If previous sum is positive, it's helpful - extend it
  • ✓ Always track global maximum separately

public int minSubArray(int[] nums) {
    // Handle empty array
    if (nums.length == 0) return 0;

    int bestEnding = nums[0];  // Min sum ending at current index
    int result = nums[0];      // Global minimum

    for (int i = 1; i < nums.length; i++) {

        // Option B: Start fresh from current element (nums[i])
        bestEnding = Math.min(bestEnding + nums[i], nums[i]);

public int maxProduct(int[] nums) {
    if (nums.length == 0) return 0;
    
    // Must track BOTH max and min (negatives flip signs!)
    int maxEnding = nums[0];
    int minEnding = nums[0];
    int result = nums[0];
    
    for (int i = 1; i < nums.length; i++) {
        // Calculate three choices:
        // 1. Current element alone
        // 2. Current × Previous Max
        // 3. Current × Previous Min (important for negatives!)
        
        int choice1 = nums[i];
        int choice2 = maxEnding * nums[i];
        int choice3 = minEnding * nums[i];
        
        // Update current max and min
        maxEnding = Math.max(choice1, Math.max(choice2, choice3));
        minEnding = Math.min(choice1, Math.min(choice2, choice3));
        
        result = Math.max(result, maxEnding);
    }
    
    return result;
}

💡 Why Track Both Max and Min?

• Positive current → wants maxEnding (big positive × positive = bigger positive)
• Negative current → wants minEnding (big negative × negative = big positive)
• Example: -2 × (-10) = 20 (good!), but -2 × 5 = -10 (bad!)
