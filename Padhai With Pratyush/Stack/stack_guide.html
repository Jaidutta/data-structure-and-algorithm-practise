Stack

Last In, First Out (LIFO)

🎯 When to Use Stack

Core Philosophy: "Most Recent Kept Item"

Use a Stack when you process items sequentially and need to access the most recent valid item that hasn't been discarded yet.

Key Indicators

🔄
LIFO Requirement

Last In, First Out property needed. The most recent item must be processed first.

🔗
Matching/Canceling

Matching brackets, canceling adjacent duplicates, or cleaning data based on conditions.

📊
Next/Previous Element

"Next Greater," "Previous Smaller," "Next Warmer" - need to look back at recent elements.

↩️
Reversal

Any logic requiring reversal of data (e.g., string reversal).

The Universal Template

For every item, make three decisions:

1. When to POP? Should we discard the top element(s)?
2. When to PUSH? Should the current item be added?
3. Result? Calculate the answer based on stack's state.

📚 Stack Basics

What is a Stack?

A Stack is like a Box or Pile with only one opening at the top.

  • PUSH: Add item to top
  • POP: Remove item from top
  • PEEK/TOP: View top item without removing

Visual Representation

C ← TOP
B
A

PUSH 'C' → POP removes 'C'

Types of Stacks

  1. Simple Stack: Standard storage, no special ordering
  2. Monotonic Stack: Maintains elements in increasing or decreasing order
  3. Greedy Stack: Maintains order based on specific problem conditions/priority

Problem 1: Remove Adjacent Duplicates

Problem: Given a string like "abbaca", remove adjacent duplicates in pairs. Repeat until no duplicates remain.

LeetCode: #1047

🔍 Why Simple Loop Fails

A simple i vs i-1 check fails because removing items creates new adjacencies (holes close up). You need to track the "last item remaining" after deletions.

🎯 The Stack Solution

Stack acts like a "pocket" - you hold items, but if a new item matches the one at the top, you throw both away!

💻 Java Code

/**
 * Remove All Adjacent Duplicates
 * LeetCode 1047
 * 
 * Time: O(n) - single pass through string
 * Space: O(n) - stack storage
 */
public String removeDuplicates(String s) {
    // Stack acts as the "pocket" to hold valid characters
    Stack<Character> st = new Stack<>();
    
    for (char c : s.toCharArray()) {
        // If stack not empty AND top matches current → Remove pair
        if (!st.isEmpty() && st.peek() == c) {
            st.pop(); // Discard both
        } else {
            // Otherwise, push current character
            st.push(c);
        }
    }
    
    // Reconstruct string from stack (LIFO comes out reversed)
    StringBuilder sb = new StringBuilder();
    while (!st.isEmpty()) {
        sb.append(st.pop());
    }
    return sb.reverse().toString();
}

Problem 2: Valid Parentheses

Problem: Validate if brackets () [] {} are correctly matched and nested.

LeetCode: #20

🔍 How It Works

This mimics how compilers work:

1. Opening Brackets: Always PUSH to stack (store for later matching)
2. Closing Brackets: Must match the Most Recent opening bracket (Stack Top)
3. End: Stack must be EMPTY (no unmatched brackets)

📊 Example: "({[]})"

Step-by-Step Process
Step 1: Read '(' → PUSH
Stack: [(]
Step 2: Read '{' → PUSH
Stack: [( {]
Step 3: Read '[' → PUSH
Stack: [( { []
Step 4: Read ']' → Matches '[' at top? YES → POP
Stack: [( {]
Step 5: Read '}' → Matches '{' at top? YES → POP
Stack: [(]
Step 6: Read ')' → Matches '(' at top? YES → POP
Stack: [] (EMPTY!)
✓ VALID! Stack is empty at the end.

❌ Invalid Example: "([)]"

Step 1-3: Push '(', '[', read ')'
Stack at Step 3: [(, []
Step 4: Read ')' → Top is '[' not '('
✗ MISMATCH!

💻 Java Code

/**
 * Valid Parentheses
 * LeetCode 20
 * 
 * Time: O(n), Space: O(n)
 */
public boolean isValid(String s) {
    Stack<Character> st = new Stack<>();
    
    for (char c : s.toCharArray()) {
        // Opening brackets: Always PUSH
        if (c == '(' || c == '{' || c == '[') {
            st.push(c);
        } 
        // Closing brackets: Check for match
        else {
            // If stack empty → no opening bracket to match
            if (st.isEmpty()) return false;
            
            char top = st.peek();
            
            // Check if top matches current closing bracket
            if ((c == ')' && top == '(') || 
                (c == '}' && top == '{') || 
                (c == ']' && top == '[')) {
                st.pop(); // Valid match → remove pair
            } else {
                return false; // Mismatch!
            }
        }
    }
    
    // Valid only if all brackets matched (stack empty)
    return st.isEmpty();
}

Problem 3: Daily Temperatures

Problem: For an array of temperatures, find how many days you must wait for a warmer temperature.

LeetCode: #739

🔍 Algorithm Explained

This is a "Next Greater Element" problem with a twist: we need the distance, not the value.

Key Trick: Store INDICES in the stack (not values)!
This allows us to calculate: current_index - stack_top_index

🎯 The Strategy

Why Process Backwards?

We're looking for the "Next" warmer day → start from the end and work backwards.

When to Pop?

Pop days that are colder or equal (they're "useless" - blocked by current day).

💻 Java Code

/**
 * Daily Temperatures
 * LeetCode 739
 * 
 * Pattern: Next Greater Element (with distance)
 * Key: Store INDICES in stack
 * 
 * Time: O(n), Space: O(n)
 */
public int[] dailyTemperatures(int[] temperatures) {
    int n = temperatures.length;
    int[] res = new int[n]; // Default 0
    Stack<Integer> st = new Stack<>(); // Stores INDICES
    
    // Iterate backwards (Next Greater pattern)
    for (int i = n - 1; i >= 0; i--) {
        // Pop useless days (colder or equal temperature)
        while (!st.isEmpty() && temperatures[st.peek()] <= temperatures[i]) {
            st.pop();
        }
        
        // If stack not empty, top is next warmer day
        if (!st.isEmpty()) {
            res[i] = st.peek() - i; // Calculate distance
        } else {
            res[i] = 0; // No warmer day found
        }
        
        // Push current index
        st.push(i);
    }
    
    return res;
}

Problem 4: Next Greater Element II (Circular)

Problem: Find Next Greater Element in a circular array (last element connects to first).

LeetCode: #503

🔍 The Challenge

In a circular array, the last element needs to "see" the beginning elements.

Example: [1, 2, 1] is actually: ...1, 2, 1, 1, 2, 1... (infinite loop)

🎯 The Solution: Pre-loading

Key Trick:

BEFORE processing the array normally (backwards),
PRE-LOAD the stack with elements from index n-2 down to 0.

This gives the last elements a "chance" to see the beginning elements!

📊 Example: [1, 2, 3, 4, 3]

Pre-loading Phase: Push indices 3→0
0 1
1 2
2 3
3 4
4 3
Pre-load stack with VALUES from indices 3, 2, 1, 0
Stack after pre-loading: [1, 2, 3, 4]
(Bottom to Top)
Processing: Index 4 (value=3)
0 1
1 2
2 3
3 4
4 3
Pop elements ≤ 3: Pop 3, 2, 1
Top = 4 > 3 → Found! result[4] = 4
Push 3
Stack: [4, 3]
Processing: Index 3 (value=4)
0 1
1 2
2 3
3 4
4 3
Pop elements ≤ 4: Pop 3, 4
Stack empty → No greater element → result[3] = -1
Push 4
Stack: [4]

💻 Java Code

/**
 * Next Greater Element II (Circular Array)
 * LeetCode 503
 * 
 * Pattern: Next Greater with Pre-loading
 * Key: Pre-fill stack with elements n-2 to 0
 * 
 * Time: O(n), Space: O(n)
 */
public int[] nextGreaterElements(int[] nums) {
    int n = nums.length;
    int[] res = new int[n];
    Stack<Integer> st = new Stack<>();
    
    // STEP 1: Pre-load stack with elements n-2 down to 0
    // This simulates the circular nature for last elements
    for (int i = n - 2; i >= 0; i--) {
        st.push(nums[i]);
    }
    
    // STEP 2: Standard Next Greater Logic (Right to Left)
    for (int i = n - 1; i >= 0; i--) {
        // Pop smaller or equal elements
        while (!st.isEmpty() && st.peek() <= nums[i]) {
            st.pop();
        }
        
        // If empty, no greater element found
        if (st.isEmpty()) {
            res[i] = -1;
        } else {
            res[i] = st.peek();
        }
        
        // Push current element
        st.push(nums[i]);
    }
    
    return res;
}

Problem 5: Remove Adjacent Duplicates K Times

Problem: Remove duplicates only when they appear K times consecutively.

LeetCode: #1209

Example: "deeedbbcccbdaa", K=3 → "aa"

🔍 Why Simple Approach Fails

You can't just check against top because you need to COUNT occurrences.
If K=3, need to track: Is this the 1st, 2nd, or 3rd occurrence?

🎯 The Solution: Store Pairs

Key Trick:

Store (Character, Count) pairs in the stack!

• If incoming char ≠ top char → Push (char, 1)
• If incoming char = top char → Increment count
• If count reaches K → POP (explosion! remove K items)

📊 Example: "deeedbbcccbdaa", K=3

Steps 1-3: Read 'd', 'e', 'e'
Read 'd' → Push (d, 1)
Read 'e' → Push (e, 1)
Read 'e' → Match! Increment → (e, 2)

Stack: [(d,1), (e,2)]
Step 4: Read 'e' (K=3 reached!)
Read 'e' → Match top (e,2)! Increment → (e, 3)
Count = K → EXPLOSION! POP (e,3)

Stack: [(d,1)]
(All 3 e's removed at once!)
Steps 5-7: Read 'd', 'b', 'b'
Read 'd' → Match (d,1)! Increment → (d, 2)
Read 'b' → New char → Push (b, 1)
Read 'b' → Match! Increment → (b, 2)

Stack: [(d,2), (b,2)]
Steps 8-10: Read 'c', 'c', 'c' (EXPLOSION!)
Read 'c' → Push (c, 1)
Read 'c' → Increment → (c, 2)
Read 'c' → Increment → (c, 3) → POP!

Stack: [(d,2), (b,2)]
Continuing this process...
Final Stack: [(a,2)]
Result: "aa"

💻 Java Code

/**
 * Remove All Adjacent Duplicates in String II
 * LeetCode 1209
 * 
 * Pattern: Stack with Pair (Character, Count)
 * 
 * Time: O(n), Space: O(n)
 */
 
// Helper class to store (char, count) pairs
class Pair {
    char character;
    int count;
    
    Pair(char c, int count) {
        this.character = c;
        this.count = count;
    }
}

public String removeDuplicates(String s, int k) {
    Stack<Pair> st = new Stack<>();
    
    for (char c : s.toCharArray()) {
        // Case 1: Stack not empty AND matches top
        if (!st.isEmpty() && st.peek().character == c) {
            // Increment count of top element
            st.peek().count++;
            
            // If count reaches k → EXPLOSION! Remove pair
            if (st.peek().count == k) {
                st.pop();
            }
        } 
        // Case 2: Stack empty OR mismatch → Push new Pair
        else {
            st.push(new Pair(c, 1)); // Count starts at 1
        }
    }
    
    // Reconstruct string from stack
    StringBuilder sb = new StringBuilder();
    for (Pair p : st) {
        // Append character 'count' times
        for (int i = 0; i < p.count; i++) {
            sb.append(p.character);
        }
    }
    
    return sb.toString();
}

Problem 6: Monotonic Stack Pattern

Concept: Find the Previous/Next Greater/Smaller element for every element in an array.

The "Stone" Analogy: Elements are stones. Find the nearest stone that meets a criteria.

🎯 The Universal Algorithm

4 Steps

  1. Iterate: Move through array
  2. Clean Up: Pop "bad" stones (elements that can't be answer)
  3. Record: Top of stack = answer
  4. Push: Current element for future

4 Variations

  • Previous Greater: Loop 0→N, Pop smaller
  • Previous Smaller: Loop 0→N, Pop greater
  • Next Greater: Loop N-1→0, Pop smaller
  • Next Smaller: Loop N-1→0, Pop greater

📊 Example: Previous Greater Element

Array: [2, 1, 5, 6, 2, 3]

1 1
2 5

4 2
5 3
First element → No previous → result[0] = -1
Push 2
Stack: [2]
Step 2: i=1, value=1
0 2
1 1
2 5
3 6
4 2
5 3
Top = 2 > 1 → Found! result[1] = 2
Push 1
Stack: [2, 1]
Step 3: i=2, value=5
0 2
1 1

5 3
Clean up: Pop while top ≤ 5
Pop 1 (useless), Pop 2 (useless)
Stack empty → result[2] = -1
Push 5

Continuing...
Final Result: [-1, 2, -1, -1, 6, 6]

💻 Java Code

/**
 * Previous Greater Element
 * 
 * Pattern: Monotonic Stack (Decreasing)
 * Direction: Left to Right (0 → N)
 * Pop: Elements smaller or equal
 * 
 * Time: O(n), Space: O(n)
 */
public int[] previousGreaterElement(int[] arr) {
    int n = arr.length;
    int[] res = new int[n];
    Stack<Integer> st = new Stack<>();
    
    // First element has no previous greater
    res[0] = -1;
    st.push(arr[0]);
    
    for (int i = 1; i < n; i++) {
        // CLEAN UP: Pop useless elements (smaller or equal)
        // Why useless? They can't be the answer for current
        // or future elements because current blocks them
        while (!st.isEmpty() && st.peek() <= arr[i]) {
            st.pop();
        }
        
        // RECORD: If empty, no greater found. Else top is answer.
        res[i] = st.isEmpty() ? -1 : st.peek();
        
        // PUSH: Current element for future elements
        st.push(arr[i]);
    }
    
    return res;
}

🎨 The 4 Variations Reference

// Previous Greater: Loop LEFT→RIGHT, Pop SMALLER
for (int i = 0; i < n; i++) {
    while (!st.empty() && st.peek() <= arr[i]) st.pop();
    res[i] = st.empty() ? -1 : st.peek();
    st.push(arr[i]);
}

// Previous Smaller: Loop LEFT→RIGHT, Pop GREATER  
for (int i = 0; i < n; i++) {
    while (!st.empty() && st.peek() >= arr[i]) st.pop();
    res[i] = st.empty() ? -1 : st.peek();
    st.push(arr[i]);
}

// Next Greater: Loop RIGHT→LEFT, Pop SMALLER
for (int i = n-1; i >= 0; i--) {
    while (!st.empty() && st.peek() <= arr[i]) st.pop();
    res[i] = st.empty() ? -1 : st.peek();
    st.push(arr[i]);
}

// Next Smaller: Loop RIGHT→LEFT, Pop GREATER
for (int i = n-1; i >= 0; i--) {
    while (!st.empty() && st.peek() >= arr[i]) st.pop();
    res[i] = st.empty() ? -1 : st.peek();
    st.push(arr[i]);
}

📝 Summary

The Key Question:

"Do I need to process items and look back at the most recently processed valid item?"

If YES, use a Stack:

  • In Strings: Retrieve last valid character after deletions
  • In Parentheses: Retrieve last unclosed opening bracket
  • In Next Greater: Retrieve nearest candidate not "blocked" by larger number
  • In Reversal: LIFO naturally reverses order
