🎯 Path Sum II - Backtracking

Finding All Root-to-Leaf Paths with Target Sum

Master the backtracking pattern for tree problems

📋 Problem Statement

What We Need to Find:

Given a Binary Tree and a target sum, return all root-to-leaf paths where the sum of node values equals the target.

Important Points:

  • Root-to-Leaf: Path must start at root and end at a leaf node
  • Leaf node: A node with no children (both left and right are null)
  • Return all paths: Not just one, but all paths that match
  • Path values: List of node values in order from root to leaf

🔄 Understanding Backtracking

🎯 What is Backtracking?

Backtracking is like exploring a maze:

1. Choose: Pick a path and walk down it
2. Explore: See where it leads
3. Check: Did we find what we're looking for?
4. Backtrack: Return and try a different path

🎮 The Three Steps of Backtracking:

Step 1: Make a Choice

Add the current node to our path
currentPath.add(node.val)

Step 2: Explore

If it's a leaf, check if sum equals target
If not a leaf, recursively explore children

Step 3: Backtrack (Undo)

Remove the current node from path before returning
currentPath.remove(currentPath.size() - 1)

✨ Why Backtrack?

We use a single list (currentPath) to build all paths.
After exploring one branch, we need to remove nodes to try other branches.

Without backtracking, our path would keep growing and never reset!

💡 The Algorithm

🎯 High-Level Strategy:

  1. Start at root with an empty path
  2. At each node:
    • Add current node to path
    • Subtract node value from remaining sum
    • If it's a leaf AND sum is 0 → Found a valid path!
    • If not a leaf → Explore left and right children
    • Remove current node from path (backtrack)
  3. Collect all valid paths in a result list

🔑 Key Insight:

Instead of checking if sum == target,
we subtract each node's value and check if remaining == 0 at leaves.

This is cleaner and avoids passing the original target everywhere!

💻 Java Implementation

import java.util.*;

class Solution {
    public List<List<Integer>> pathSum(Node root, int targetSum) {
        List<List<Integer>> result = new ArrayList<>();
        List<Integer> currentPath = new ArrayList<>();
        
        backtrack(root, targetSum, currentPath, result);
        
        return result;
    }
    
    private void backtrack(Node node, int remainingSum, 
                           List<Integer> currentPath, 
                           List<List<Integer>> result) {
        // BASE CASE: Null node
        if (node == null) return;
        
        // STEP 1: CHOOSE - Add current node to path
        currentPath.add(node.val);
        
        // STEP 2: EXPLORE - Check if we found a valid path
        if (node.left == null && node.right == null) {
            // It's a leaf node - check if sum matches
            if (remainingSum == node.val) {
                // Found a valid path! Save it
                result.add(new ArrayList<>(currentPath));
            }
        } else {
            // Not a leaf - explore children
            backtrack(node.left, remainingSum - node.val, currentPath, result);
            backtrack(node.right, remainingSum - node.val, currentPath, result);
        }
        
        // STEP 3: BACKTRACK - Remove current node before returning
        currentPath.remove(currentPath.size() - 1);
    }
}

// Example usage:
public static void main(String[] args) {
    Solution sol = new Solution();
    
    // Build tree:      5
    //                /   \
    //               4     8
    //              /     / \
    //             11    13  4
    //            / \         \
    //           7   2         1
    
    Node root = new Node(5);
    root.left = new Node(4);
    root.right = new Node(8);
    root.left.left = new Node(11);
    root.left.left.left = new Node(7);
    root.left.left.right = new Node(2);
    root.right.left = new Node(13);
    root.right.right = new Node(4);
    root.right.right.right = new Node(1);
    
    List<List<Integer>> result = sol.pathSum(root, 22);
    System.out.println(result);  // Output: [[5, 4, 11, 2]]
}

🎯 Code Explanation:

  • Line 5-6: Initialize result list and current path
  • Line 17: Base case - null node, just return
  • Line 20: CHOOSE - Add current node to path
  • Line 23-27: If leaf node, check if sum matches target
  • Line 26: CRITICAL - Make a copy of currentPath before adding to result!
  • Line 29-31: If not leaf, recursively explore children
  • Line 35: BACKTRACK - Remove current node before returning

⚠️ Common Mistake - MUST Make a Copy!

// WRONG:
result.add(currentPath);  ❌

// This adds a REFERENCE to currentPath
// When we backtrack and modify currentPath, the result changes too!
// Final result would be all empty arrays!

// CORRECT:
result.add(new ArrayList<>(currentPath));  ✅

// This creates a NEW list with the same values
// The copy stays intact even when we backtrack

⚡ Complexity Analysis

Metric Complexity Explanation
Time O(N²) Visit each node once (N), but copying paths takes O(N) time each
Space O(N) Recursion stack (H) + current path (N) = O(N)

📊 Why O(N²) Time?

We visit N nodes → O(N)
But when we find a valid path, we copy it → O(N) per copy

Worst case: Every leaf is a valid path
In a complete binary tree with N nodes, there are ~N/2 leaves
Copying each path: O(N/2) × O(N) = O(N²)

Average case: Usually much better, closer to O(N)

📊 Space Breakdown:

  • Recursion stack: O(H) where H = height of tree
  • Current path: O(N) in worst case (all nodes on one path)
  • Result storage: Not counted in space complexity (it's the output)

🎯 The Backtracking Pattern

✨ General Backtracking Template:

void backtrack(state, choices) {
    // BASE CASE
    if (reached_goal(state)) {
        result.add(copy(state));  // IMPORTANT: Make a copy!
        return;
    }
    
    // EXPLORE all choices
    for (choice : choices) {
        // 1. CHOOSE - Make a choice
        state.add(choice);
        
        // 2. EXPLORE - Recursively explore
        backtrack(state, next_choices);
        
        // 3. BACKTRACK - Undo the choice
        state.remove(choice);
    }
}

🎯 Applying to Path Sum:

  • State: currentPath (list of nodes so far)
  • Choices: left child or right child
  • Goal: Reach a leaf with sum = target
  • Choose: Add current node to path
  • Explore: Recursively visit children
  • Backtrack: Remove current node from path

✨ Key Takeaways

Remember These Forever:

  1. Backtracking = Choose, Explore, Undo
  2. Add node to path BEFORE recursion
  3. Remove node from path AFTER recursion (backtrack)
  4. Check sum only at leaf nodes (both children null)
  5. MUST copy the path before adding to result
  6. Subtract as you go: remainingSum - node.val
  7. Use same list for all paths (memory efficient)
  8. Time: O(N²) worst case, O(N) average
  9. Space: O(N) for recursion + current path

🧠 Memory Trick:

"Hiking with Breadcrumbs"

Imagine hiking through a forest (tree):
• Drop a breadcrumb (add to path) when entering a trail
• If you reach a treasure (leaf with correct sum), mark the route!
• Pick up the breadcrumb (backtrack) when leaving that trail
• Try another path with fresh breadcrumbs

You use the SAME bag of breadcrumbs for all paths!

🎓 Interview Tips:

  • Start by explaining the backtracking pattern
  • Draw a small tree and trace through it step by step
  • Emphasize the importance of making a copy when saving paths
  • Mention that we only check sum at leaf nodes, not internal nodes
  • Be ready to handle edge cases: empty tree, negative values
  • Discuss the difference from Path Sum I (which just returns boolean)

⚠️ Common Pitfalls:

  • ❌ Forgetting to backtrack (remove node from path)
  • ❌ Not making a copy when adding path to result
  • ❌ Checking sum at non-leaf nodes
  • ❌ Checking if node.val == remainingSum instead of at leaves
  • ❌ Modifying the tree structure (never needed for this problem)
