🔍 Binary Search Tree (BST) Search

O(log N) Efficient Search Algorithm

How to find a value in a BST in logarithmic time

📋 Problem Statement

What We Need to Solve:

Given a Binary Search Tree (BST) and a target value, find and return the node containing that value. If the value doesn't exist, return null.

Example:

BST: [10, 5, 15, 3, 7, 12, 18]

Target: 12

Return: Node with value 12

🎯 What is a Binary Search Tree (BST)?

A BST is a binary tree with a special property:

  • Left child is ALWAYS smaller than parent
  • Right child is ALWAYS larger than parent
  • This property applies to every node in the tree

🌳 Example BST

                    ┌─────┐
                    │ 10  │  ← Root
                    └──┬──┘
          ┌────────────┴────────────┐
       ┌──┴──┐                   ┌──┴──┐
       │  5  │                   │ 15  │
       └──┬──┘                   └──┬──┘
     ┌────┴────┐              ┌────┴────┐
  ┌──┴──┐   ┌──┴──┐        ┌──┴──┐   ┌──┴──┐
  │  3  │   │  7  │        │ 12  │   │ 18  │
  └─────┘   └─────┘        └─────┘   └─────┘

BST Property: All LEFT < Parent < All RIGHT

💡 Algorithm Intuition

🎯 Key Insight: It's Like Binary Search in a Sorted Array!

When you search in a sorted array, you compare with the middle element:

  • If target < middle → Search LEFT half (eliminate right half)
  • If target > middle → Search RIGHT half (eliminate left half)
  • If target == middle → Found it!

BST search works EXACTLY the same way!

Binary Search (Array)

Array: [3, 5, 7, 10, 12, 15, 18]
Target: 12

Step 1: Middle = 10
  12 > 10 → Go RIGHT
  
Step 2: Middle = 15
  12 < 15 → Go LEFT
  
Step 3: Found 12!

BST Search (Tree)

Tree: Same values
Target: 12

Step 1: Root = 10
  12 > 10 → Go RIGHT
  
Step 2: Node = 15
  12 < 15 → Go LEFT
  
Step 3: Found 12!

✨ The Power of BST Search:

At each step, we eliminate HALF the tree by making a single comparison!

This is why BST search is O(log N) - same as binary search!

🔄 Step-by-Step Execution

Let's search for target = 12 in our BST

Step 1
              10 ← Current node
              / \
             5   15
            / \  / \
           3  7 12 18
→
Compare: target (12) vs root (10)
Result: 12 > 10
Decision: Go RIGHT
Why: Target is larger, so it MUST be in right subtree

❌ Eliminated:

Entire LEFT subtree (5, 3, 7)

We'll never look at these nodes!

Step 2
               10
              / \
          5   15 ← Current node
         / \  / \
        3  7 12 18
→
Compare: target (12) vs current (15)
Result: 12 < 15
Decision: Go LEFT
Why: Target is smaller, so it MUST be in left subtree

❌ Eliminated:

Right child (18)

Step 3
               10
              / \
          5   15
         / \  / \
        3  7 12 18 ← FOUND!
→
Compare: target (12) vs current (12)
Result: 12 == 12
Decision: FOUND! Return this node
✅ SUCCESS! Returned node with value 12

Total comparisons: 3 (out of 7 nodes)
Nodes visited: 3 (43% of tree)

💻 Java Implementation

class Node {
    int data;
    Node left, right;
    
    Node(int data) {
        this.data = data;
        this.left = this.right = null;
    }
}

class BSTSearch {
    
    // Recursive approach
    Node searchBST(Node root, int target) {
        // BASE CASE 1: Empty tree or not found
        if (root == null) return null;
        
        // BASE CASE 2: Found the target!
        if (root.data == target) return root;
        
        // RECURSIVE CASE: Target is smaller → Go LEFT
        if (target < root.data) {
            return searchBST(root.left, target);
        }
        
        // RECURSIVE CASE: Target is larger → Go RIGHT
        return searchBST(root.right, target);
    }
    
    // Iterative approach (more efficient - no recursion stack)
    Node searchBSTIterative(Node root, int target) {
        Node current = root;
        
        while (current != null) {
            if (current.data == target) {
                return current;  // Found!
            } else if (target < current.data) {
                current = current.left;   // Go left
            } else {
                current = current.right;  // Go right
            }
        }
        
        return null;  // Not found
    }
    
    public static void main(String[] args) {
        BSTSearch bst = new BSTSearch();
        
        // Build BST:      10
        //                /  \
        //               5    15
        //              / \   / \
        //             3  7  12 18
        
        Node root = new Node(10);
        root.left = new Node(5);
        root.right = new Node(15);
        root.left.left = new Node(3);
        root.left.right = new Node(7);
        root.right.left = new Node(12);
        root.right.right = new Node(18);
        
        // Test searches
        Node result1 = bst.searchBST(root, 12);
        System.out.println(result1 != null ? "Found: " + result1.data : "Not found");
        // Output: Found: 12
        
        Node result2 = bst.searchBST(root, 6);
        System.out.println(result2 != null ? "Found: " + result2.data : "Not found");
        // Output: Not found
    }
}

🎯 Code Explanation:

  • Base Case 1: if (root == null) - Reached end without finding, return null
  • Base Case 2: if (root.data == target) - Found it! Return this node
  • Recursive Case 1: if (target < root.data) - Go LEFT, eliminate right subtree
  • Recursive Case 2: else - Go RIGHT, eliminate left subtree

⚡ Complexity Analysis

Case Time Complexity Space Complexity Explanation
Best Case O(1) O(1) Target is at root - found immediately
Average Case O(log N) O(log N) recursive
O(1) iterative		 Balanced tree - eliminate half at each step
Worst Case O(N) O(N) recursive
O(1) iterative		 Skewed tree (becomes linked list)

📊 Why O(log N) for Balanced Tree?

At each step, we eliminate HALF the remaining tree:
• Start: N nodes
• After step 1: N/2 nodes remain
• After step 2: N/4 nodes remain
• After step 3: N/8 nodes remain
• ...
• After k steps: N/(2^k) nodes remain

When N/(2^k) = 1, we've found the target.
Solving: k = log₂(N) ✅

🏆 BST Search vs Linear Search

Aspect Linear Search (Array) BST Search
Time (Worst) O(N) O(log N) for balanced tree
Nodes Checked All N nodes Only log N nodes
Example (1000 nodes) Check up to 1000 nodes Check only ~10 nodes
Example (1,000,000 nodes) Check up to 1,000,000 nodes Check only ~20 nodes

💪 The Power of BST:

For 1 million nodes:
• Linear search: Up to 1,000,000 comparisons
• BST search: Only ~20 comparisons
That's 50,000x faster! 🚀

✨ Key Takeaways

Remember These Forever:

  1. BST Property: Left < Parent < Right (for every node)
  2. Search Strategy: Compare with current, go left if smaller, right if larger
  3. Base Cases: Found (return node) or null (not found)
  4. Efficiency: O(log N) for balanced tree - eliminate half each step
  5. Like Binary Search: Same principle, just on a tree structure
  6. Two Implementations: Recursive (simpler) or Iterative (no stack space)
  7. Worst Case: O(N) for skewed tree (becomes linked list)
  8. Returns: The node itself (not just true/false)
