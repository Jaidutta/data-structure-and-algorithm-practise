🌲 LCA in Binary Search Tree

The "Split Point" Method

Using BST properties for O(log N) solution

📋 Problem Statement

What We Need to Solve:

Given a Binary Search Tree (BST) and two nodes P and Q, find their Lowest Common Ancestor (LCA).

Key Difference from Binary Tree:

In a BST, we can use the ordering property to find LCA much more efficiently!
We don't need to visit the entire tree.

🎯 BST Property Reminder:

  • Left subtree values are ALL smaller than parent
  • Right subtree values are ALL larger than parent
  • This property holds for every node in the tree

🌳 Example BST

                    │  6  │  ← Root
                    └──┬──┘
          ┌────────────┴────────────┐
       ┌──┴──┐                   ┌──┴──┐

  ┌──┴──┐   ┌──┴──┐        ┌──┴──┐   ┌──┴──┐

All left < Parent < All right

🔀 The "Split Point" Concept

💡 The Key Insight:

The LCA is the first node where the paths to P and Q diverge (split).

Think of it as a fork in the road:
• Before the fork, P and Q travel together
• At the fork, they go different directions
• The fork is the LCA!

🎯 Three Cases to Consider:

Case 1: Both P and Q are SMALLER than current node

Meaning: Both P and Q are in the LEFT subtree
Action: Go LEFT (continue search in left subtree)
Why: LCA must also be in left subtree

Case 2: Both P and Q are LARGER than current node

Meaning: Both P and Q are in the RIGHT subtree
Action: Go RIGHT (continue search in right subtree)
Why: LCA must also be in right subtree

Case 3: One is SMALLER, one is LARGER (or current node is one of them)

Meaning: P and Q are in DIFFERENT subtrees (SPLIT!)
Action: Current node is the LCA!
Why: This is the split point - paths diverge here

✨ The Beauty of BST:

We don't need to search the entire tree!
We follow a single path from root, making O(log N) comparisons instead of O(N).

💻 Java Implementation

🎯 Two Approaches:

We can solve this either recursively or iteratively.
Both are simple and efficient!

Approach 1: Recursive Solution

class Solution {
    public Node lowestCommonAncestor(Node root, Node p, Node q) {
        // BASE CASE: If we reach null, no LCA found (shouldn't happen with valid input)
        if (root == null) return null;
        
        // CASE 1: Both P and Q are SMALLER than current
        // Meaning: Both are in LEFT subtree
        if (p.data < root.data && q.data < root.data) {
            return lowestCommonAncestor(root.left, p, q);
        }
        
        // CASE 2: Both P and Q are LARGER than current
        // Meaning: Both are in RIGHT subtree
        if (p.data > root.data && q.data > root.data) {
            return lowestCommonAncestor(root.right, p, q);
        }
        
        // CASE 3: SPLIT POINT!
        // Either: One left, one right OR current is one of the targets
        return root;
    }
}

Approach 2: Iterative Solution (More Efficient)

class Solution {
    public Node lowestCommonAncestor(Node root, Node p, Node q) {
        Node current = root;
        
        while (current != null) {
            // Both in left subtree
            if (p.data < current.data && q.data < current.data) {
                current = current.left;
            }
            // Both in right subtree
            else if (p.data > current.data && q.data > current.data) {
                current = current.right;
            }
            // Split point found!
            else {
                return current;
            }
        }
        
        return null;  // Should never reach here with valid input
    }
}

Complete Example with Test:

class Node {
    int data;
    Node left, right;
    
    Node(int data) {
        this.data = data;
        this.left = this.right = null;
    }
}

public static void main(String[] args) {
    Solution sol = new Solution();
    
    // Build BST:      6
    //               /   \
    //              2     8
    //             / \   / \
    //            0  4  7  9
    //              / \
    //             3   5
    
    Node root = new Node(6);
    root.left = new Node(2);
    root.right = new Node(8);
    root.left.left = new Node(0);
    root.left.right = new Node(4);
    root.left.right.left = new Node(3);
    root.left.right.right = new Node(5);
    root.right.left = new Node(7);
    root.right.right = new Node(9);
    
    // Test cases
    Node p = root.left;           // Node 2
    Node q = root.right;          // Node 8
    Node lca = sol.lowestCommonAncestor(root, p, q);
    System.out.println("LCA(2, 8) = " + lca.data);  // Output: 6
    
    p = root.left;                // Node 2
    q = root.left.right;          // Node 4
    lca = sol.lowestCommonAncestor(root, p, q);
    System.out.println("LCA(2, 4) = " + lca.data);  // Output: 2
}

🎯 Code Explanation:

  • Line 8-10: If both nodes smaller, LCA must be in left subtree
  • Line 14-16: If both nodes larger, LCA must be in right subtree
  • Line 20: Otherwise, we found the split point!
  • Iterative version: More efficient - no recursion stack overhead

⚡ Complexity Analysis

Approach Time Complexity Space Complexity Explanation
Recursive O(H) O(H) H = height of tree; visit nodes along one path; recursion stack
Iterative O(H) O(1) H = height of tree; visit nodes along one path; no extra space

📊 What is H (Height)?

Balanced BST: H = log(N) → Time = O(log N) ✅
Skewed BST: H = N → Time = O(N) ❌

For a balanced BST, this is much faster than the O(N) binary tree approach!

🏆 BST LCA vs Binary Tree LCA

Aspect Binary Tree LCA BST LCA
Approach Count targets in subtrees Compare values, find split point
Nodes Visited All nodes (entire tree) Nodes along one path only
Time (Best) O(N) O(log N) for balanced tree
Time (Worst) O(N) O(N) for skewed tree
Space (Recursive) O(H) O(H)
Space (Iterative) Not applicable O(1)
Uses BST Property No Yes!

✨ Key Advantage:

For a balanced BST:
• Binary Tree LCA: O(N) - visit all nodes
• BST LCA: O(log N) - visit only path nodes

For 1,000,000 nodes:
Binary Tree: 1,000,000 nodes visited
BST: ~20 nodes visited
That's 50,000x faster! 🚀

✨ Key Takeaways

Remember These Forever:

  1. Use BST property! Compare values to decide direction
  2. Three cases: Both left, both right, or split
  3. Split point = LCA: When paths diverge
  4. Follow one path only: Don't need to visit entire tree
  5. Time: O(H) = O(log N) for balanced BST
  6. Iterative is better: O(1) space vs O(H) for recursive
  7. Much faster than Binary Tree: O(log N) vs O(N)
  8. Simple logic: Just 3 if-else cases

🧠 Memory Trick:

"The Fork in the Road"

Imagine two people (P and Q) walking from the root:
• If both go LEFT → Not split yet, keep going left
• If both go RIGHT → Not split yet, keep going right
• If one goes LEFT and one goes RIGHT → FORK! This is the LCA!

🎯 Decision Flow (Simplified):

while (current != null) {
    if (both < current) {
        go left;
    } else if (both > current) {
        go right;
    } else {
        return current;  // SPLIT!
    }
}

⚠️ Common Mistakes to Avoid:

  • Don't use the Binary Tree LCA approach - it wastes the BST property!
  • Don't forget the case where current node is one of the targets
  • Don't traverse both subtrees - we only need one path!
  • Remember: This ONLY works for BST, not regular Binary Tree
