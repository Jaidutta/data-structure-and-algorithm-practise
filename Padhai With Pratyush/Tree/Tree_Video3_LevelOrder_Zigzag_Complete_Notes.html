🌊 Level Order & Zigzag Traversal

Breadth-First Search (BFS) on Trees

Visit nodes level by level using a Queue

📋 Problem Statement

What We Need to Solve:

Given a Binary Tree, visit nodes level by level (horizontally) instead of following branches vertically.

Two Variations:

  1. Level Order: Visit each level Left → Right
  2. Zigzag: Alternate direction each level (L→R, then R→L, then L→R...)

🌳 Example Tree

                    │  1  │  ← Level 0 (L0)
                    └──┬──┘

📦 The Queue Concept (FIFO)

🎯 What is a Queue?

A Queue is a First-In-First-Out (FIFO) data structure, like a line at a coffee shop:

  • First person in line gets served first
  • add() - Join the back of the line
  • poll() - Remove from the front of the line

🔄 Level Order Algorithm Explained

🎯 The Strategy:

  1. Start with root in queue
  2. While queue is not empty:
    • Count how many nodes in queue (size)
    • Process exactly that many nodes (one level)
    • Add each node's children to queue (for next level)
  3. Repeat until queue is empty

💡 The KEY Insight: The "size" Variable

Why do we need size?

The queue contains nodes from CURRENT level AND NEXT level mixed together.
size tells us exactly how many nodes belong to the CURRENT level.

We process exactly size nodes, then move to the next level.

⚡ Zigzag Traversal Explained

🎯 The Zigzag Pattern:

Instead of always going Left → Right, we alternate direction each level:

  • Level 0: Left → Right (normal)
  • Level 1: Right → Left (reversed!)
  • Level 2: Left → Right (normal)
  • Level 3: Right → Left (reversed!)
  • And so on...

       4  5 6  7       L2: [4,5,6,7]   (L→R)

           1
          / \

        / \ / \        L1: [3, 2]      (R→L) ← REVERSED!
       4  5 6  7       L2: [4,5,6,7]   (L→R)

ZIGZAG:     [[1], [3,2], [4,5,6,7]]

💡 How to Implement Zigzag:

Method 1: Use a boolean flag leftToRight

  • Start with leftToRight = true
  • If leftToRight, fill array from index 0 → size-1 (normal)
  • If NOT leftToRight, fill array from index size-1 → 0 (reversed)
  • After each level, flip the flag: leftToRight = !leftToRight

💻 Java Implementation

Level Order Traversal:

import java.util.*;

class Solution {
    public List<List<Integer>> levelOrder(Node root) {
        List<List<Integer>> result = new ArrayList<>();
        if (root == null) return result;
        
        Queue<Node> queue = new LinkedList<>();
        queue.add(root);  // Start with root
        
        while (!queue.isEmpty()) {
            int size = queue.size();  // CRITICAL: nodes in current level
            List<Integer> currentLevel = new ArrayList<>();
            
            // Process exactly 'size' nodes (one complete level)
            for (int i = 0; i < size; i++) {
                Node current = queue.poll();  // Remove from front
                currentLevel.add(current.data);
                
                // Add children to queue (for next level)
                if (current.left != null) {
                    queue.add(current.left);
                }
                if (current.right != null) {
                    queue.add(current.right);
                }
            }
            
            result.add(currentLevel);
        }
        
        return result;
    }
}

Zigzag Traversal:

public List<List<Integer>> zigzagLevelOrder(Node root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    
    Queue<Node> queue = new LinkedList<>();
    queue.add(root);
    boolean leftToRight = true;  // Direction flag
    
    while (!queue.isEmpty()) {
        int size = queue.size();
        Integer[] level = new Integer[size];  // Use array for zigzag
        
        for (int i = 0; i < size; i++) {
            Node current = queue.poll();
            
            // KEY LOGIC: Decide index based on direction
            int index = leftToRight ? i : (size - 1 - i);
            level[index] = current.data;
            
            // Add children (same as level order)
            if (current.left != null) queue.add(current.left);
            if (current.right != null) queue.add(current.right);
        }
        
        result.add(Arrays.asList(level));
        leftToRight = !leftToRight;  // Flip direction for next level
    }
    
    return result;
}

🎯 Zigzag Index Logic Explained:

For level with size = 4:

LEFT-TO-RIGHT (leftToRight = true):
  i=0 → index = 0  → level[0] = node
  i=1 → index = 1  → level[1] = node
  i=2 → index = 2  → level[2] = node
  i=3 → index = 3  → level[3] = node
  Result: [node0, node1, node2, node3]

RIGHT-TO-LEFT (leftToRight = false):
  i=0 → index = 3  → level[3] = node
  i=1 → index = 2  → level[2] = node
  i=2 → index = 1  → level[1] = node
  i=3 → index = 0  → level[0] = node
  Result: [node3, node2, node1, node0] ← REVERSED!

⚡ Complexity Analysis

Algorithm Time Complexity Space Complexity Explanation
Level Order O(N) O(W) Visit each node once; W = max width of tree (max nodes in queue)
Zigzag O(N) O(W) Same as level order; direction doesn't affect complexity

📊 Understanding Space Complexity O(W):

W = Maximum width of tree (max nodes at any level)

Best case: Skewed tree → W = 1
Worst case: Perfect binary tree → W = N/2 (last level)
Average case: Balanced tree → W = N/2 

Perfect Binary Tree (7 nodes):
       1
      / \
     2   3        W at L0 = 1
    / \ / \       W at L1 = 2
   4  5 6  7      W at L2 = 4 ← Maximum

Max W = 4 (which is 7/2 rounded up)
Space = O(4) = O(N/2) = O(N)

🎯 Real-World Use Cases

When to Use Level Order Traversal:

  • Find shortest path in unweighted tree (BFS guarantees shortest)
  • Print tree level by level (each level on new line)
  • Find level of a node (which generation)
  • Check if tree is complete (no gaps before leaves)
  • Serialize/deserialize tree (save to file)
  • Connect nodes at same level (right pointers)
  • Find cousins (nodes at same level, different parents)

When to Use Zigzag:

  • Display tree in a zigzag pattern (aesthetic UI)
  • Spiral order traversal (specific interview question)
  • Alternative representation for visualization

✨ Key Takeaways

Remember These Forever:

  1. Use a Queue (FIFO) - First in, first out
  2. The "size" variable is KEY - tells us nodes in current level
  3. Process exactly size nodes per level
  4. Add children while processing - they're for next level
  5. Time: O(N) - visit every node once
  6. Space: O(W) - width of tree (max nodes in queue)
  7. Zigzag uses boolean flag - flip direction each level
  8. Zigzag index formula: leftToRight ? i : (size - 1 - i)
  9. BFS vs DFS: Level order is BFS (breadth-first search)

🧠 Quick Memory Aid:

"Generation by Generation"
• Grandparents (Level 0)
• Parents (Level 1)
• Children (Level 2)
• Grandchildren (Level 3)

Queue ensures FIFO: Older generation leaves before younger enters!

🔑 The Most Important Line:

int size = queue.size();

Without this, you can't separate levels!
This ONE line makes level-order traversal work.
