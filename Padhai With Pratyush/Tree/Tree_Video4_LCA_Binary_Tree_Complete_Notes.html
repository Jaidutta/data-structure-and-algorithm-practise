👔 Lowest Common Ancestor (LCA)

The "Boss Count" Method

Finding the lowest common ancestor in a Binary Tree

📋 Problem Statement

What We Need to Solve:

Given a Binary Tree and two nodes P and Q, find their Lowest Common Ancestor (LCA).

What is LCA?

The LCA is the lowest node (deepest in the tree) that has both P and Q as descendants.
Note: A node can be a descendant of itself.

Example 1: LCA(5, 1) = ?

Answer: Node 3

5 is in left subtree, 1 is in right subtree. They meet at 3.

Example 2: LCA(5, 4) = ?

Answer: Node 5

4 is descendant of 5. A node can be LCA of itself and descendant.

👔 The "Boss Count" Analogy

🏢 The Company Hierarchy Metaphor

Imagine a company org chart (tree structure):

• CEO at the top (root)
• Managers below (internal nodes)
• Individual contributors at leaves

We want to find the lowest-level manager who is the boss of BOTH employee P and employee Q.

🎯 The Algorithm Intuition:

We ask every node in the tree: "How many of the two target employees (P or Q) are you the boss of?"

Possible answers:

  • 0: Neither P nor Q work under me
  • 1: Either P or Q works under me (but not both)
  • 2: BOTH P and Q work under me! 🎯

The FIRST node (starting from bottom) that answers "2" is the LCA!

💡 How Do We Count?

Each node counts by asking three questions:

  1. Am I one of the targets? (Am I P or Q?) → self = 1 or 0
  2. How many does my left child see? → left = 0, 1, or 2
  3. How many does my right child see? → right = 0, 1, or 2

Total Count = left + right + self

If total == 2 → This is the LCA! ✅

🔄 Recursion Flow Visualization

🎯 Bottom-Up Processing:

Notice how we start from the LEAVES and work our way UP to the root. This is postorder-style recursion!

  │
  ├─ countTargets(5)  ← Returns 1
  │    │
  │    ├─ countTargets(6) ← Returns 0
  │    │
  │    └─ countTargets(2) ← Returns 0

       │

Summary:
• Leaf nodes return 0 (not targets)
• Node 5 returns 1 (is target P)
• Node 1 returns 1 (is target Q)
• Node 3 receives 1 from left, 1 from right → Total = 2 → LCA!

💻 Java Implementation

class Solution {
    Node lca = null;  // Store the LCA when found
    
    public Node lowestCommonAncestor(Node root, Node p, Node q) {
        countTargets(root, p, q);
        return lca;
    }
    
    // Returns: How many targets (P or Q) are in this subtree
    private int countTargets(Node root, Node p, Node q) {
        // BASE CASE: Null node sees 0 targets
        if (root == null) return 0;
        
        // Ask left subtree: How many targets do you see?
        int left = countTargets(root.left, p, q);
        
        // Ask right subtree: How many targets do you see?
        int right = countTargets(root.right, p, q);
        
        // Am I one of the targets?
        int self = (root == p || root == q) ? 1 : 0;
        
        // Total count = left + right + self
        int total = left + right + self;
        
        // If I see both targets (total == 2) and LCA not found yet
        if (total == 2 && lca == null) {
            lca = root;  // I'm the LCA!
        }
        
        // Return my count to parent
        return total;
    }
}

// Example Usage:
public static void main(String[] args) {
    Solution sol = new Solution();
    
    // Build the tree from our example
    Node root = new Node(3);
    root.left = new Node(5);
    root.right = new Node(1);
    root.left.left = new Node(6);
    root.left.right = new Node(2);
    root.left.right.left = new Node(7);
    root.left.right.right = new Node(4);
    root.right.left = new Node(0);
    root.right.right = new Node(8);
    
    Node p = root.left;       // Node 5
    Node q = root.right;      // Node 1
    
    Node result = sol.lowestCommonAncestor(root, p, q);
    System.out.println("LCA: " + result.data);  // Output: LCA: 3
}

🎯 Code Explanation:

  • Line 1: Global variable to store LCA when found
  • Line 9: Base case - null node contributes 0
  • Line 12-15: Ask both children for their counts (postorder!)
  • Line 18: Check if current node is one of the targets
  • Line 21: Add up all three counts
  • Line 24-26: If total is 2 and LCA not set, current node is LCA
  • Line 29: Return count to parent for their calculation

💡 Why Check "lca == null"?

Once we find the LCA, we want to keep it! Higher nodes might also have total == 2, but we want the LOWEST (first one found from bottom-up).

Example: If LCA is node 5, then node 3 will also see both targets. But we already found 5 first (it's lower), so we keep it.

⚡ Complexity Analysis

Metric Complexity Explanation
Time O(N) Visit every node once in worst case
Space O(H) Recursion stack depth = height of tree

📊 Why O(N) Time?

In the worst case, we visit every single node to count targets.

Best case: If LCA is near root, we might not visit all nodes, but we still analyze this as O(N) for worst case.

Why not stop early? We could optimize to stop once LCA is found, but the basic algorithm visits all nodes.

🔄 Alternative Approach (Classic Method)

📝 The Traditional Approach:

There's another popular way to solve this without counting: 

// Classic LCA approach (returns the LCA node directly)
public Node lowestCommonAncestor(Node root, Node p, Node q) {
    // BASE CASE: Empty tree or found one of the targets
    if (root == null || root == p || root == q) {
        return root;
    }
    
    // Search in left and right subtrees
    Node left = lowestCommonAncestor(root.left, p, q);
    Node right = lowestCommonAncestor(root.right, p, q);
    
    // If both sides found something, current node is LCA
    if (left != null && right != null) {
        return root;
    }
    
    // Otherwise return whichever side found something
    return left != null ? left : right;
}

🎯 Comparison:

Aspect Boss Count Method Classic Method
Intuition Count targets in subtrees Return nodes when found
Code Length Longer (counting logic) Shorter (simpler)
Clarity Very explicit (count = 2) More abstract
Complexity O(N) time, O(H) space O(N) time, O(H) space

Both work perfectly! The Boss Count method is easier to understand conceptually.

✨ Key Takeaways

Remember These Forever:

  1. LCA = Lowest Common Ancestor - deepest node that has both targets as descendants
  2. Boss Count Method: Each node counts how many targets in its subtree
  3. Count Formula: total = left + right + self
  4. LCA Condition: First node with count == 2
  5. Bottom-Up Processing: Start from leaves, work up (postorder style)
  6. A node can be LCA of itself: If one target is ancestor of the other
  7. Time: O(N) - visit every node once
  8. Space: O(H) - recursion stack depth
  9. Check "lca == null": Keep the FIRST (lowest) node found

"Who's Your Boss?"

Ask every employee (node): "How many of these 2 people work under you?"

Count = 0: Neither works for me

🎯 Common Interview Follow-Ups:

  • What if P or Q doesn't exist in the tree?
    → Need to verify both exist first (separate traversal)
  • What if there are more than 2 nodes?
    → Find LCA of all pairs, then find LCA of those results
  • Can you do it iteratively?
    → Yes, but requires storing paths from root to P and Q, then comparing
