📋 Problem Statement

What We Need to Solve:

Given a Binary Tree, visit every node in three different orders:

🔵

INORDER

Left → Node → Right

🔴

PREORDER

Node → Left → Right

🟢

POSTORDER

Left → Right → Node

🍫 The Chocolate Rule (Simple Analogy)

Imagine: A Parent Gets ONE Chocolate

The parent must decide: When do I eat it? vs. When do my children get their turn?

The family rule determines the order. This is exactly how tree traversals work!

🔵 INORDER Rule

"Give to LEFT child first, then I EAT, then give to RIGHT child"

1. LEFT child eats first

2. Parent EATS (prints)

3. RIGHT child eats last

🔴 PREORDER Rule

"I EAT first, then give to LEFT child, then RIGHT child"

1. Parent EATS first (prints)

2. LEFT child eats

3. RIGHT child eats

🟢 POSTORDER Rule

"Give to LEFT child, then RIGHT child, I EAT LAST"

1. LEFT child eats first

2. RIGHT child eats

3. Parent EATS last (prints)

🔴 PREORDER Traversal - Visual Comparison

Rule: NODE → LEFT → RIGHT

We print IMMEDIATELY when we arrive at a node, then explore children.

Preorder Path Through Tree:

       1  ← 🖨️ Print immediately (Step 1)
      / \
     2   3  ← After printing 1, go LEFT to 2
    / \   \    🖨️ Print 2 (Step 2)
   4   5   6
   ↑
   🖨️ Print 4 (Step 3)
   
   Then 5 (Step 4), then 3 (Step 5), then 6 (Step 6)
   
   
   Output: 1 2 4 5 3 6

Key Difference from Inorder:

✅ Inorder: Go left first, print when returning
✅ Preorder: Print FIRST, then go left

🟢 POSTORDER Traversal - Visual Comparison

Rule: LEFT → RIGHT → NODE

We print LAST, only after both children are completely done.

Postorder Path Through Tree:

       1  ← 🖨️ Print LAST (Step 11)
      / \
     2   3  ← Print 2 after its children (Step 6)
    / \   \    Print 3 after its children (Step 10)
   4   5   6
   ↑
   🖨️ Print 4 FIRST (Step 1) - it's a leaf
   
   Then 5 (Step 2), then 2, then 6, then 3, finally 1
   
   
   Output: 4 5 2 6 3 1

Key Pattern:

✅ Leaves print first (4, 5, 6)
✅ Then their parents (2, 3)
✅ Finally the root (1)
✅ Children before parent!

💻 Java Implementation

The Secret: All three functions are IDENTICAL except for WHERE the print statement goes! 

class Node {
    int data;
    Node left, right;
    
    Node(int data) {
        this.data = data;
        this.left = this.right = null;
    }
}

class TreeTraversals {
    
    // INORDER: Left → Node → Right
    void inorder(Node node) {
        if (node == null) return;  // BASE CASE
        
        inorder(node.left);              // 1. LEFT subtree
        System.out.print(node.data + " "); // 2. PRINT (between children)
        inorder(node.right);             // 3. RIGHT subtree
    }
    
    // PREORDER: Node → Left → Right
    void preorder(Node node) {
        if (node == null) return;
        
        System.out.print(node.data + " "); // 1. PRINT FIRST
        preorder(node.left);              // 2. LEFT subtree
        preorder(node.right);             // 3. RIGHT subtree
    }
    
    // POSTORDER: Left → Right → Node
    void postorder(Node node) {
        if (node == null) return;
        
        postorder(node.left);              // 1. LEFT subtree
        postorder(node.right);             // 2. RIGHT subtree
        System.out.print(node.data + " "); // 3. PRINT LAST
    }
    
    public static void main(String[] args) {
        TreeTraversals tree = new TreeTraversals();
        
        // Build tree:
        Node root = new Node(1);
        root.left = new Node(2);
        root.right = new Node(3);
        root.left.left = new Node(4);
        root.left.right = new Node(5);
        root.right.right = new Node(6);
        
        System.out.print("Inorder:   ");
        tree.inorder(root);
        // Output: 4 2 5 1 3 6
        
        System.out.print("\nPreorder:  ");
        tree.preorder(root);
        // Output: 1 2 4 5 3 6
        
        System.out.print("\nPostorder: ");
        tree.postorder(root);
        // Output: 4 5 2 6 3 1
    }
}

🎯 Code Pattern Recognition:

void traverse(Node node) {
    if (node == null) return;
    
    ← PREORDER prints here
    traverse(node.left);
    ← INORDER prints here
    traverse(node.right);
    ← POSTORDER prints here
}

⚡ Complexity Analysis

Traversal Time Complexity Space Complexity Reason
All Three O(N) O(H) Visit each node once; recursion stack depth = tree height

Where N = number of nodes, H = height of tree:

  • Balanced tree: H = log(N), so Space = O(log N)
  • Skewed tree: H = N, so Space = O(N)
  • Why recursion uses space: Each function call added to call stack

✨ Key Takeaways

Remember These Forever:

  1. Only print position changes! Everything else is identical
  2. Base case is critical: if (node == null) return;
  3. Chocolate rule: When does parent eat vs. children eat?
  4. Inorder of BST = sorted! Most important property!
  5. All three are O(N) time - visit every node once
  6. Space = O(H) - recursion stack depth = height
  7. Preorder: Root first (good for copying)
  8. Postorder: Root last (good for deleting)
  9. Inorder: Root in middle (gives sorted for BST)

🧠 Memory Trick:

IN-order: Print IN between children
PRE-order: Print PREviously (before children)
POST-order: Print POST (after children)
