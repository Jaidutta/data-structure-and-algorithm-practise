📏 Diameter of Binary Tree

Longest Path Between Any Two Nodes

Understanding the concept of diameter and how to calculate it

📋 Problem Statement

What We Need to Find:

Given a Binary Tree, find its diameter.

Definition of Diameter:

The diameter (also called width) of a binary tree is the length of the longest path between any two nodes in the tree.

Important: This path may or may not pass through the root!
Length: Number of edges (connections), not nodes.

💡 The Key Insight

🎯 Critical Observation:

For any node N, the longest path passing through N is:

Height of left subtree + Height of right subtree

Visual Explanation:

For node 2:
       1
      / \
     2   3  ← We're analyzing this node
      / \
     4   5

Left subtree of 2:     Right subtree of 2:
       4                      5

Height of left = 1     Height of right = 1


Longest path through node 2:
= height_left + height_right
= 1 + 1
= 2 edges (path: 4 → 2 → 5)

🎯 The Algorithm Strategy:

For EVERY node in the tree:
1. Calculate height of left subtree
2. Calculate height of right subtree
3. Potential diameter through this node = left_height + right_height
4. Track the maximum diameter seen so far

Final answer: The maximum diameter across all nodes!

✨ Why This Works:

The longest path in the entire tree MUST pass through some node.
By checking every node, we're guaranteed to find the longest path!

📏 Understanding Height

📐 Height Definition:

The height of a node is the number of edges in the longest path from that node down to a leaf.

  • Leaf node height: 0 (no edges below it)
  • Null node height: -1 (by convention, for easier calculation)
  • Internal node height: 1 + max(left height, right height)

Height Examples:

         1  ← height = 2
        / \
       2   3  ← height of 2 = 1, height of 3 = 0
      / \
     4   5  ← height of 4 = 0, height of 5 = 0 (leaves)

Calculation bottom-up:
• Node 4: Leaf → height = 0
• Node 5: Leaf → height = 0
• Node 3: Leaf → height = 0
• Node 2: max(height(4), height(5)) + 1 = max(0, 0) + 1 = 1
• Node 1: max(height(2), height(3)) + 1 = max(1, 0) + 1 = 2

💻 Java Implementation

class Solution {
    int diameter = 0;  // Track maximum diameter found
    
    public int diameterOfBinaryTree(Node root) {
        height(root);
        return diameter;
    }
    
    // Returns: height of the subtree (edges from node to deepest leaf)
    private int height(Node node) {
        if (node == null) return -1;  // -1 for null makes calculation easier
        
        // Get height of left and right subtrees
        int leftHeight = height(node.left);
        int rightHeight = height(node.right);
        
        // Diameter through THIS node = path going through left + right
        int currentDiameter = leftHeight + rightHeight + 2;
        // Why +2? Because we add 1 edge to left child and 1 edge to right child
        // If child is null, height is -1, so -1 + 1 = 0 (correct!)
        
        // Update global maximum
        diameter = Math.max(diameter, currentDiameter);
        
        // Return height of this subtree
        return Math.max(leftHeight, rightHeight) + 1;
    }
}

// Alternative: Using 0 for null height (more intuitive)
class Solution2 {
    int diameter = 0;
    
    public int diameterOfBinaryTree(Node root) {
        height(root);
        return diameter;
    }
    
    private int height(Node node) {
        if (node == null) return 0;  // 0 edges for null
        
        int leftHeight = height(node.left);
        int rightHeight = height(node.right);
        
        // Diameter through this node
        diameter = Math.max(diameter, leftHeight + rightHeight);
        
        // Return height: longest path down + 1 edge to this node
        return Math.max(leftHeight, rightHeight) + 1;
    }
}

// Example usage:
public static void main(String[] args) {
    Solution sol = new Solution();
    
    // Build tree:  1
    //             / \
    //            2   3
    //           / \
    //          4   5
    
    Node root = new Node(1);
    root.left = new Node(2);
    root.right = new Node(3);
    root.left.left = new Node(4);
    root.left.right = new Node(5);
    
    int result = sol.diameterOfBinaryTree(root);
    System.out.println("Diameter: " + result);  // Output: 3
}

⚡ Complexity Analysis

Metric Complexity Explanation
Time O(N) Visit each node exactly once
Space O(H) Recursion stack depth = height of tree

📊 Why O(N) Time?

We visit each node exactly once in a postorder traversal.
At each node, we do O(1) work (compare and update).
Total: O(N) × O(1) = O(N)

✨ Key Takeaways

Remember These Forever:

  1. Diameter = longest path between any two nodes (count edges, not nodes)
  2. Path may not pass through root - check every node!
  3. Diameter through a node: left_height + right_height
  4. Height = edges to deepest leaf (not including current node)
  5. Null node height = 0 (makes calculations cleaner)
  6. Use postorder traversal: Process children first, then current
  7. Track global maximum: Update diameter at each node
  8. Time: O(N) - visit every node once
  9. Space: O(H) - recursion stack

🧠 Memory Trick:

"The Longest Bridge"

Think of the tree as islands (nodes) connected by bridges (edges).
The diameter is the longest bridge route between any two islands.

At each island, ask: "If I build a bridge through here, how long would it be?"
Answer: Distance to furthest island on left + distance to furthest island on right!

🎯 Quick Formula:

For each node:

diameter = height(left) + height(right)

height = max(height(left), height(right)) + 1

Keep track of maximum diameter seen across all nodes!
