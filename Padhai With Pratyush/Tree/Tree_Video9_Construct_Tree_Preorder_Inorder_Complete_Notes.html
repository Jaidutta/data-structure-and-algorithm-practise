🏗️ Construct Tree from Traversals

Building Binary Tree from Preorder & Inorder

Understanding the recursive construction pattern

📋 Problem Statement

What We Need to Build:

Given two arrays representing the preorder and inorder traversals of a binary tree, reconstruct the original tree.

Example:

Preorder: [3, 9, 20, 15, 7]

Inorder: [9, 3, 15, 20, 7]

Reconstructed Tree:

       3
      / \
     9   20
        /  \
       15   7

🎯 Key Assumptions:

  • No duplicates: All values in the tree are unique
  • Valid input: Arrays represent valid preorder and inorder of same tree
  • Same length: Both arrays have the same number of elements

📚 Traversal Review

Quick Reminder:

Preorder (Node → Left → Right):

• Visit the root first, then left subtree, then right subtree
• First element is always the root!

Inorder (Left → Node → Right):

• Visit left subtree first, then root, then right subtree
• Root divides the array into left and right subtrees!

Example Tree with Traversals:

Tree:        3
            / \
           9   20
              /  \
             15   7

Preorder:  [3, 9, 20, 15, 7]
              ↑  ↑   ↑   ↑   ↑
            Root L  Root L   R  (of right subtree)

Inorder:   [9, 3, 15, 20, 7]
              ↑  ↑   ↑   ↑   ↑
              L Root L  Root R  (of right subtree)

💡 The Key Insight

🔑 Two Critical Properties:

Property 1: Preorder First Element

The first element of preorder is ALWAYS the root of the (sub)tree!

Property 2: Inorder Root Position

Once we find the root in inorder array, it divides the array:
• Everything to the left = left subtree
• Everything to the right = right subtree

✨ The Algorithm Strategy:

1. Take first element from preorder → This is the root
2. Find root in inorder array → This gives us the boundary
3. Split inorder into left and right parts
4. Split preorder into left and right parts (using same sizes)
5. Recursively build left and right subtrees
6. Return the root node with attached subtrees

💻 Java Implementation

import java.util.*;

class Solution {
    // HashMap for O(1) lookup of root index in inorder
    Map<Integer, Integer> inorderMap = new HashMap<>();
    int preorderIndex = 0;  // Tracks current position in preorder
    
    public Node buildTree(int[] preorder, int[] inorder) {
        // Build hashmap for O(1) index lookup
        for (int i = 0; i < inorder.length; i++) {
            inorderMap.put(inorder[i], i);
        }
        
        return buildTreeHelper(preorder, 0, inorder.length - 1);
    }
    
    private Node buildTreeHelper(int[] preorder, int inStart, int inEnd) {
        // BASE CASE: No elements to construct tree
        if (inStart > inEnd) return null;
        
        // STEP 1: Pick current element from preorder as root
        int rootValue = preorder[preorderIndex++];
        Node root = new Node(rootValue);
        
        // STEP 2: Find root position in inorder (O(1) with hashmap)
        int inorderIndex = inorderMap.get(rootValue);
        
        // STEP 3: Build left subtree (elements before root in inorder)
        root.left = buildTreeHelper(preorder, inStart, inorderIndex - 1);
        
        // STEP 4: Build right subtree (elements after root in inorder)
        root.right = buildTreeHelper(preorder, inorderIndex + 1, inEnd);
        
        return root;
    }
}

// Alternative: Passing array slices (less efficient but clearer)
class Solution2 {
    public Node buildTree(int[] preorder, int[] inorder) {
        if (preorder.length == 0) return null;
        
        // Root is first element of preorder
        int rootValue = preorder[0];
        Node root = new Node(rootValue);
        
        // Find root in inorder
        int rootIndex = 0;
        for (int i = 0; i < inorder.length; i++) {
            if (inorder[i] == rootValue) {
                rootIndex = i;
                break;
            }
        }
        
        // Split arrays
        int[] leftInorder = Arrays.copyOfRange(inorder, 0, rootIndex);
        int[] rightInorder = Arrays.copyOfRange(inorder, rootIndex + 1, inorder.length);
        
        int[] leftPreorder = Arrays.copyOfRange(preorder, 1, 1 + leftInorder.length);
        int[] rightPreorder = Arrays.copyOfRange(preorder, 1 + leftInorder.length, preorder.length);
        
        // Recursively build subtrees
        root.left = buildTree(leftPreorder, leftInorder);
        root.right = buildTree(rightPreorder, rightInorder);
        
        return root;
    }
}

// Example usage:
public static void main(String[] args) {
    Solution sol = new Solution();
    
    int[] preorder = {3, 9, 20, 15, 7};
    int[] inorder = {9, 3, 15, 20, 7};
    
    Node root = sol.buildTree(preorder, inorder);
    
    // Verify by printing inorder of constructed tree
    printInorder(root);  // Should output: 9 3 15 20 7
}

static void printInorder(Node node) {
    if (node == null) return;
    printInorder(node.left);
    System.out.print(node.val + " ");
    printInorder(node.right);
}

🎯 Code Explanation (Optimized Version):

  • Line 5: HashMap for O(1) lookup of index in inorder
  • Line 6: Global preorderIndex to track current position
  • Lines 10-12: Pre-populate hashmap with inorder indices
  • Line 18: Base case - invalid range means no tree
  • Line 21: Take next element from preorder, increment index
  • Line 25: Find root position in inorder (O(1) with hashmap)
  • Line 28: Build left subtree with elements before root in inorder
  • Line 31: Build right subtree with elements after root in inorder

💡 Why Use HashMap?

Without HashMap: We'd need to search for root in inorder each time → O(N) per call
With HashMap: Pre-store all indices → O(1) lookup per call

Impact on complexity:
• Without: O(N²) overall
• With: O(N) overall ✅

⚡ Complexity Analysis

Approach Time Complexity Space Complexity Notes
With HashMap O(N) O(N) Optimal solution - visit each node once
Without HashMap O(N²) O(N) Searching for root takes O(N) each time
Array Slicing O(N²) O(N²) Creating subarrays is expensive

📊 Time Complexity Breakdown (Optimal):

  • Building HashMap: O(N) - one pass through inorder
  • Recursive Calls: O(N) - each node processed exactly once
  • Finding Root: O(1) per call - hashmap lookup
  • Total: O(N) + O(N) = O(N)

📊 Space Complexity Breakdown:

  • HashMap: O(N) - stores all inorder indices
  • Recursion Stack: O(H) where H = height
  • Worst Case (Skewed): O(N)
  • Best Case (Balanced): O(log N) recursion + O(N) map = O(N)

⚠️ Important Edge Cases

Edge Case 1: Empty Tree

Preorder: []
Inorder: []

Result: null (empty tree)

Handled by: if (inStart > inEnd) return null;

Edge Case 2: Single Node

Preorder: [1]
Inorder: [1]

Result:  1  (single node, no children)

Handled by: Base case returns null for left and right

Edge Case 3: Left Skewed Tree

Tree:    3
        /
       2
      /
     1

Preorder: [3, 2, 1]
Inorder:  [1, 2, 3]

Works fine! Right subtree is always empty.

Edge Case 4: Right Skewed Tree

Tree:  1
        \
         2
          \
           3

Preorder: [1, 2, 3]
Inorder:  [1, 2, 3]

Works fine! Left subtree is always empty.

✨ Key Takeaways

Remember These Forever:

  1. Preorder first = root (always!)
  2. Inorder divides into left and right at root position
  3. Use HashMap for O(1) lookup of root index in inorder
  4. Track preorder position globally or pass indices
  5. Recursive pattern: Build root → Build left → Build right
  6. Base case: inStart > inEnd means empty subtree
  7. Time: O(N) with hashmap optimization
  8. Space: O(N) for hashmap + recursion
  9. No duplicates assumed (critical for hashmap to work)

🧠 Memory Trick:

"Pre-FIRST, In-DIVIDE"

• PREorder: First element is the root
• INorder: Root divides left and right

Together they give us everything we need to rebuild the tree!

🎯 Quick Algorithm Summary:

buildTree(preorder, inorder):
    1. Take first element from preorder → ROOT
    2. Find ROOT in inorder → INDEX
    3. Everything left of INDEX in inorder → LEFT subtree
    4. Everything right of INDEX in inorder → RIGHT subtree
    5. Recursively build left and right subtrees
    6. Attach subtrees to root and return
