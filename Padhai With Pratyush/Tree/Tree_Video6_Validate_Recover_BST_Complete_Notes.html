✅ Validate & Recover BST

Using Inorder Traversal Property

Two problems, one solution: The power of inorder traversal

📋 Two Related Problems

Problem 1: Validate BST

Given a Binary Tree, determine if it is a valid Binary Search Tree (BST).

Example:

Input: [2, 1, 3]

Output: true (valid BST)

Problem 2: Recover BST

A BST has exactly two nodes swapped by mistake. Recover the tree by swapping them back.

Example:

Input: [3, 1, 4, null, null, 2] (1 and 2 swapped)

Output: [2, 1, 4, null, null, 3] (corrected BST)

💡 The Key Insight: Inorder Property

🔑 The Golden Rule of BST:

Inorder traversal of a BST MUST produce a SORTED sequence!

This is the MOST IMPORTANT property of BST. If you remember nothing else, remember this!

Valid BST Example:

       4
      / \
     2   6
    / \ / \
   1  3 5  7

Inorder Traversal: 1 → 2 → 3 → 4 → 5 → 6 → 7

Inorder Sequence:

1
2
3
4
5
6
7

✅ SORTED → Valid BST!

Invalid BST Example:

       4
      / \
     2   6
    / \ / \
   1  5 3  7  ← 5 and 3 are in wrong positions!

Inorder Traversal: 1 → 2 → 5 → 4 → 3 → 6 → 7

Inorder Sequence:

1
2
5
4
3
6
7

❌ NOT SORTED → Invalid BST!

Violations: 5 > 4 and 3 < 4

🎯 Why This Works:

Inorder traversal visits nodes in this order: Left → Node → Right

For a BST:
• All LEFT nodes are smaller
• Current node is in the middle
• All RIGHT nodes are larger

So naturally, inorder gives us: small → medium → large (sorted!)

✅ Problem 1: Validate BST

🎯 Strategy:

Do an inorder traversal and check if each node is greater than the previous node.

💡 The "Previous Pointer" Trick:

Instead of creating an array to store all values, we use a prev pointer:

• Keep track of the last visited node
• Compare current node with prev
• If current ≤ prev → Invalid BST!
• If current > prev → Continue checking

Example Execution
Tree:    4
        / \
       2   6
      / \ / \
     1  3 5  7

Inorder walk:
Step 1: Visit 1, prev = null → OK, set prev = 1
Step 2: Visit 2, prev = 1 → 2 > 1? YES → OK, set prev = 2
Step 3: Visit 3, prev = 2 → 3 > 2? YES → OK, set prev = 3
Step 4: Visit 4, prev = 3 → 4 > 3? YES → OK, set prev = 4
Step 5: Visit 5, prev = 4 → 5 > 4? YES → OK, set prev = 5
Step 6: Visit 6, prev = 5 → 6 > 5? YES → OK, set prev = 6
Step 7: Visit 7, prev = 6 → 7 > 6? YES → OK, set prev = 7

Result: All comparisons passed → Valid BST! ✅

Java Implementation:

class Solution {
    Node prev = null;  // Track the previous node in inorder
    
    public boolean isValidBST(Node root) {
        if (root == null) return true;
        
        // Check LEFT subtree first (inorder: Left → Node → Right)
        if (!isValidBST(root.left)) return false;
        
        // Check CURRENT node: Must be greater than previous
        if (prev != null && root.data <= prev.data) {
            return false;  // Violation found!
        }
        
        // Update prev to current node
        prev = root;
        
        // Check RIGHT subtree
        return isValidBST(root.right);
    }
}

// Alternative: Using an Integer wrapper to track prev value
class Solution2 {
    Integer prev = null;  // Use Integer to allow null
    
    public boolean isValidBST(Node root) {
        if (root == null) return true;
        
        if (!isValidBST(root.left)) return false;
        
        if (prev != null && root.data <= prev) {
            return false;
        }
        prev = root.data;  // Store value, not reference
        
        return isValidBST(root.right);
    }
}

🎯 Code Explanation:

  • Line 2: Global prev pointer to track last visited node
  • Line 7: Check left subtree first (inorder starts left)
  • Line 10-12: Current must be > prev, else invalid
  • Line 15: Update prev before moving right
  • Line 18: Check right subtree
  • Key: This is essentially inorder with a comparison at each step

🔧 Problem 2: Recover BST

🎯 The Problem:

Two nodes in a BST were swapped by mistake. We need to find them and swap them back.

💡 Strategy:

Do an inorder traversal and find where the sorted order is broken.

There will be exactly 2 violations:
• First violation (G1): The larger wrongly placed node
• Second violation (G2): The smaller wrongly placed node

Swap G1 and G2 to fix the tree!

Example: Nodes 3 and 5 are Swapped

Original (Correct) BST:
       4
      / \
     2   6
    / \ / \
   1  3 5  7

Inorder: 1 2 3 4 5 6 7  ✅ Sorted

After Swapping 3 and 5 (Broken BST):
       4
      / \
     2   6
    / \ / \
   1  5 3  7

Inorder: 1 2 5 4 3 6 7  ❌ Not sorted!

Finding violations:
Comparison: 2 vs 5 → OK (2 < 5)
Comparison: 5 vs 4 → ❌ VIOLATION 1! (5 > 4) → G1 = 5
Comparison: 4 vs 3 → ❌ VIOLATION 2! (4 > 3) → G2 = 3
Comparison: 3 vs 6 → OK (3 < 6)
Comparison: 6 vs 7 → OK (6 < 7)

Solution: Swap G1 (5) and G2 (3) back!

⚠️ Two Possible Cases:

Case 1: Adjacent Nodes Swapped

Example: [1, 3, 2, 4, 5] (2 and 3 swapped)

Only ONE violation in sequence: 3 > 2

In this case: G1 = first element, G2 = second element

Case 2: Non-Adjacent Nodes Swapped

Example: [1, 2, 5, 4, 3, 6, 7] (3 and 5 swapped)

TWO violations: 5 > 4 and 4 > 3

In this case: G1 = first violation's first element (5), G2 = second violation's second element (3)

Java Implementation:

class Solution {
    Node first = null;   // G1 - first wrongly placed node
    Node second = null;  // G2 - second wrongly placed node
    Node prev = null;    // Track previous node in inorder
    
    public void recoverTree(Node root) {
        // Find the two swapped nodes using inorder
        findSwapped(root);
        
        // Swap their values back
        if (first != null && second != null) {
            int temp = first.data;
            first.data = second.data;
            second.data = temp;
        }
    }
    
    private void findSwapped(Node root) {
        if (root == null) return;
        
        // Inorder: Left → Node → Right
        findSwapped(root.left);
        
        // Check if current node violates BST property
        if (prev != null && root.data < prev.data) {
            // Violation found!
            if (first == null) {
                // First violation: Mark both nodes
                first = prev;   // The larger one (wrongly on left)
                second = root;  // The smaller one (wrongly on right)
            } else {
                // Second violation: Update second only
                second = root;  // The actual smaller swapped node
            }
        }
        
        prev = root;  // Update prev for next iteration
        
        findSwapped(root.right);
    }
}

// Example usage:
public static void main(String[] args) {
    Solution sol = new Solution();
    
    // Broken BST: 1 2 5 4 3 6 7 (3 and 5 swapped)
    Node root = new Node(4);
    root.left = new Node(2);
    root.right = new Node(6);
    root.left.left = new Node(1);
    root.left.right = new Node(5);   // Should be 3
    root.right.left = new Node(3);  // Should be 5
    root.right.right = new Node(7);
    
    sol.recoverTree(root);
    // Tree is now fixed!
}

🎯 Code Explanation:

  • Lines 2-4: Track the two swapped nodes and previous node
  • Lines 11-15: After finding them, swap their values
  • Line 24: Detect violation: current < prev (should be current > prev)
  • Lines 26-29: First violation: tentatively mark both nodes
  • Lines 30-33: Second violation: update second node (the real culprit)
  • Why update second twice? For adjacent swaps, second is set correctly in first violation. For non-adjacent, second gets updated in second violation.

🔍 Step-by-Step Recovery Example

Broken Tree: Nodes 15 and 5 Swapped

Correct BST should have been:
       10
      /  \
     5    15
    / \   / \
   3  7  13 17

But 5 and 15 are swapped:
       10
      /  \
    15    5
    / \   / \
   3  7  13 17

Inorder of broken tree: 3, 15, 7, 10, 5, 13, 17
Trace Execution
Inorder walk with comparisons:

Visit 3:  prev = null → No comparison, set prev = 3
Visit 15: prev = 3 → 15 > 3? YES ✅ OK, set prev = 15
Visit 7:  prev = 15 → 7 > 15? NO ❌ VIOLATION 1!
          first = 15 (prev), second = 7 (current)
          Set prev = 7
Visit 10: prev = 7 → 10 > 7? YES ✅ OK, set prev = 10
Visit 5:  prev = 10 → 5 > 10? NO ❌ VIOLATION 2!
          first stays 15, second = 5 (updated!)
          Set prev = 5
Visit 13: prev = 5 → 13 > 5? YES ✅ OK, set prev = 13
Visit 17: prev = 13 → 17 > 13? YES ✅ OK, set prev = 17

Result: first = 15, second = 5
Action: Swap 15 and 5 → Tree recovered! ✅

⚡ Complexity Analysis

Problem Time Complexity Space Complexity Explanation
Validate BST O(N) O(H) Visit each node once; recursion stack depth = height
Recover BST O(N) O(H) One inorder traversal; recursion stack depth = height

📊 Why O(N) Time?

We visit each node exactly once during inorder traversal.
Even though we might find violations early, worst case still requires visiting all N nodes.

📊 Space Complexity Breakdown:

  • Best case (balanced tree): H = log(N) → O(log N) space
  • Worst case (skewed tree): H = N → O(N) space
  • Space used by: Recursion call stack (implicit)
  • Optimization: Morris Traversal can achieve O(1) space (advanced)

⚠️ Common Mistakes to Avoid

❌ Mistake 1: Checking Only Immediate Children

// WRONG APPROACH:
boolean isValidBST(Node root) {
    if (root.left != null && root.left.data >= root.data) return false;
    if (root.right != null && root.right.data <= root.data) return false;
    return isValidBST(root.left) && isValidBST(root.right);
}

Problem: This only checks immediate parent-child relationship!

Counter-example:
       5
      / \
     1   7
        / \
       3   9

This passes the test but 3 < 5, so it's INVALID!
The left child of 7 (which is 3) is not greater than root (5).

✅ Correct: Use inorder traversal with prev pointer!

❌ Mistake 2: Forgetting to Update Prev

// WRONG:
if (prev != null && root.data <= prev.data) return false;
// Missing: prev = root;
return isValidBST(root.right);

Without updating prev, comparisons are meaningless!

❌ Mistake 3: Using >= Instead of >

// WRONG:
if (prev != null && root.data >= prev.data) return false;

This rejects valid BSTs with duplicate values!

If duplicates are allowed (depends on problem),
you might want to allow equality.

✨ Key Takeaways

Remember These Forever:

  1. Golden Rule: Inorder of BST MUST be sorted!
  2. Validate BST: Use prev pointer during inorder, check current > prev
  3. Recover BST: Find 2 violations in inorder sequence
  4. First violation: G1 = prev (larger element)
  5. Second violation: G2 = current (smaller element)
  6. Adjacent swap: Only one violation, G2 from first violation
  7. Non-adjacent swap: Two violations, G2 from second violation
  8. Time: O(N) - visit every node once
  9. Space: O(H) - recursion stack

🧠 Memory Trick:

"Inorder = In Order"

If you do an IN-ORDER traversal of a BST,
the values come out IN ORDER (sorted)!

If they're NOT in order → Invalid or broken BST!

🎯 Quick Decision Guide:

Need to validate BST?

→ Do inorder, check if strictly increasing

Need to recover BST?

→ Do inorder, find where it breaks, swap the culprits

Key insight for both?

→ Inorder traversal of BST is sorted!

🎓 Interview Tips:

  • Always mention the inorder = sorted property first
  • Explain why checking only immediate children doesn't work
  • Walk through a small example to show your understanding
  • Mention the O(1) space Morris Traversal optimization if asked
  • Be clear about handling duplicates (problem-dependent)
