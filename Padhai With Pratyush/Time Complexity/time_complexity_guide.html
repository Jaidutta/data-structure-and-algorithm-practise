🎯 The Magic Rule

10⁸
Operations Per Second

Online judges (LeetCode, Codeforces, GeeksforGeeks) allow approximately 10⁸ simple operations.

If your algorithm exceeds this limit → Time Limit Exceeded (TLE)

What Counts as a "Simple Operation"?

// Each line is ONE simple operation (O(1))
int a = 10;           // Assignment
int b = 20;           // Assignment
int sum = a + b;      // Addition
print(sum);           // Print
return sum;           // Return

// All together = 5 operations = O(1) constant time
// NOT dependent on input size!

💡 Language Differences

  • C++: ~10⁸ operations in 1 second
  • Python: ~10⁸ operations in 2 seconds (judges give more time)
  • JavaScript: ~10⁸ operations in 3 seconds

The number of operations stays the same, but judges adjust time limits based on language speed.

🔍 What About N-1?

If the loop runs i < n - 1, technically it's N-1 operations.

But in Big O notation, we only care about the highest power:
N-1 = N¹ → Still O(N)
N+50 = N¹ → Still O(N)
100N = N¹ → Still O(N)

Example 2: Nested Loops - O(n²)

for (int i = 0; i < n; i++) {       // Outer loop: N times
    for (int j = 0; j < n; j++) {   // Inner loop: N times
        print("Hello");         // O(1)
    }
}

Step-by-Step Calculation

1 Outer loop runs:

N times (i goes from 0 to n-1)

2 For EACH i, inner loop runs:

N times (j goes from 0 to n-1)

3 Total operations:
N × (N × 1) = N²

When i=0: Inner loop runs N times
When i=1: Inner loop runs N times
...
When i=N-1: Inner loop runs N times
Total = N + N + N + ... (N times) = N²

4 Final Answer:
O(N²)

Example 3: Complex Program - The MAX Rule

When a program has multiple INDEPENDENT sections, take the MAXIMUM complexity! 

void solve(vector<int>& arr, int n) {
    // SECTION 1: Simple print
    print("Hello");  // O(1)
    
    // SECTION 2: Linear loop
    for (int i = 0; i < n; i++) {
        print("Hello");  // O(n)
    }
    
    // SECTION 3: Sorting
    sort(arr.begin(), arr.end());  // O(n log n)
}

Step-by-Step Analysis

1 Section 1 Complexity:

Single print statement → O(1)

2 Section 2 Complexity:

Loop runs N times → O(N)

3 Section 3 Complexity:

Standard library sort() → O(N log N)

(Why? Merge sort / Quick sort uses recursion and divide-and-conquer)

4 Compare all sections:
Section 1
O(1)
Section 2
O(N)
Section 3
O(N log N)

← MAXIMUM!

5 Final Answer:
O(N log N)

We take the LARGEST complexity because it dominates the total runtime

🧮 Why the MAX Rule?

Think of it in terms of powers of N:

• O(1) = N⁰
• O(N) = N¹
• O(N log N) = N¹ × log N
• O(N²) = N²

The highest power dominates!
When N=10,000: N² = 100,000,000 vs N = 10,000
The N² term makes the others negligible.

🔗 Dependent vs Independent Code

The Golden Rule

Independent code: Runs a FIXED number of times regardless of input N → O(1)

Dependent code: Number of operations GROWS with input N → O(N), O(N²), etc.

Examples

// INDEPENDENT (O(1)) - always runs 100 times
for (int i = 0; i < 100; i++) {
    print("Hello");
}

// DEPENDENT (O(N)) - runs N times
for (int i = 0; i < n; i++) {
    print("Hello");
}

// INDEPENDENT (O(1)) - just 4 simple operations
int a = 10;
int b = 20;
int c = a + b;
print(c);

🔥 Advanced Loop Patterns

Not all loops increment by +1!
Different increment patterns lead to different time complexities.

Pattern 1: Loop with i += 2 (Skip by 2)

// Single loop incrementing by 2
for (int i = 0; i < n; i += 2) {  // i = i + 2
    print("Hello");
}
1 How many iterations?

i goes: 0, 2, 4, 6, 8, ..., N-2
Number of iterations = N/2

2 Apply Big O:
N/2 = N × (1/2) = Still O(N)

Constants are dropped! N/2, N/3, N/100 → All O(N)

Pattern 1b: Nested Loop with i += 2

// Outer loop: i += 1, Inner loop: j += 2
for (int i = 0; i < n; i++) {         // N iterations
    for (int j = 0; j < n; j += 2) {   // N/2 iterations
        print("Hello");
    }
}
N × (N/2) = N²/2 = O(N²)

Constants dropped! The N² term dominates.

Pattern 2: Loop with i *= 2 (Doubling)

// Loop where i doubles each time
for (int i = 1; i < n; i *= 2) {  // i = i * 2
    print("Hello");
}
1 Trace the values:

i goes: 1, 2, 4, 8, 16, 32, ..., until i ≥ N
Each iteration: i = 2⁰, 2¹, 2², 2³, ..., 2^k where 2^k ≈ N

2 Solve for k:

2^k = N
Take log₂ of both sides:
k = log₂(N)

Number of iterations = log₂(N)

3 Final Answer:
O(log N)

💡 Key Insight: Exponential Growth → Logarithmic Complexity

When your loop variable multiplies (i *= 2, i *= 3, i *= 10), you get O(log N)

Why? You're covering the space exponentially faster.
Instead of N steps, you only need log(N) steps to reach N.

Pattern 2b: Nested Loop with Doubling

// Outer: regular, Inner: doubles
for (int i = 0; i < n; i++) {         // N iterations
    for (int j = 1; j < n; j *= 2) {   // log N iterations
        print("Hello");
    }
}
N × log(N) = O(N log N)

Common in efficient sorting algorithms!

Pattern 3: Loop with i /= 2 (Halving)

// Loop where i halves each time (starts from N)
for (int i = n; i >= 1; i /= 2) {  // i = i / 2
    print("Hello");
}
1 Trace the values:

i goes: N, N/2, N/4, N/8, ..., 2, 1
Each iteration: i = N/2⁰, N/2¹, N/2², N/2³, ..., N/2^k where N/2^k = 1

2 Solve for k:

N / 2^k = 1
2^k = N
k = log₂(N)

Number of iterations = log₂(N)

3 Final Answer:
O(log N)

Same as Pattern 2! Halving is the reverse of doubling.

⚠️ Classic Example: Binary Search

// Binary Search uses halving
int low = 0, high = n - 1;
while (low <= high) {
    int mid = (low + high) / 2;
    if (arr[mid] == target) return mid;
    
    if (arr[mid] < target)
        low = mid + 1;      // Cut left half
    else
        high = mid - 1;     // Cut right half
}

Each iteration cuts the search space in HALF → O(log N)
This is why binary search is so fast!

🔥 Complex Nested Patterns

Case 1: Both loops double

for (int i = 1; i < n; i *= 2) {         // log N iterations
    for (int j = 1; j < n; j *= 2) {   // log N iterations
        print("Hello");
    }
}
log(N) × log(N) = O(log² N)

Case 2: Outer normal, Inner depends on i

// Inner loop runs i times (not N times!)
for (int i = 0; i < n; i++) {
    for (int j = 0; j < i; j++) {   // Runs i times
        print("Hello");
    }
}
1 Count total operations:

When i=0: Inner runs 0 times
When i=1: Inner runs 1 time
When i=2: Inner runs 2 times
...
When i=N-1: Inner runs N-1 times

Total = 0 + 1 + 2 + 3 + ... + (N-1)
= N(N-1)/2 = (N² - N)/2

2 Apply Big O:
(N² - N)/2 → Keep highest power → O(N²)

Case 3: Inner loop starts from i

// Inner loop: j starts from i, not 0!
for (int i = 0; i < n; i++) {
    for (int j = i; j < n; j++) {   // Runs (N-i) times
        print("Hello");
    }
}
1 Count operations:

When i=0: Inner runs N times (j: 0→N)
When i=1: Inner runs N-1 times (j: 1→N)
When i=2: Inner runs N-2 times (j: 2→N)
...
When i=N-1: Inner runs 1 time

Total = N + (N-1) + (N-2) + ... + 1
= N(N+1)/2 ≈ N²/2

2 Final Answer:
O(N²)

Case 4: Inner depends on i with doubling

// Outer: regular, Inner: doubles but limited by i
for (int i = 1; i < n; i++) {
    for (int j = 1; j < i; j *= 2) {   // Runs log(i) times
        print("Hello");
    }
}
1 Analyze carefully:

When i=1: Inner runs log(1) = 0 times
When i=2: Inner runs log(2) = 1 time
When i=4: Inner runs log(4) = 2 times
When i=8: Inner runs log(8) = 3 times
...
When i=N: Inner runs log(N) times

Total ≈ log(1) + log(2) + log(3) + ... + log(N)
= log(N!) ≈ N log(N)

2 Final Answer:
O(N log N)

🎯 How to Evaluate ANY Loop Pattern

  1. Write out the sequence: What values does i take? (0,1,2,3... or 1,2,4,8...)
  2. Find the pattern: Is it arithmetic (+k) or geometric (×k)?
  3. Count iterations:
    • Arithmetic (i+=k) → N/k iterations → O(N)
    • Geometric (i*=k or i/=k) → log(N) iterations → O(log N)
  4. For nested loops: Multiply the iteration counts
  5. Simplify using Big O rules: Drop constants, keep highest power

🎯 Pro Tips

2 Compare complexities with real numbers

N² is much bigger!
