1. What are Merge Intervals? 🤔

Simple Definition:

Merge Intervals is a pattern for working with time ranges or number ranges represented as pairs [start, end]. The core technique involves sorting by start time and then processing overlaps systematically.

What is an Interval?

📊 UNDERSTANDING INTERVALS

An interval is a pair: [start, end]

Visual Representation:

Interval [2, 5]:
  0   1   2   3   4   5   6   7
  ────●━━━━━━━━━━━●────────────
      ↑           ↑
     Start       End
     
Key Properties:
• start ≤ end (always)
• Represents a continuous range
• Can be time (hours), prices ($), positions, etc.

Examples:
• Meeting time: [9:00 AM, 10:30 AM]
• Price range: [$10, $15]
• Train platform: [arrival, departure]

The Core Problem:

Given multiple intervals, we need to:
✓ Merge overlapping intervals
✓ Insert new intervals while maintaining order
✓ Find intersections between two sets
✓ Count maximum concurrent intervals

Recognition Keywords:

3. Why Sorting is CRITICAL ⚠️

🚨 Rule #1: ALWAYS SORT BY START TIME FIRST!

Without sorting, you cannot efficiently detect overlaps or process intervals in order.

The Overlap Check Formula:

After sorting by start time:

end₁ ≥ start₂ → OVERLAP

If the first interval's end time is ≥ the second interval's start time,
they overlap!

5. Problem 1: Merge Intervals 👕

Problem Statement - LeetCode 56

Given an array of intervals where intervals[i] = [start_i, end_i], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input. 

Example 1:
Input: intervals = [[1,3],[2,6],[8,10],[15,18]]
Output: [[1,6],[8,10],[15,18]]
Explanation: [1,3] and [2,6] overlap → merge to [1,6]

Example 2:
Input: intervals = [[1,4],[4,5]]
Output: [[1,5]]
Explanation: [1,4] and [4,5] touch at 4 → merge to [1,5]

The Merge Logic - Detailed:

🎯 UNDERSTANDING THE OVERLAP CHECK

Given two sorted intervals: [s1, e1] and [s2, e2]
(Where s1 ≤ s2 because we sorted by start)

Scenario A: OVERLAP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [s1 -------- e1]
       [s2 ----------- e2]
       
  Condition: e1 >= s2
  Action: Merge!
  Result: [s1, max(e1, e2)]

Scenario B: NO OVERLAP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [s1 ---- e1]
                    [s2 ---- e2]
                    
  Condition: e1 < s2
  Action: Save current, start new
  Result: Two separate intervals

Edge Case: TOUCHING (Considered Overlap)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [s1 ---- e1]
              [s2 ---- e2]
              
  If e1 == s2, they touch → Merge!
  Example: [1,4] and [4,5] → [1,5]

Complete Java Implementation:

import java.util.*;

public class MergeIntervals {
    public int[][] merge(int[][] intervals) {
        // Handle empty input
        if (intervals.length == 0) return new int[0][0];
        
        // STEP 1: Sort by start time (CRITICAL!)
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
        
        // Result list to store merged intervals
        List<int[]> result = new ArrayList<>();
        
        // STEP 2: Initialize with first interval
        int currentStart = intervals[0][0];
        int currentEnd = intervals[0][1];
        
        // STEP 3: Process each interval
        for (int i = 1; i < intervals.length; i++) {
            int nextStart = intervals[i][0];
            int nextEnd = intervals[i][1];
            
            // Check for overlap
            if (currentEnd >= nextStart) {
                // OVERLAP: Merge by extending currentEnd
                // Start stays the same (already sorted)
                // End becomes the maximum of both ends
                currentEnd = Math.max(currentEnd, nextEnd);
            } else {
                // NO OVERLAP: Save current and start new
                result.add(new int[]{currentStart, currentEnd});
                currentStart = nextStart;
                currentEnd = nextEnd;
            }
        }
        
        // STEP 4: Don't forget to add the last interval!
        result.add(new int[]{currentStart, currentEnd});
        
        // Convert list to array
        return result.toArray(new int[result.size()][]);
    }
}

// Time Complexity: O(n log n) - dominated by sorting
// Space Complexity: O(n) - for the result list

💡 Key Takeaways:

  • ✓ Always sort first by start time
  • ✓ Overlap check: currentEnd >= nextStart
  • ✓ When merging: extend end to max(end1, end2)
  • ✓ When no overlap: save current, start new
  • ✓ Don't forget to add the final interval!

⚠️ Edge Cases, Tricky Scenarios & Common Mistakes

Edge Case 1: Touching Intervals (CRITICAL!)

🎯 THE TOUCHING QUESTION

Input: [[1,4], [4,5]]

Question: Do [1,4] and [4,5] overlap?

Visual:
  0   1   2   3   4   5   6
  ────●━━━━━━━━━━━●──────────  [1,4]
  ────────────────●━━●────────  [4,5]
                  ↑
              Touch at 4!

The Math:
Check: end₁ (4) >= start₂ (4)?
       4 >= 4? 
       YES ✅

Decision: THEY OVERLAP (by definition)!
Result: Merge to [1,5]

⚠️ CRITICAL MISTAKE TO AVOID:
❌ WRONG: if (end₁ > start₂)  // Misses touching!
✅ RIGHT: if (end₁ >= start₂) // Catches touching!

Why ">=" matters:
In interval problems, touching = overlapping.
If you use >, you'll miss intervals that share a boundary!

Edge Case 2: Single Interval

Input: [[5,10]]

Process:
- Initialize with [5,10]
- No more intervals to process
- Add [5,10] to result

Output: [[5,10]]

Edge case handling in code:
if (intervals.length == 0) return new int[0][0];
// Always check for empty input!

Edge Case 3: Identical Intervals

Input: [[1,3], [1,3], [1,3]]

Visual:
All three are EXACTLY the same:
  ●━━━━━━●
  ●━━━━━━●
  ●━━━━━━●

Process:
[1,3] + [1,3]: 3 >= 1? YES → [1, max(3,3)] = [1,3]
[1,3] + [1,3]: 3 >= 1? YES → [1, max(3,3)] = [1,3]

Output: [[1,3]]

All duplicates merge into one!

Edge Case 4: Reverse Sorted (Unsorted Input)

Input: [[8,10], [2,6], [1,3]]  ← NOT sorted!

Step 1: MUST SORT FIRST!
After sort: [[1,3], [2,6], [8,10]]

Why sorting matters:
Without sort:
  [8,10] → [2,6]: Check 10 >= 2? YES
  But [2,6] comes BEFORE [8,10] on timeline!
  ❌ Logic breaks!

With sort:
  [1,3] → [2,6]: Check 3 >= 2? YES ✅ Correct!
  [1,6] → [8,10]: Check 6 >= 8? NO ✅ Correct!

⚠️ ALWAYS SORT FIRST!
Arrays.sort(intervals, (a,b) -> Integer.compare(a[0], b[0]));

Complete Java Code:

public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0, n = intervals.length;
    
    // Phase 1: Add all intervals BEFORE new interval (no overlap)
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i]);
        i++;
    }
    
    // Phase 2: Merge all overlapping intervals
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);
    
    // Phase 3: Add remaining intervals AFTER merged blob
    while (i < n) {
        result.add(intervals[i]);
        i++;
    }
    
    return result.toArray(new int[result.size()][]);
}

7. Problem 3: Interval Intersections 🤝

Problem Statement - LeetCode 986

Given two lists of intervals, return the intersection (common time slots).

🤝 Finding Meeting Time

Person A is free: [1-5], [8-10]
Person B is free: [2-6]
When can they BOTH meet? [2-5] ← The intersection!

Visual Intersection Logic:

🔍 FINDING THE COMMON GROUND

Person A: [1 ------------- 5]
Person B:      [2 -------------- 6]

Intersection Formula:
Start: max(1, 2) = 2  (Latest start)
End:   min(5, 6) = 5  (Earliest end)

Result: [2, 5] ✅

Timeline:
  0   1   2   3   4   5   6   7
  ────●━━━━━━━━━━━━━━━●───────  Person A [1,5]
  ────────●━━━━━━━━━━━━━━━●───  Person B [2,6]
  ────────●━━━━━━━━━●─────────  Intersection [2,5]
          ↑           ↑
     Latest Start  Earliest End

Who Moves Next?
The person who finishes FIRST (earliest end) moves to their next slot.
Why? They can't possibly overlap with future slots of the other person!

Complete Java Code:

public int[][] intervalIntersection(int[][] A, int[][] B) {
    List<int[]> result = new ArrayList<>();
    int i = 0, j = 0;
    
    while (i < A.length && j < B.length) {
        // Find intersection (common ground)
        int start = Math.max(A[i][0], B[j][0]);  // Latest start
        int end = Math.min(A[i][1], B[j][1]);    // Earliest end
        
        // If valid interval (start <= end), add it
        if (start <= end) {
            result.add(new int[]{start, end});
        }
        
        // Move pointer of interval that ends first
        if (A[i][1] < B[j][1]) {
            i++;
        } else {
            j++;
        }
    }
    
    return result.toArray(new int[result.size()][]);
}

8. Problem 4: Minimum Meeting Rooms 🏨

Problem Statement - LeetCode 253

Given meeting time intervals, find the minimum number of conference rooms required.

🏨 The Hotel Manager

You're a hotel manager. You DON'T merge guests - you need to know the maximum number of rooms needed at the busiest time!

Guest arrives (start) → Build room (+1)
Guest leaves (end) → Free room (-1)

Ends:   [10, 20, 30] ← All departure times
Both sorted independently!

  ● Rooms needed = 2  (Meeting 1 still ongoing!)

Complete Java Code:

public int minMeetingRooms(int[][] intervals) {
    if (intervals.length == 0) return 0;
    
    int n = intervals.length;
    int[] starts = new int[n];
    int[] ends = new int[n];
    
    // Separate starts and ends
    for (int i = 0; i < n; i++) {
        starts[i] = intervals[i][0];
        ends[i] = intervals[i][1];
    }
    
    // Sort both arrays independently
    Arrays.sort(starts);
    Arrays.sort(ends);
    
    int rooms = 0, maxRooms = 0;
    int sPtr = 0, ePtr = 0;
    
    // Process all start times
    while (sPtr < n) {
        // If next event is an arrival (start < end)
        if (starts[sPtr] < ends[ePtr]) {
            rooms++;        // Need a room
            sPtr++;
        } else {
            rooms--;        // Free a room
            ePtr++;
        }
        maxRooms = Math.max(maxRooms, rooms);
    }
    
    return maxRooms;
}

  ────●━━━●───────────────────  [1,3]
  ────────●━━━━━━━━━━━●───────  [2,6]

Complete Java Code:

public boolean canAttendMeetings(int[][] intervals) {
    // Sort by start time
    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
    
    // Check consecutive intervals for overlap
    for (int i = 1; i < intervals.length; i++) {
        // If previous end >= current start → OVERLAP!
        if (intervals[i-1][1] >= intervals[i][0]) {
            return false;  // Cannot attend all
        }
    }
    
    return true;  // Can attend all meetings
}
