1пёҸвғЈ Introduction - Why Linked Lists?

The Core Problem

Arrays require CONTINUOUS memory. If your RAM is fragmented, you might not find a continuous block even if you have enough total space!

рҹ“Ҡ Arrays vs Linked Lists

рҹҡӮ Array - Continuous Train

[1][2][3][4][5]

All bogies connected
MUST be continuous!

Problem: Need 10 continuous blocks
RAM: [x][x][ ][ ][x][x][ ][ ][ ][ ]
     вқҢ Can't fit! (fragmented)

рҹ”— Linked List - Scattered Train

Memory:
[1@200] ... [2@500] ... [3@100]
   вҶ“           вҶ“           вҶ“
Address 200  Address 500  Address 100

Bogies scattered anywhere!
Connected by addresses (pointers)

вң“ Can use scattered blocks!

вң… Solution: Linked Lists

Use scattered memory boxes located anywhere in the system.
Keep them connected by storing the address (or "House Number") of the next box in each node!

рҹҸ  The House Number Analogy

Think of a scavenger hunt:

рҹҸ  House #100: "Find the treasure at House #200"
рҹҸ  House #200: "Find the treasure at House #300"
рҹҸ  House #300: "No more houses!" (null)

Each house tells you where the next house is!

4пёҸвғЈ Java Implementation

рҹ’» Node Class Definition

/**
 * Node Class - The Building Block
 * 
 * Represents a single "box" in the linked list
 * Contains: data value + reference to next node
 */
class Node {
    int data;       // The value stored
    Node next;      // Address (reference) of next node

    // Constructor to create a new node
    Node(int data) {
        this.data = data;
        this.next = null;  // By default, points to null
    }
}

Key Differences from C++:

вҖў Java uses dot notation (.): head.data
вҖў C++ uses arrow: head->data

5пёҸвғЈ Traversal - The Postman Analogy

рҹҡ¶ The Postman Method

Act like a postman delivering letters:
вҖў Visit house, read address, go to next house
вҖў Repeat until you reach null (no more houses)

вҡ пёҸ GOLDEN RULE:

NEVER move the HEAD variable directly!
If you lose HEAD, you lose the entire list!

рҹ“Ҡ Traversal Step-by-Step

Step 0: Initialize
HEAD = 100        temp = 100 (copy HEAD, don't move it!)
  в”Ӯ                 в”Ӯ
  в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

Step 1: Visit first node
temp вҶ’ [1|200]    Print: 1    Move: temp = 200

Step 2: Visit second node  
temp вҶ’ [2|300]    Print: 2    Move: temp = 300

Step 3: Visit third node
temp вҶ’ [3|null]   Print: 3    Move: temp = null

Step 4: Stop
temp = null       Stop! (end of list)

Output: 1 2 3

HEAD still = 100 вң“ (Never changed!)

рҹ’» Traversal Code (Fully Commented)

/**
 * Traversal - Visit every node and print data
 * 
 * Time: O(N) - visit each node once
 * Space: O(1) - only one temporary pointer
 */
void traverse(Node head) {
    // Create temporary pointer - NEVER move head!
    Node t = head;
    
    // Loop until we reach null (end of list)
    while (t != null) {
        // Print current node's data
        System.out.print(t.data + " ");
        
        // Move to next node (follow the address)
        t = t.next;
    }
}

рҹ’» Insert at Head - Code (Fully Commented)

/**
 * Insert at Beginning (Head)
 * 
 * Time: O(1) - constant time, no traversal needed
 * Space: O(1) - only creating one new node
 */
Node insertAtHead(Node head, int value) {
    // Step 1: Create new node
    // System allocates memory, let's say at address 400
    Node newNode = new Node(value);
    
    // Step 2: Connect new node to existing list FIRST
    // newNode.next = head (400 points to 100)
    // This preserves the chain!
    newNode.next = head;
    
    // Step 3: Update HEAD to point to new node
    // head = newNode (HEAD now stores 400)
    head = newNode;
    
    // Return updated head
    return head;
}

/*
 * CRITICAL: Why order matters
 * 
 * вқҢ WRONG ORDER:
 * head = newNode;        // Lost the old list!
 * newNode.next = head;   // Now points to itself!
 * 
 * вң“ CORRECT ORDER:
 * newNode.next = head;   // Save the chain first
 * head = newNode;        // Now safe to update HEAD
 */

рҹ’» Insert in Middle - Code (Fully Commented)

/**
 * Insert After a Given Node (Middle Insertion)
 * 
 * Example: Insert after head (becomes 2nd node)
 * 
 * Time: O(1) - if we already have the node reference
 * Space: O(1) - only one new node created
 */
void insertAfter(Node prevNode, int value) {
    // Safety check
    if (prevNode == null) {
        System.out.println("Previous node cannot be null");
        return;
    }
    
    // Step 1: Create new node (system gives it address 400)
    Node newNode = new Node(value);
    
    // Step 2: FIX RIGHT SIDE FIRST! (Critical!)
    // Save the address of node 2 BEFORE we lose it
    // newNode.next = prevNode.next (400.next = 200)
    // This connects new node to rest of list
    newNode.next = prevNode.next;
    
    // Step 3: FIX LEFT SIDE (Now safe)
    // Connect previous node to new node
    // prevNode.next = newNode (100.next = 400)
    prevNode.next = newNode;
    
    // Done! Chain: 1 вҶ’ 4 вҶ’ 2 вҶ’ 3
}

/*
 * MEMORY AID: "Save before you break"
 * 
 * Think of it like a bridge:
 * 1. Build the NEW bridge section first (right side)
 * 2. THEN connect it to the old bridge (left side)
 * 
 * If you break the old bridge first, people fall!
 * If you build the new section first, everyone stays safe!
 * 
 * Same with linked lists:
 * 1. Connect new node to REST of list (save address)
 * 2. THEN connect previous node to new node
 */

8пёҸвғЈ Golden Rules

вң… Two Rules to Rule Them All

Rule #1: HEAD Check

Does HEAD contain the address of the VERY FIRST node?

вң“ If yes, you can access the list
вң— If no, list is lost!

Rule #2: NEXT Check

Does every node's NEXT contain the correct address of the following node?

вң“ Last node should point to null
вң“ All others should point to valid addresses

рҹ“қ Problem-Solving Checklist

  1. вңҸпёҸ DRAW the nodes and addresses on paper
  2. рҹ”Қ IDENTIFY which links are wrong
  3. рҹ“Ӣ WRITE the fix order (right side first!)
  4. рҹ’» CODE the solution
  5. вң… VERIFY: Check both golden rules!
