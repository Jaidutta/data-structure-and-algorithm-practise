📚 Stack Fundamentals

Before diving into patterns, let's understand how stacks work!

🎯 What is a Stack?

A stack is a linear data structure that follows the LIFO (Last In, First Out) principle. Think of it like a stack of plates - you can only add or remove plates from the top!

Real-Life Analogy
🥞

📚

Last book in, first book out
Key Property - LIFO:
  • The last element added is the first one removed
  • Can only access the top element
  • All operations happen at one end (the top)

💻 Complete Working Example

// Create a stack in Java
Stack<Integer> stack = new Stack<>();

// Check if empty
System.out.println(stack.isEmpty());  // Output: true

// Push elements
stack.push(10);  // Stack: [10]
stack.push(20);  // Stack: [10, 20]
stack.push(30);  // Stack: [10, 20, 30]

System.out.println(stack.peek());  // Output: 30

System.out.println(stack.size());  // Output: 1

// Check what's left
System.out.println(stack.peek());  // Output: 10

// Pop last element
stack.pop();  // Stack is now empty

// Check if empty again
System.out.println(stack.isEmpty());  // Output: true

Pattern 1-4: Next/Previous Greater/Smaller Elements

1. Next Greater Element I

Easy NGE LeetCode #496

You are given two arrays (without duplicates) nums1 and nums2 where nums1's elements are subset of nums2. Find all the next greater numbers for nums1's elements in the corresponding places of nums2.

💡 Intuition: Why Stack?

The Problem: For each element, we need to find the NEXT (to the right) element that is GREATER. Naive approach would check all elements to the right = O(n²).

Stack Insight: When we traverse right-to-left, we can "remember" elements we've seen. If current element is larger than stack top, that stack element will NEVER be the answer for anything to the left (we found something bigger). So we pop it!

Why Right-to-Left? We're looking for "next" (to the right), so we process from right to build up our knowledge of what's coming.

Key Property: Stack maintains elements in decreasing order. The first element in stack that's larger than current is the NGE!

Example: Input: nums1 = [4,1,2], nums2 = [1,3,4,2] Output: [-1,3,-1] Explanation: For number 4 in nums1, you cannot find next greater in nums2, so output -1. For number 1 in nums1, next greater in nums2 is 3. For number 2 in nums1, there is no next greater in nums2, so output -1.

Approach:
  1. Build NGE map for all nums2 elements using stack

  5. Lookup nums1 elements in the map
public int[] nextGreaterElement(int[] nums1, int[] nums2) {
    Map<Integer, Integer> map = new HashMap<>();
    Stack<Integer> stack = new Stack<>();
    
    // Traverse nums2 from right to left
    for (int i = nums2.length - 1; i >= 0; i--) {
        // Pop all smaller or equal elements
        while (!stack.isEmpty() && stack.peek() <= nums2[i]) {
            stack.pop();
        }
        
        // Map current element to its NGE
        map.put(nums2[i], stack.isEmpty() ? -1 : stack.peek());
        stack.push(nums2[i]);
    }
    
    // Build result for nums1
    int[] result = new int[nums1.length];
    for (int i = 0; i < nums1.length; i++) {
        result[i] = map.get(nums1[i]);
    }
    
    return result;
}
Time Complexity: O(n + m) where n = nums2.length, m = nums1.length
Space Complexity: O(n) for stack and map

2. Next Greater Element II (Circular Array)

Medium NGE Circular LeetCode #503

Given a circular integer array nums, return the next greater number for every element. The next greater number is the first greater number traveling from right to left in a circular manner.

💡 Intuition: Why Stack + Circular?

The Challenge: Array is circular - after last element, we wrap around to first element. So last element might have its NGE at the beginning!

Stack Solution: Same as regular NGE, but we need to "simulate" the circular nature by traversing the array TWICE (indices 0 to 2n-1).

The Trick: Use modulo operator (i % n) to map virtual indices to actual array indices. First pass builds knowledge, second pass uses it.

Why This Works: After first pass, stack knows all potential NGE candidates. Second pass resolves the elements that didn't find NGE in first pass (they wrap around).

Example 1: Input: nums = [1,2,1] Output: [2,-1,2] Explanation: First 1's NGE is 2, 2's NGE is -1, last 1's NGE is 2 (circular). Example 2: Input: nums = [1,2,3,4,3] Output: [2,3,4,-1,4]

Explanation:
• nums[0]=1 → NGE is nums[1]=2
• nums[1]=2 → No greater element (it's the max)
• nums[2]=1 → NGE is nums[1]=2 (circular!)
💡 Key Insight: The modulo operator (i % n) lets us "pretend" we have array [...nums, ...nums] without actually duplicating it! 
public int[] nextGreaterElements(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1);
    Stack<Integer> stack = new Stack<>();
    
    // Traverse twice to handle circular nature
    for (int i = 0; i < 2 * n; i++) {
        int num = nums[i % n];
        
        // Pop all indices with smaller values
        while (!stack.isEmpty() && nums[stack.peek()] < num) {
            result[stack.pop()] = num;
        }
        
        // Only push indices in first pass
        if (i < n) {
            stack.push(i);
        }
    }
    
    return result;
}
Time Complexity: O(n) - each element pushed/popped at most once
Space Complexity: O(n) for the stack

3. Next Greater Node In Linked List

Medium NGE on LinkedList LeetCode #1019

Given the head of a linked list, return an array of integers answer where answer[i] is the next greater element for the value at the ith node (0-indexed). If there is no next greater element, answer[i] = 0.

Example: Input: head = [2,1,5] Output: [5,5,0] Input: head = [2,7,4,3,5] Output: [7,0,5,5,0] 
public int[] nextLargerNodes(ListNode head) {
    // First, get all values in array
    List<Integer> list = new ArrayList<>();
    while (head != null) {
        list.add(head.val);
        head = head.next;
    }
    
    int n = list.size();
    int[] result = new int[n];
    Stack<Integer> stack = new Stack<>(); // Store indices
    
    // Traverse left to right
    for (int i = 0; i < n; i++) {
        // Pop all indices with smaller values
        while (!stack.isEmpty() && list.get(stack.peek()) < list.get(i)) {
            result[stack.pop()] = list.get(i);
        }
        stack.push(i);
    }
    
    return result;
}
Time Complexity: O(n)
Space Complexity: O(n)

4. Buildings With an Ocean View

Medium Previous Greater Variant LeetCode #1762

There are n buildings in a line. Given an integer array heights representing the building heights, return a list of indices (0-indexed) of buildings that have an ocean view (no taller building to the right).

Example: Input: heights = [4,2,3,1] Output: [0,2,3] Explanation: Buildings at 0,2,3 have ocean view (no taller building to the right). Input: heights = [4,3,2,1] Output: [0,1,2,3] All buildings have ocean view (strictly decreasing). 
public int[] findBuildings(int[] heights) {
    int n = heights.length;
    Stack<Integer> stack = new Stack<>();
    
    // Traverse from right to left
    for (int i = n - 1; i >= 0; i--) {
        // Pop all buildings shorter than or equal to current
        while (!stack.isEmpty() && heights[stack.peek()] < heights[i]) {
            stack.pop();
        }
        
        // If stack empty after popping, this building has ocean view
        if (stack.isEmpty()) {
            stack.push(i);
        }
    }
    
    // Stack now has indices in reverse order, convert to array
    int[] result = new int[stack.size()];
    for (int i = result.length - 1; i >= 0; i--) {
        result[i] = stack.pop();
    }
    
    return result;
}
Time Complexity: O(n)
Space Complexity: O(n)
Pattern 5-6: Monotonic Stack

5. Daily Temperatures

Medium Monotonic Increasing LeetCode #739

Given an array of daily temperatures, return an array where each element tells you how many days you have to wait until a warmer temperature. If there is no future day, put 0.

💡 Intuition: Why Stack?

The Problem: For each day, find how many days until warmer weather. This is essentially "next greater element" but we need the DISTANCE, not the element itself.

Why Not Nested Loops? For each day, checking all future days = O(n²). With 10,000 days, that's 100 million operations!

Stack Insight: Stack stores indices of days waiting for warmer temperature. When we find a warmer day, we can resolve ALL previous colder days immediately! Stack maintains "unresolved" days in increasing temperature order.

Why Left-to-Right? We process chronologically (day by day). When today is warmer, we know the answer for all previous colder days (today is their answer!)

Efficiency: Each index pushed once, popped once = O(n) total!

Example: Input: temperatures = [73,74,75,71,69,72,76,73] Output: [1,1,4,2,1,1,0,0] Explanation: Day 0: Next warmer is day 1 (74 > 73) → wait 1 day Day 1: Next warmer is day 2 (75 > 74) → wait 1 day Day 2: Next warmer is day 6 (76 > 75) → wait 4 days

public int[] dailyTemperatures(int[] temperatures) {
    int n = temperatures.length;
    int[] result = new int[n];
    Stack<Integer> stack = new Stack<>(); // Store indices
    
    for (int i = 0; i < n; i++) {
        // Found warmer day - resolve all previous colder days
        while (!stack.isEmpty() && 
               temperatures[i] > temperatures[stack.peek()]) {
            int idx = stack.pop();
            result[idx] = i - idx;
        }
        stack.push(i);
    }
    
    return result;
}
Time Complexity: O(n) - each index pushed/popped once
Space Complexity: O(n)

6. Online Stock Span

Medium Stock Span LeetCode #901

Design an algorithm that collects daily price quotes and returns the span of that stock's price for the current day. The span is the maximum number of consecutive days (starting from today and going backward) for which the stock price was less than or equal to today's price.

Example: StockSpanner stockSpanner = new StockSpanner(); stockSpanner.next(100); // return 1 stockSpanner.next(80); // return 1 stockSpanner.next(60); // return 1 stockSpanner.next(70); // return 2 stockSpanner.next(60); // return 1 stockSpanner.next(75); // return 4 stockSpanner.next(85); // return 6 
class StockSpanner {
    private Stack<int[]> stack; // [price, span]
    
    public StockSpanner() {
        stack = new Stack<>();
    }
    
    public int next(int price) {
        int span = 1;
        
        // Pop all prices less than or equal to current
        while (!stack.isEmpty() && stack.peek()[0] <= price) {
            span += stack.pop()[1]; // Add their spans
        }
        
        stack.push(new int[]{price, span});
        return span;
    }
}
Time Complexity: O(1) amortized per operation
Space Complexity: O(n) where n is number of calls

7. Trapping Rain Water

Hard Rain Water LeetCode #42

Given n non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it can trap after raining.

💡 Intuition: Why Stack?

The Problem: Water is trapped between bars when there's a taller bar on BOTH left and right. We need to find these "containers" efficiently.

Why Stack? Stack remembers potential LEFT boundaries as we traverse. When we encounter a taller bar (RIGHT boundary), we can calculate water trapped between left and right!

Key Insight - Layer by Layer: Stack processes water HORIZONTALLY, layer by layer from bottom up. Each time we find a right wall, we fill one layer of water.

The Magic: When current bar is taller than stack top, stack top becomes the BOTTOM of a container. The bar below it in stack is LEFT wall, current bar is RIGHT wall.

Formula: Water = width × height, where:

  • width = distance between left and right walls
  • height = min(left_height, right_height) - bottom_height

• Width: 4 - 0 - 1 = 3
• Height: min(3,4) - 2 = 1
• Water: 3 × 1 = 3 units

Stack: [0], continue checking...

4 > 3, but after popping index 0, stack is empty → stop
Push index 4 → Stack: [4]

Water: 2 + 2 + 3 = 7 units ✓
Total Water Trapped: 7 units ✓

Breakdown:
• Between indices 0-2: 2 units (one column)
• Between indices 2-4: 5 units (2 bottom + 3 top layer)

Why layer-by-layer?
Stack naturally processes from bottom to top. When we find right wall, we calculate water at current level, then move up to next level! 
public int trap(int[] height) {
    Stack<Integer> stack = new Stack<>();
    int water = 0;
    
    for (int i = 0; i < height.length; i++) {
        // Found a taller bar - calculate trapped water
        while (!stack.isEmpty() && height[i] > height[stack.peek()]) {
            int bottom = stack.pop();
            
            if (stack.isEmpty()) break;
            
            int width = i - stack.peek() - 1;
            int boundedHeight = Math.min(height[i], height[stack.peek()]) 
                                 - height[bottom];
            water += width * boundedHeight;
        }
        stack.push(i);
    }
    
    return water;
}
Time Complexity: O(n)
Space Complexity: O(n)

8. Largest Rectangle in Histogram

Hard Histogram LeetCode #84

Given an array of integers heights representing the histogram's bar height where the width of each bar is 1, return the area of the largest rectangle in the histogram.

Example: Input: heights = [2,1,5,6,2,3] Output: 10 Explanation: The largest rectangle has height 5 and width 2, area = 10 
public int largestRectangleArea(int[] heights) {
    Stack<Integer> stack = new Stack<>();
    int maxArea = 0;
    
    for (int i = 0; i <= heights.length; i++) {
        int h = (i == heights.length) ? 0 : heights[i];
        
        while (!stack.isEmpty() && h < heights[stack.peek()]) {
            int height = heights[stack.pop()];
            int width = stack.isEmpty() ? i : i - stack.peek() - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }
    
    return maxArea;
}
Time Complexity: O(n)
Space Complexity: O(n)
Time Complexity: O(n)
Space Complexity: O(n)

9. Valid Parentheses

Easy Bracket Matching LeetCode #20

Given a string containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid. An input string is valid if brackets are closed in the correct order.

💡 Intuition: Why Stack?

The Problem: Brackets must close in REVERSE order they opened. The MOST RECENTLY opened bracket must be the next one to close - this is LIFO!

Stack = Perfect Match: Stack IS a LIFO structure. Push opening brackets '(', '[', '{'. When we see closing ')', ']', '}', check if stack top matches. If yes, they cancel out (pop). If no, invalid.

Why Not Counter? Because ORDER matters! "(]" is invalid even though counts match. Stack preserves nesting structure.

Real-World: Like stacking plates - you can only remove the top plate. Same with brackets!

Example 1: Input: s = "()" Output: true Example 2: Input: s = "()[]{}" Output: true Example 3: Input: s = "(]" Output: false Example 4: Input: s = "([)]" Output: false (wrong nesting order) 
public boolean isValid(String s) {
    Stack<Character> stack = new Stack<>();
    
    for (char c : s.toCharArray()) {
        // Push opening brackets
        if (c == '(' || c == '[' || c == '{') {
            stack.push(c);
        }
        // Check closing brackets
        else {
            if (stack.isEmpty()) return false;
            
            char top = stack.pop();
            if (c == ')' && top != '(') return false;
            if (c == ']' && top != '[') return false;
            if (c == '}' && top != '{') return false;
        }
    }
    
    // All brackets matched if stack is empty
    return stack.isEmpty();
}
Time Complexity: O(n)
Space Complexity: O(n) in worst case (all opening brackets)

10. Asteroid Collision

Medium Collision Simulation LeetCode #735

We are given an array of integers representing asteroids in a row. For each asteroid, the absolute value represents its size, and the sign represents its direction (positive = right, negative = left). Find out the state of the asteroids after all collisions.

💡 Intuition: Why Stack?

The Problem: Asteroids moving right (+) collide with asteroids moving left (-). Collisions can CASCADE - one explosion triggers another!

Stack Solution: Stack holds survivors moving right. When left-moving asteroid comes, it collides with MOST RECENT right-moving asteroid (stack top). Keep popping until one survives or both explode.

Why Stack? Collisions happen with the LAST survivor going right, not the first. We need LIFO access!

Example 1: Input: asteroids = [5,10,-5] Output: [5,10] Explanation: -5 hits 10, -5 explodes. Example 2: Input: asteroids = [8,-8] Output: [] Explanation: 8 and -8 collide and both explode. Example 3: Input: asteroids = [10,2,-5] Output: [10] Explanation: -5 destroys 2, then hits 10. 10 wins. 
public int[] asteroidCollision(int[] asteroids) {
    Stack<Integer> stack = new Stack<>();
    
    for (int ast : asteroids) {
        boolean alive = true;
        
        // Left-moving asteroid collides with right-moving ones
        while (alive && ast < 0 && !stack.isEmpty() && stack.peek() > 0) {
            if (stack.peek() < -ast) {
                stack.pop(); // Right one explodes, left continues
            } else if (stack.peek() == -ast) {
                stack.pop(); // Both explode
                alive = false;
            } else {
                alive = false; // Left one explodes
            }
        }
        
        if (alive) {
            stack.push(ast);
        }
    }
    
    // Convert stack to array
    int[] result = new int[stack.size()];
    for (int i = result.length - 1; i >= 0; i--) {
        result[i] = stack.pop();
    }
    return result;
}
Time Complexity: O(n)
Space Complexity: O(n)

11. Evaluate Reverse Polish Notation

Medium Postfix Evaluation LeetCode #150

Evaluate the value of an arithmetic expression in Reverse Polish Notation (postfix). Valid operators are +, -, *, and /. Each operand may be an integer or another expression.

💡 Intuition: Why Stack?

The Problem: In postfix "3 4 + 5 *", operators come AFTER operands. We need to "remember" operands until we see their operator.

Stack Solution: Push numbers. When we hit an operator, pop TWO operands (most recent = right, next = left), compute, push result back.

Why Perfect? The TWO most recently pushed numbers are the operands - exactly LIFO! Postfix notation was DESIGNED for stack evaluation.

Example 1: Input: tokens = ["2","1","+","3","*"] Output: 9 Explanation: ((2 + 1) * 3) = 9 Example 2: Input: tokens = ["4","13","5","/","+"] Output: 6 Explanation: (4 + (13 / 5)) = 6 
public int evalRPN(String[] tokens) {
    Stack<Integer> stack = new Stack<>();
    
    for (String token : tokens) {
        if (token.equals("+") || token.equals("-") || 
            token.equals("*") || token.equals("/")) {
            
            // Pop two operands (order matters!)
            int b = stack.pop(); // Right operand
            int a = stack.pop(); // Left operand
            
            int result = 0;
            if (token.equals("+")) result = a + b;
            else if (token.equals("-")) result = a - b;
            else if (token.equals("*")) result = a * b;
            else if (token.equals("/")) result = a / b;
            
            stack.push(result);
        } else {
            // It's a number, push to stack
            stack.push(Integer.parseInt(token));
        }
    }
    
    return stack.pop();
}
Time Complexity: O(n)
Space Complexity: O(n)

12. Decode String

Medium Nested Decoding LeetCode #394

Given an encoded string, return its decoded string. The encoding rule is: k[encoded_string], where the encoded_string inside the square brackets is repeated exactly k times.

💡 Intuition: Why Stack?

The Problem: Nested encoding like "3[a2[c]]" requires processing innermost first: 2[c]→cc, then a2[c]→acc, then 3[acc]→accaccacc.

Stack Solution: Use TWO stacks (counts + strings). When we see '[', save current state and start fresh. When we see ']', pop saved state, repeat current string, append to previous context.

The Magic: Each '[' creates a new nesting level. Stack suspends outer level, works on inner, then resumes outer with inner's result!

Example 1: Input: s = "3[a]2[bc]" Output: "aaabcbc" Example 2: Input: s = "3[a2[c]]" Output: "accaccacc" Example 3: Input: s = "2[abc]3[cd]ef" Output: "abcabccdcdcdef" 
public String decodeString(String s) {
    Stack<Integer> countStack = new Stack<>();
    Stack<String> stringStack = new Stack<>();
    String currentString = "";
    int k = 0;
    
    for (char c : s.toCharArray()) {
        if (Character.isDigit(c)) {
            k = k * 10 + (c - '0'); // Build multi-digit number
        } 
        else if (c == '[') {
            // Save current state, start new level
            countStack.push(k);
            stringStack.push(currentString);
            currentString = "";
            k = 0;
        } 
        else if (c == ']') {
            // Complete current level
            StringBuilder decodedString = new StringBuilder(stringStack.pop());
            int repeatTimes = countStack.pop();
            for (int i = 0; i < repeatTimes; i++) {
                decodedString.append(currentString);
            }
            currentString = decodedString.toString();
        } 
        else {
            currentString += c; // Regular character
        }
    }
    
    return currentString;
}
Time Complexity: O(n)
Space Complexity: O(n)
