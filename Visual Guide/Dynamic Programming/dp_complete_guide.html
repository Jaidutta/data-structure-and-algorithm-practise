📖 What is Dynamic Programming?

Definition

Dynamic Programming (DP): An optimization technique that solves complex problems by breaking them down into simpler subproblems and storing the results to avoid redundant calculations.

DP = Recursion + Memoization
(Store results to avoid recomputation)

The Simple Explanation:

Imagine you're climbing stairs and someone asks "How many ways to reach step 10?" Instead of recounting every path from scratch each time, you write down the answer and reuse it. That's DP!

🎯 Real-World Analogy: The Student's Homework Problem

Scenario: You have 10 math problems to solve.

❌ WITHOUT DP (Naive Approach):

• Problem 5 requires solving problem 3

• Problem 7 ALSO requires solving problem 3

• Problem 9 ALSO requires solving problem 3

• You solve problem 3 THREE TIMES - wasting time!

✅ WITH DP (Smart Approach):

• Solve problem 3 once

• Write down the answer in your notebook

• When problem 5 needs it → look at notebook!

• When problem 7 needs it → look at notebook!

• When problem 9 needs it → look at notebook!

You solved problem 3 ONCE, used the answer THREE TIMES!

Key Characteristics:

1 Optimal Substructure: Solution to problem can be constructed from solutions to subproblems

Example: Shortest path from A to C = Shortest path from A to B + Shortest path from B to C

If you know the best way to get to an intermediate point, you can use that to find the best overall path.

2 Overlapping Subproblems: Same subproblems are solved multiple times

Example: Calculating Fibonacci(5) requires Fibonacci(3) twice

Without DP: compute Fibonacci(3) two separate times (wasteful!)

With DP: compute Fibonacci(3) once, store it, reuse it (efficient!)

3 Memoization/Tabulation: Store results to avoid redundant computation

Memoization (Top-Down): Store results in a cache as you compute them

Tabulation (Bottom-Up): Fill a table from smallest to largest subproblem

⚠️ When NOT to Use DP

DP is NOT the right tool when:

  • No overlapping subproblems: If each subproblem is unique, DP doesn't help (example: Merge Sort - each partition is different)
  • Greedy works: If a greedy choice always leads to optimal solution (example: Activity Selection)
  • Simple recursion is fast enough: Small input sizes where redundancy doesn't matter

🔄 Top-Down vs Bottom-Up Approaches

Top-Down (Memoization)

Approach: Start from the problem, recursively break down

Method: Recursion + Cache (HashMap/Array)

// Top-Down Template
int[] memo;

int solve(int n) {
    // Base case
    if (n <= 1) return n;
    
    // Check cache
    if (memo[n] != -1) {
        return memo[n];
    }
    
    // Compute & store
    memo[n] = solve(n-1) + solve(n-2);
    return memo[n];
}

Pros:

  • Natural to write (follows recursion)
  • Only computes needed subproblems
  • Easier to understand initially

Cons:

  • Recursion overhead (stack space)
  • Risk of stack overflow for large n

Bottom-Up (Tabulation)

Approach: Start from base case, build up to solution

Method: Iteration + DP Table (Array)

// Bottom-Up Template
int solve(int n) {
    // Create DP table
    int[] dp = new int[n + 1];
    
    // Base cases
    dp[0] = 0;
    dp[1] = 1;
    
    // Fill table iteratively
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    
    return dp[n];
}

Pros:

  • No recursion overhead
  • Better space optimization possible
  • Faster in practice

Cons:

  • Computes all subproblems (even unneeded)
  • Less intuitive initially

Pattern 1: Fibonacci / 1D DP (Linear Sequence)

📝 Pattern 1: Fibonacci-Style Problems

Problem Type: Each state depends on 1-2 previous states

Recurrence: dp[i] = f(dp[i-1], dp[i-2], ...)

Examples: Fibonacci, Climbing Stairs, House Robber

🎯 When to Use This Pattern?

  • ✓ Single variable changing (n, position, step)
  • ✓ Current state depends on previous 1-2 states
  • ✓ "How many ways to reach step n?"
  • ✓ "Maximum profit up to house i"
  • ✓ Array or 1D problem

❌ Problem: Massive Redundancy!

Repeated Calculations:

  • fib(3): computed 2 times
  • fib(2): computed 3 times
  • fib(1): computed 5 times
  • fib(0): computed 3 times

Complexity:

Total Function Calls: 15

Unique Subproblems: 6

Wasted Calls: 9 (60%!)

Time: O(2n) - Exponential 🔥

✅ Solution: Dynamic Programming!

With memoization, each subproblem is computed only once:

1. Compute fib(0) = 0, cache it ✓

2. Compute fib(1) = 1, cache it ✓

3. Compute fib(2) = fib(1) + fib(0) = 1, cache it ✓

4. Compute fib(3) = fib(2) + fib(1) = 2, cache it ✓

5. When fib(3) needed again → return from cache! 🚀

Time: O(n) - Linear ✨
From 15 calls → 6 calls (60% reduction!)

✅ Solution: Memoization

Call fib(5):
  Call fib(4):
    Call fib(3):
      Call fib(2): returns 1, CACHE[2] = 1
      Call fib(1): returns 1
      fib(3) = 2, CACHE[3] = 2
    Call fib(2): FOUND IN CACHE! returns 1
    fib(4) = 3, CACHE[4] = 3
  Call fib(3): FOUND IN CACHE! returns 2
  fib(5) = 5, CACHE[5] = 5

Each subproblem computed ONCE! Time: O(n)
            
// Approach 1: Top-Down (Recursion + Memoization)
class Solution {
    public int fib(int n) {
        int[] memo = new int[n + 1];
        Arrays.fill(memo, -1);
        return fibHelper(n, memo);
    }
    
    private int fibHelper(int n, int[] memo) {
        // Base cases
        if (n <= 1) return n;
        
        // Check if already computed
        if (memo[n] != -1) {
            return memo[n];  // Return cached result
        }
        
        // Compute and cache
        memo[n] = fibHelper(n - 1, memo) + fibHelper(n - 2, memo);
        return memo[n];
    }
}

// Approach 2: Bottom-Up (Tabulation)
public int fib(int n) {
    if (n <= 1) return n;
    
    // Create DP table
    int[] dp = new int[n + 1];
    
    // Base cases
    dp[0] = 0;
    dp[1] = 1;
    
    // Fill table from bottom up
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    
    return dp[n];
}

// Approach 3: Space Optimized Bottom-Up
public int fib(int n) {
    if (n <= 1) return n;
    
    // Only need previous 2 values
    int prev2 = 0, prev1 = 1;
    
    for (int i = 2; i <= n; i++) {
        int current = prev1 + prev2;
        prev2 = prev1;
        prev1 = current;
    }
    
    return prev1;
}
// Space: O(n) → O(n) → O(1) optimization!

  • ROB this house: Take its value + rob from house (i-2)
  • SKIP this house: Rob from house (i-1)

dp[i] = max(nums[i] + dp[i-2], dp[i-1])

• dp[0] = 2 (rob first house)

• dp[2] = max(7, 9+2) = 11 (rob houses 0 and 2)

• dp[3] = max(11, 3+7) = 11 (don't rob house 3)

• dp[4] = max(11, 1+11) = 12 (rob house 4)

Pattern 2: 0/1 Knapsack

📝 Pattern 2: 0/1 Knapsack Problems

Problem Type: Make binary choice (include/exclude) for each item

Recurrence: dp[i][w] = max(include item i, exclude item i)

Examples: Subset Sum, Partition Equal, Target Sum

🎯 When to Use This Pattern?

  • ✓ Have a set of items with weights/values
  • ✓ Can ONLY use each item ONCE (0/1 choice)
  • ✓ Have a capacity/target constraint
  • ✓ "Maximum value with constraint"
  • ✓ "Can you achieve target sum?"
  • ✓ "Count subsets that sum to target"

Example: 0/1 Knapsack

Problem:

Items: weights = [1, 3, 4, 5], values = [1, 4, 5, 7]

Capacity: W = 7

Goal: Maximize value without exceeding capacity

🌳 Decision Tree: Knapsack with First 2 Items, Capacity 7

                         knapsack(items=4, capacity=7)
                                    max value = ?
                              ╱                              ╲
                            ╱                                  ╲
                          ╱                                      ╲
              INCLUDE item 0                          EXCLUDE item 0
              (weight=1, value=1)                          (skip item 0)
                      │                                            │
                      ↓                                            ↓
        value=1, remaining=6                    value=0, remaining=7
         knapsack(items=3, cap=6)              knapsack(items=3, cap=7)
                 ╱          ╲                                 ╱          ╲
               ╱              ╲                             ╱              ╲
     INCLUDE item 1  EXCLUDE item 1       INCLUDE item 1  EXCLUDE item 1
     (w=3, v=4)           (skip)            (w=3, v=4)           (skip)
          │                  │                   │                  │
   val=5          val=1            val=4          val=0
   cap=3          cap=6            cap=4          cap=7
    ...              ...               ...              ...

🎯 The Two Choices at Each Step:

✅ INCLUDE Item i

Condition: weight[i] ≤ capacity

Action: Take value[i]

Recurse: Solve for remaining items with reduced capacity

value[i] + knapsack(i+1, cap-weight[i])

⏭️ EXCLUDE Item i

Condition: Always possible

Action: Skip this item (take 0 value)

Recurse: Solve for remaining items with same capacity

knapsack(i+1, capacity)

dp[i][w] = max(include, exclude)

Take the maximum of both choices!

⚠️ Without DP: Exponential Time!

Each item has 2 choices → 2 × 2 × 2 × 2 = 2n paths to explore!

For 20 items: 220 = 1,048,576 function calls!

// Top-Down Approach
class Solution {
    public int knapsack(int[] weights, int[] values, int W) {
        int n = weights.length;
        int[][] memo = new int[n][W + 1];
        for (int[] row : memo) Arrays.fill(row, -1);
        return knapsackHelper(0, W, weights, values, memo);
    }
    
    private int knapsackHelper(int i, int w, int[] weights, 
                                   int[] values, int[][] memo) {
        // Base case: no items left or no capacity
        if (i >= weights.length || w == 0) {
            return 0;
        }
        
        // Check cache
        if (memo[i][w] != -1) return memo[i][w];
        
        // Choice 1: Exclude current item
        int exclude = knapsackHelper(i + 1, w, weights, values, memo);
        
        // Choice 2: Include current item (if it fits)
        int include = 0;
        if (weights[i] <= w) {
            include = values[i] + knapsackHelper(i + 1, w - weights[i], 
                                                  weights, values, memo);
        }
        
        // Take maximum and cache
        memo[i][w] = Math.max(include, exclude);
        return memo[i][w];
    }
}

// Bottom-Up Approach
public int knapsack(int[] weights, int[] values, int W) {
    int n = weights.length;
    int[][] dp = new int[n + 1][W + 1];
    
    // Fill table
    for (int i = 1; i <= n; i++) {
        for (int w = 1; w <= W; w++) {
            // Exclude current item
            dp[i][w] = dp[i - 1][w];
            
            // Include current item if it fits
            if (weights[i - 1] <= w) {
                dp[i][w] = Math.max(dp[i][w], 
                                    values[i - 1] + dp[i - 1][w - weights[i - 1]]);
            }
        }
    }
    
    return dp[n][W];
}

Pattern 3: Longest Common Subsequence (LCS)

📝 Pattern 3: String DP - LCS Style

Problem Type: Compare two strings, find common patterns

Recurrence: if s1[i]==s2[j]: dp[i][j] = 1 + dp[i-1][j-1], else: max(dp[i-1][j], dp[i][j-1])

Examples: LCS, Edit Distance, Distinct Subsequences

🎯 When to Use This Pattern?

  • ✓ Have TWO strings to compare
  • ✓ "Longest common subsequence"
  • ✓ "Edit distance" / "Minimum operations"
  • ✓ "How many distinct ways" (string matching)
  • ✓ 2D DP table with strings as dimensions

Example: Longest Common Subsequence

Problem:

String 1: "ABCDE"

String 2: "ACE"

Goal: Find length of longest subsequence common to both

Answer: 3 (subsequence "ACE")

🎯 The Decision at Each Step:

When comparing s1[i] and s2[j]:

✅ Case 1: Characters MATCH (s1[i] == s2[j])

This character is part of LCS! Include it.

LCS[i][j] = 1 + LCS[i-1][j-1]

Move both pointers forward (i-1, j-1) and add 1 to count

❌ Case 2: Characters DON'T MATCH (s1[i] ≠ s2[j])

Try skipping one character from either string, take the max!

LCS[i][j] = max(LCS[i-1][j], LCS[i][j-1])

Either skip s1[i] OR skip s2[j], whichever gives longer LCS

📊 Calculation for LCS("ABC", "AC"):

  • Compare A vs A: MATCH! → 1 + LCS("BC", "C")
  • Compare B vs C: NO MATCH → max(LCS("C","C"), LCS("BC",""))
  • Compare C vs C: MATCH! → 1 + LCS("", "")
  • Base case: LCS("", "") = 0

Result: 1 + (1 + 0) = 2

LCS = "AC" (length 2) 

// Bottom-Up LCS
public int longestCommonSubsequence(String text1, String text2) {
    int m = text1.length();
    int n = text2.length();
    int[][] dp = new int[m + 1][n + 1];
    
    // Fill table
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                // Characters match - extend LCS
                dp[i][j] = 1 + dp[i - 1][j - 1];
            } else {
                // Characters don't match - take max of excluding one
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }
    
    return dp[m][n];
}

Pattern 4: Longest Increasing Subsequence (LIS)

📝 Pattern 4: LIS and Variants

Problem Type: Find longest subsequence with specific ordering property

Recurrence: dp[i] = max(dp[j] + 1) where arr[j] < arr[i] for all j < i

Time: O(N²) DP or O(N log N) with Binary Search

🎯 When to Use This Pattern?

  • ✓ "Longest increasing subsequence"
  • ✓ "Maximum length with ordering constraint"
  • ✓ "Number of LIS"
  • ✓ Each element depends on ALL previous elements
// LIS - O(N²) DP Solution
public int lengthOfLIS(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1);  // Each element is LIS of length 1
    
    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[i] > nums[j]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
    }
    
    // Return maximum value in dp array
    return Arrays.stream(dp).max().getAsInt();
}

Pattern 5: Palindromes

📝 Pattern 5: Palindrome Problems

Problem Type: Check/find palindromic substrings or subsequences

Recurrence: dp[i][j] = (s[i] == s[j]) && dp[i+1][j-1]

Key: Expand from center or use 2D table

🎯 When to Use This Pattern?

  • ✓ "Palindrome" in problem name
  • ✓ "Longest palindromic substring/subsequence"
  • ✓ "Minimum cuts for palindrome partitioning"
  • ✓ Need to check if substring is palindrome
// Longest Palindromic Substring
public String longestPalindrome(String s) {
    int n = s.length();
    boolean[][] dp = new boolean[n][n];
    int start = 0, maxLen = 1;
    
    // Every single character is a palindrome
    for (int i = 0; i < n; i++) {
        dp[i][i] = true;
    }
    
    // Check substrings of length 2
    for (int i = 0; i < n - 1; i++) {
        if (s.charAt(i) == s.charAt(i + 1)) {
            dp[i][i + 1] = true;
            start = i;
            maxLen = 2;
        }
    }
    
    // Check substrings of length 3+
    for (int len = 3; len <= n; len++) {
        for (int i = 0; i < n - len + 1; i++) {
            int j = i + len - 1;
            if (s.charAt(i) == s.charAt(j) && dp[i + 1][j - 1]) {
                dp[i][j] = true;
                start = i;
                maxLen = len;
            }
        }
    }
    
    return s.substring(start, start + maxLen);
}

🎯 The Pattern:

To reach any cell (i, j), you can ONLY come from two places:

➡️ From LEFT (i, j-1)

Move RIGHT to reach (i, j)

    [i, j-1] ──→ [i, j]

⬇️ From TOP (i-1, j)

Move DOWN to reach (i, j)

    [i-1, j]
        │
        ↓
    [i, j]

paths[i][j] = paths[i-1][j] + paths[i][j-1]

Total paths = (paths from top) + (paths from left)

Step-by-Step Calculation:

• dp[0][0] = 1 (start position)

• dp[0][1] = 1 (can only come from left)

• dp[0][2] = 1 (can only come from left)

• dp[1][0] = 1 (can only come from top)

• dp[1][1] = dp[0][1] + dp[1][0] = 1 + 1 = 2

• dp[1][2] = dp[0][2] + dp[1][1] = 1 + 2 = 3

• dp[2][0] = 1 (can only come from top)

• dp[2][1] = dp[1][1] + dp[2][0] = 2 + 1 = 3

• dp[2][2] = dp[1][2] + dp[2][1] = 3 + 3 = 6 ✓

// Unique Paths - Bottom-Up
public int uniquePaths(int m, int n) {
    int[][] dp = new int[m][n];
    
    // Base case: first column (can only go down)
    for (int i = 0; i < m; i++) {
        dp[i][0] = 1;
    }
    
    // Base case: first row (can only go right)
    for (int j = 0; j < n; j++) {
        dp[0][j] = 1;
    }
    
    // Fill rest of table
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            // Paths from top + paths from left
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
        }
    }
    
    return dp[m - 1][n - 1];
}

// Min Path Sum - Similar pattern but take minimum
public int minPathSum(int[][] grid) {
    int m = grid.length, n = grid[0].length;
    int[][] dp = new int[m][n];
    
    dp[0][0] = grid[0][0];
    
    // First column
    for (int i = 1; i < m; i++) {
        dp[i][0] = dp[i - 1][0] + grid[i][0];
    }
    
    // First row
    for (int j = 1; j < n; j++) {
        dp[0][j] = dp[0][j - 1] + grid[0][j];
    }
    
    // Fill table - take MINIMUM path
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            dp[i][j] = grid[i][j] + Math.min(dp[i - 1][j], dp[i][j - 1]);
        }
    }
    
    return dp[m - 1][n - 1];
}

💡 Intuition

Why does this work?

  1. Optimal Substructure: To reach cell (i,j), you MUST come from either (i-1,j) or (i,j-1)
  2. Combine Solutions: Number of ways to (i,j) = ways to (i-1,j) + ways to (i,j-1)
  3. Build Up: Start from base cases (first row/column) and build to target

Pattern 7: Unbounded Knapsack

📝 Pattern 7: Unbounded Knapsack (Unlimited Use)

Problem Type: Like 0/1 Knapsack BUT can use each item UNLIMITED times

Key Difference: After using item i, can use it again → dp[i][w-weight] instead of dp[i-1][w-weight]

Examples: Coin Change, Rod Cutting, Combination Sum

🎯 When to Use This Pattern?

  • ✓ "Unlimited" or "as many times as you want"
  • ✓ "Minimum coins to make amount"
  • ✓ "Number of ways to make amount"
  • ✓ Can reuse items/coins/elements
  • ✓ Different from 0/1 where each item used ONCE

Example: Coin Change

Coins: [1, 2, 5]

Amount: 11

Goal: Minimum number of coins to make 11

// Coin Change - Unbounded Knapsack
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);  // Initialize to impossible value
    dp[0] = 0;  // Base case: 0 coins for 0 amount
    
    // For each amount from 1 to target
    for (int i = 1; i <= amount; i++) {
        // Try each coin
        for (int coin : coins) {
            if (i >= coin) {
                // KEY: Can use same coin again → dp[i - coin]
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
    }
    
    return dp[amount] > amount ? -1 : dp[amount];
}

// Coin Change 2 - Count number of ways
public int change(int amount, int[] coins) {
    int[] dp = new int[amount + 1];
    dp[0] = 1;  // 1 way to make 0: use no coins
    
    // For each coin type
    for (int coin : coins) {
        // Update all amounts that can include this coin
        for (int i = coin; i <= amount; i++) {
            dp[i] += dp[i - coin];  // Add ways using this coin
        }
    }
    
    return dp[amount];
}

💡 Key Difference from 0/1 Knapsack

0/1 Knapsack (each item ONCE):

dp[i][w] = max(dp[i-1][w], value[i] + dp[i-1][w-weight[i]])
                                                     ↑ i-1 (can't reuse)

Unbounded Knapsack (unlimited use):

dp[i] = min(dp[i], 1 + dp[i-coin])
                               ↑ i (can reuse same coin!)

Pattern 8: DP on Trees

📝 Pattern 8: Tree DP (Bottom-Up on Tree Structure)

Problem Type: Tree structure, need to compute something for each subtree

Method: DFS + Return values from children to parent

Examples: House Robber III, Binary Tree Cameras, Tree Diameter

🎯 When to Use This Pattern?

  • ✓ Input is a binary tree or tree structure
  • ✓ Need to make decisions at each node
  • ✓ Decision depends on children's results
  • ✓ "Maximum/minimum value in tree with constraint"
  • ✓ Return values from subtrees

Example: House Robber III

Problem:

Rob houses arranged as binary tree. Can't rob two directly-connected houses.

Constraint: If rob parent, can't rob children

        3
       / \
      2   3
       \   \ 
        3   1

Answer: 7 (rob 3 + 3 + 1)

// House Robber III - Tree DP
public int rob(TreeNode root) {
    int[] result = robHelper(root);
    return Math.max(result[0], result[1]);
}

/**
 * Returns [robRoot, notRobRoot]
 * robRoot: max money if we ROB current node
 * notRobRoot: max money if we DON'T rob current node
 */
private int[] robHelper(TreeNode node) {
    if (node == null) {
        return new int[]{0, 0};
    }
    
    // Get results from children (postorder - children first!)
    int[] left = robHelper(node.left);
    int[] right = robHelper(node.right);
    
    // If we rob current node, can't rob children
    int robRoot = node.val + left[1] + right[1];
    
    // If we don't rob current, take max from children (rob or not)
    int notRobRoot = Math.max(left[0], left[1]) + 
                        Math.max(right[0], right[1]);
    
    return new int[]{robRoot, notRobRoot};
}

🌳 Execution Trace

Tree:     3
         / \
        2   3
         \   \
          3   1

Process (Postorder - bottom up):

1. Node(3 - leftmost leaf):
   left = [0,0], right = [0,0]
   rob = 3 + 0 + 0 = 3
   notRob = 0
   return [3, 0]

2. Node(2):
   left = [0,0], right = [3,0]
   rob = 2 + 0 + 0 = 2
   notRob = max(0,0) + max(3,0) = 3
   return [2, 3]

3. Node(1 - rightmost leaf):
   return [1, 0]

4. Node(3 - right of root):
   left = [0,0], right = [1,0]
   rob = 3 + 0 + 0 = 3
   notRob = max(0,0) + max(1,0) = 1
   return [3, 1]

5. Root Node(3):
   left = [2,3], right = [3,1]
   rob = 3 + 3 + 1 = 7
   notRob = max(2,3) + max(3,1) = 3 + 3 = 6
   return [7, 6]

Answer: max(7, 6) = 7

💡 Intuition

Why return TWO values?

  1. Track Both States: Need to know max if we rob AND if we don't rob
  2. Parent Depends on Children: Robbing parent affects what we can do with children
  3. Postorder Traversal: Process children first, then use their results for parent

// Stock with Cooldown - State Machine DP
public int maxProfit(int[] prices) {
    if (prices.length <= 1) return 0;
    
    int n = prices.length;
    
    // Three states
    int hold = -prices[0];  // Bought first stock
    int sold = 0;           // Haven't sold yet
    int rest = 0;           // At rest
    
    for (int i = 1; i < n; i++) {
        int prevHold = hold;
        int prevSold = sold;
        int prevRest = rest;
        
        // Update each state
        hold = Math.max(prevHold, prevRest - prices[i]);  // Keep holding OR buy
        sold = prevHold + prices[i];                      // Sell stock
        rest = Math.max(prevRest, prevSold);              // Rest OR was sold
    }
    
    // Return max of not holding stock
    return Math.max(sold, rest);
}

// Stock with Transaction Fee - Similar approach
public int maxProfit(int[] prices, int fee) {
    int cash = 0;    // Max profit if we don't own stock
    int hold = -prices[0];  // Max profit if we own stock
    
    for (int i = 1; i < prices.length; i++) {
        cash = Math.max(cash, hold + prices[i] - fee);  // Sell (pay fee)
        hold = Math.max(hold, cash - prices[i]);        // Buy
    }
    
    return cash;
}

Pattern 10: Digit DP

📝 Pattern 10: Digit DP (Count Numbers with Constraints)

Problem Type: Count numbers in range with digit-level constraints

Method: Build numbers digit by digit, track tight bound

Examples: Numbers with unique digits, digit sum constraints

🎯 When to Use This Pattern?

  • ✓ "Count numbers in range [L, R]"
  • ✓ "With digit constraints" (unique, sum, etc.)
  • ✓ "Numbers at most N digits"
  • ✓ Build numbers digit by digit
  • ✓ Need to track if still at boundary

Example: Count Numbers with Unique Digits

Problem:

Given n, count numbers from 0 to 10^n - 1 with all unique digits

Example: n=2 → count from 0 to 99

Answer: 91 (all two-digit numbers with unique digits + single digits)

// Count Numbers with Unique Digits
public int countNumbersWithUniqueDigits(int n) {
    if (n == 0) return 1;
    
    int result = 10;  // For n=1: 0-9
    int uniqueDigits = 9;
    int availableDigits = 9;
    
    // For each additional digit position
    for (int i = 2; i <= n && availableDigits > 0; i++) {
        uniqueDigits = uniqueDigits * availableDigits;
        result += uniqueDigits;
        availableDigits--;
    }
    
    return result;
}

// Advanced Digit DP Template (for range queries)
public int countInRange(int low, int high) {
    return countUpTo(high) - countUpTo(low - 1);
}

private int countUpTo(int num) {
    String s = String.valueOf(num);
    int n = s.length();
    
    // dp[pos][tight][... other states]
    // pos: current position in number
    // tight: whether we're still bounded by original number
    
    return solve(0, true, s);
}

private int solve(int pos, boolean tight, String num) {
    // Base case: built complete number
    if (pos == num.length()) {
        return 1;
    }
    
    // Determine max digit we can use at this position
    int maxDigit = tight ? (num.charAt(pos) - '0') : 9;
    
    int count = 0;
    for (int digit = 0; digit <= maxDigit; digit++) {
        // Recurse for next position
        count += solve(pos + 1, tight && (digit == maxDigit), num);
    }
    
    return count;
}

💡 Intuition

Building numbers digit by digit:

  1. Position-wise: Build number from left to right (most significant first)
  2. Tight Bound: Track if we're still at the upper limit (affects which digits we can use)
  3. State Tracking: Keep track of constraints (digits used, sum, etc.)

💡 Pro Tips & Common Mistakes

✅ Pro Tips

  1. Start with recursion: Write recursive solution first, then add memoization
  2. Identify states: What variables change? These become DP dimensions
  3. Base cases first: Always handle base cases explicitly
  4. Draw it out: Sketch recursion tree or DP table to visualize
  5. Space optimization: After getting it working, optimize space (often O(N) → O(1))
  6. Check constraints: Array bounds, negative indices, empty cases

⚠️ Common Mistakes

  1. Forgetting base case: Leads to stack overflow or wrong answer
  2. Wrong initialization: Arrays not initialized to correct default values
  3. Off-by-one errors: Array index mistakes, especially in 2D DP
  4. Not checking memo: Forgetting to check cache before recursive call
  5. Using wrong index: In bottom-up, using dp[i] vs dp[i-1] incorrectly
  6. Greedy when need DP: Not all optimization problems are greedy!

🎓 DP Interview Strategy

Step-by-Step Approach:

1 Recognize it's DP:
  • Keywords: max/min, count ways, optimization
  • Overlapping subproblems visible
2 Define state:
  • What does dp[i] or dp[i][j] represent?
  • "dp[i] = maximum profit using first i houses"
3 Find recurrence relation:
  • How to compute dp[i] from previous states?
  • "dp[i] = max(rob[i] + dp[i-2], dp[i-1])"
4 Identify base cases:
  • dp[0] = ?, dp[1] = ?
  • Empty input, single element cases
5 Code it:
  • Start with top-down (easier to think)
  • Convert to bottom-up if needed
  • Optimize space as final step

// Template 1: 1D DP (Fibonacci style)
int[] dp = new int[n + 1];
dp[0] = base1;
dp[1] = base2;
for (int i = 2; i <= n; i++) {
    dp[i] = f(dp[i-1], dp[i-2]);
}
return dp[n];

// Template 2: 2D DP (Grid/Knapsack)
int[][] dp = new int[m + 1][n + 1];
// Initialize base cases
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        dp[i][j] = f(dp[i-1][j], dp[i][j-1]);
    }
}
return dp[m][n];

// Template 3: Top-Down with Memo
int[] memo = new int[n + 1];
Arrays.fill(memo, -1);

int solve(int i) {
    if (i < 0 || i == base) return baseValue;
    if (memo[i] != -1) return memo[i];
    memo[i] = f(solve(i-1), solve(i-2));
    return memo[i];
}

📖 Complete Beginner's Framework: How to Solve ANY DP Problem

🎯 The 5-Step DP Framework (Learn This Once, Use Forever)

Follow these 5 steps for EVERY DP problem. Master this framework and you'll never be stuck again!

📍 Step 1: Recognize if it's a DP Problem

Ask These Magic Questions:

Question If YES... Example
1. Does it ask for optimal value?
("maximum", "minimum", "longest", "shortest")		 → Likely DP! "Maximum profit", "Shortest path"
2. Does it ask to count ways?
("how many ways", "number of paths")		 → Likely DP! "Count ways to climb stairs"
3. Can I break it into smaller identical subproblems? → Likely DP! fib(n) = fib(n-1) + fib(n-2)
4. Do the same subproblems appear multiple times? → Definitely DP! Computing fib(3) appears twice in fib(5)

🔥 The Overlap Test (Most Important!)

Draw a small recursion tree for your problem. If you see the SAME function call appearing in MULTIPLE places → It's DP!

Example: Fibonacci(5) Recursion Tree

                                fib(5)
                               /      \
                          fib(4)      fib(3) ← appears here
                         /      \      /    \
                    fib(3)    fib(2) fib(2) fib(1)
                   /     \    /   \
              fib(2) fib(1) f(1) f(0)
              /   \
          f(1)  f(0)

❌ Without DP (Naive Recursion):

• fib(3) computed 2 times

• fib(2) computed 3 times

• fib(1) computed 5 times

Total function calls: 15 (for just fib(5)!)

Time Complexity: O(2n) - EXPONENTIAL! 🔥

✅ With DP (Memoization):

• fib(3) computed ONCE, then cached!

• fib(2) computed ONCE, then cached!

• fib(1) computed ONCE, then cached!

Total unique computations: 6 (only compute each once!)

Time Complexity: O(n) - LINEAR! ✨

💡 The Key Insight:

We have overlapping subproblems (same calculations repeated) AND optimal substructure (answer built from smaller answers). This is the PERFECT scenario for Dynamic Programming!

📍 Step 2: Define Your State (THE MOST CRITICAL STEP!)

⚠️ This is where most people get stuck! Master this and DP becomes easy.

The Question to Ask:

"What information do I need to uniquely identify a subproblem?"

Let's Practice with Real Examples:

Example 1: Climbing Stairs

Problem: How many ways to climb n stairs? (Can take 1 or 2 steps)

Think:

• What am I trying to find? → Ways to reach step n

• What changes as I solve subproblems? → The current step number

• What do I need to know to solve for any step? → Just which step I'm at!

State Definition: dp[i] = number of ways to reach step i

Why this works: Knowing step number is ENOUGH to solve that subproblem!

Example 2: Knapsack

Problem: Max value with weight limit W using items with weights and values?

Think:

• What am I trying to find? → Max value with certain items and capacity

• What changes? → Which items I've considered AND remaining capacity

• Need to track TWO things!

State Definition: dp[i][w] = max value using first i items with capacity w

Why 2D: Need BOTH pieces of info to solve any subproblem!

Example 3: Longest Common Subsequence

Problem: Find LCS of two strings s1 and s2?

Think:

• What am I trying to find? → LCS of two strings

• What changes? → Position in s1 AND position in s2

• Need to track position in BOTH strings!

State Definition: dp[i][j] = LCS length of first i chars of s1 and first j chars of s2

💡 Pro Tip: How Many Dimensions?

  • 1 variable changing? → 1D array (dp[i])
  • 2 variables changing? → 2D array (dp[i][j])
  • 3 variables? → 3D array (dp[i][j][k])

📍 Step 3: Find the Recurrence Relation (The Magic Formula)

How to express dp[i] using previous states?

🎯 The Thought Process:

  1. Think: "How can I reach state i?"
  2. List all options: What are ALL the ways to get here?
  3. Express mathematically: Combine those options

Complete Walkthrough: Climbing Stairs

Question: How can I reach step 5?

Think step by step:

1. I can take 1 step or 2 steps at a time

2. To reach step 5, where could I have come from?

→ Option A: From step 4 (take 1 step)

→ Option B: From step 3 (take 2 steps)

3. These are the ONLY two options!

4. Total ways = (ways to reach 4) + (ways to reach 3)

Recurrence Relation:

dp[i] = dp[i-1] + dp[i-2]

Visual Representation:

    Step 3 ──────── take 2 steps ────────→ Step 5
                                              ↑
                                              │
    Step 4 ──────── take 1 step  ────────────┘

Given Information:

dp[3] = 3

There are 3 ways to reach step 3

dp[4] = 5

There are 5 ways to reach step 4

Therefore:

dp[5] = dp[4] + dp[3]

dp[5] = 5 + 3

dp[5] = 8 ways ✓

💡 Why This Works:

To reach step 5, I MUST come from either step 3 or step 4 (no other options!). So the total ways = (all ways to reach step 3 where I take 2 steps) + (all ways to reach step 4 where I take 1 step). Since these are the ONLY two possibilities, we just add them!
