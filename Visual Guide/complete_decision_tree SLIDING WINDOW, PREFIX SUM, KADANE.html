🎯 Complete Algorithm Decision Guide

Comprehensive decision tree for array/string problems

🔍 First: Understand the Problem Type
Subarray vs Subsequence: Subarrays are CONTIGUOUS (elements must be adjacent), Subsequences can skip elements.
Single vs Multiple Arrays: Working with one array or comparing/merging multiple?
Modify vs Find: Can you modify the array or just find/count things?
1
Are elements CONTIGUOUS (subarray) or can they be NON-CONTIGUOUS (subsequence)?
This is the most important distinction
CONTIGUOUS (Subarray/Substring)
Continue to Step 2
Elements must be adjacent. Use Sliding Window, Prefix Sum, Kadane, Two Pointers, etc.
Examples: "longest substring", "subarray with sum k", "max sum subarray"
NON-CONTIGUOUS (Subsequence)
✅ Dynamic Programming or Greedy
Elements can be picked from anywhere. Usually DP or Greedy algorithms.
Examples: "longest increasing subsequence", "longest common subsequence", "subset sum"
2
Do you need RANGE SUM queries or count ALL subarrays with a property?
Multiple queries on sums, or counting subarrays
YES ✓
✅ Prefix Sum (+ HashMap)
Precompute cumulative sums for O(1) range queries.

Prefix Sum alone: Range sum queries
Prefix Sum + HashMap: Count subarrays with sum = k, divisible by k, etc.
Examples: "sum from i to j", "count subarrays with sum = k", "subarray sum divisible by k"
NO →
Continue to Step 3
Not about range sums or counting all subarrays
3
Looking for MAXIMUM or MINIMUM subarray sum/product?
Finding the optimal subarray, possibly with special constraints
YES ✓
✅ Kadane's Algorithm (or variant)
Track max/min ending at each position.

Standard: Max subarray sum
Circular: Max of (normal Kadane OR total - min subarray)
Product: Track both max and min (negatives flip signs)
Examples: "max sum subarray", "max product subarray", "max sum circular subarray"
NO →
Continue to Step 4
Not finding optimal sum/product
4
Contiguous elements with CONSTRAINTS or FREQUENCY tracking?
Longest/shortest substring/subarray satisfying some condition, or tracking character/element frequency
YES ✓
✅ Sliding Window (+ HashMap/Set)
Maintain a window that expands/shrinks to satisfy constraints.

When to use: Longest/shortest with condition, frequency problems, anagrams/permutations in string
🔍 What type of window?
Fixed Size
Window always has exactly k elements. Slide one at a time.
"Max sum of k consecutive elements"
Variable Size
Window grows/shrinks based on validity. Often uses HashMap/Set.
"Longest substring with ≤ k distinct chars", "min window substring"
Examples: "longest substring without repeating chars", "permutation in string", "find all anagrams"
NO →
Continue to Step 5
Not about constrained windows
5
Working with SORTED array(s) to find pairs/triplets or merge/compare?
Array is sorted or you're merging/comparing sorted arrays
YES ✓
✅ Two Pointers (or Multiple Pointers)
Use pointers moving from opposite ends or in parallel.

Opposite ends: Two sum, three sum, container with most water
Parallel: Merge sorted arrays, intersection of arrays
In-place modification: Remove duplicates, move zeros
Examples: "two sum (sorted)", "3sum", "merge sorted arrays", "remove duplicates", "intersection of arrays"
NO →
Continue to Step 6
Not using sorted property
6
Track MIN/MAX efficiently in sliding window?
Need min/max for each window position as you slide through array
YES ✓
✅ Monotonic Deque
Maintain elements in monotonically increasing/decreasing order.

When to use: Sliding window max/min in O(n) time
Examples: "max in each window of size k", "min in sliding window"
NO →
Continue to Step 7
Not tracking window min/max
7
Finding NEXT/PREVIOUS greater/smaller element?
Problems involving monotonic increasing/decreasing patterns
YES ✓
✅ Monotonic Stack
Maintain stack in monotonic order to find next/previous greater/smaller.

When to use: Next greater element, largest rectangle, stock span, valid parentheses variations
Examples: "next greater element", "largest rectangle in histogram", "trapping rain water", "daily temperatures"
NO →
Continue to Step 8
Not about next/previous element relationships
8
Can you use BINARY SEARCH on answer or sorted data?
Search space is monotonic or you're searching in sorted array
YES ✓
✅ Binary Search
Search in sorted space or binary search on answer.

Standard: Search in sorted array
On answer: "Minimize maximum" or "Maximize minimum" problems
Examples: "find in rotated sorted array", "capacity to ship packages", "koko eating bananas", "median of two sorted arrays"
NO →
Other Techniques
Consider:
• Dynamic Programming (overlapping subproblems)
• Greedy (local optimal leads to global)
• Divide & Conquer
• Graph algorithms (BFS/DFS)
• Backtracking (generate all possibilities)
