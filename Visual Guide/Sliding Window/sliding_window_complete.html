⚖️ Fixed Window vs Variable Window - Side by Side

🔒 Fixed Window

Visual Pattern:

Window size = k = 3
[■ ■ ■] _ _ _ → Step 1
_ [■ ■ ■] _ _ → Step 2
_ _ [■ ■ ■] _ → Step 3
_ _ _ [■ ■ ■] → Step 4

Characteristics:

  • Window size is constant
  • Always move both pointers together
  • Remove left, add right
  • Process every window

Code Pattern:

for (i = k to n):
  remove arr[i-k]
  add arr[i]
  update result

🔄 Variable Window

Visual Pattern:

Window grows & shrinks
[■] _ _ _ _ _ → Expand
[■ ■ ■] _ _ _ → Expand
_ _ [■] _ _ _ → Shrink
_ _ [■ ■ ■ ■] → Expand

Characteristics:

  • Window size changes
  • Pointers move independently
  • Expand when valid
  • Shrink when invalid

Code Pattern:

for (right = 0 to n):
  add arr[right]
  while (invalid):
    remove arr[left]
    left++
  update result

🔵 Fixed Window Template

Use When: Window size is given (k)

Examples: "maximum sum of k elements", "average of k numbers"

Time Complexity: O(n)

Space Complexity: O(1)

// Fixed Window Template - Generic
// Use this when: window size (k) is given in the problem
// Time Complexity: O(n) - each element processed exactly once
// Space Complexity: O(1) - only using a few variables

public ReturnType fixedWindow(int[] arr, int k) {
    // ========== STEP 1: Initialize Variables ==========
    // windowSum: tracks the sum of current window
    int windowSum = 0;
    
    // result: stores the best answer found so far
    // Use Integer.MIN_VALUE for maximum problems
    // Use Integer.MAX_VALUE for minimum problems
    int result = 0;
    
    // ========== STEP 2: Build First Window ==========
    // Calculate sum/value for the first k elements
    // This is our initial window: arr[0] to arr[k-1]
    for (int i = 0; i < k; i++) {
        windowSum += arr[i];  // Add each element to window
    }
    
    // Store first window's value as initial result
    result = windowSum;
    
    // ========== STEP 3: Slide the Window ==========
    // Start from index k (right after first window)
    // Process remaining elements one by one
    for (int i = k; i < arr.length; i++) {
        // REMOVE: Subtract element leaving the window (leftmost)
        // Window was [i-k, i-k+1, ..., i-1]
        // We're removing arr[i-k]
        windowSum -= arr[i - k];
        
        // ADD: Add new element entering the window (rightmost)
        // New window becomes [i-k+1, i-k+2, ..., i]
        // We're adding arr[i]
        windowSum += arr[i];
        
        // UPDATE: Compare current window with best result
        // Use Math.max for maximum problems
        // Use Math.min for minimum problems
        result = Math.max(result, windowSum);
    }
    
    // Return the best value found across all windows
    return result;
}

/*
 * HOW THIS WORKS:
 * - Window always contains EXACTLY k elements
 * - We slide by: removing leftmost, adding rightmost
 * - Each iteration processes one new element
 * - Total iterations: n - k + 1 windows
 * 
 * EXAMPLE: arr = [1,3,2,5], k = 2
 * Initial window: [1,3] sum=4
 * Slide 1: Remove 1, Add 2 → [3,2] sum=5
 * Slide 2: Remove 3, Add 5 → [2,5] sum=7 (max)
 */

Example: Maximum Sum Subarray of Size K

/**
 * Problem: Find maximum sum of any contiguous subarray of size k
 * Example: arr = [2, 1, 5, 1, 3, 2], k = 3
 * Answer: 9 (subarray [5, 1, 3])
 * 
 * Approach: Fixed Sliding Window
 * 1. Calculate sum of first k elements
 * 2. Slide window: subtract left, add right
 * 3. Track maximum sum seen
 */

public int maxSumSubarray(int[] arr, int k) {
    // Edge case: if array smaller than k, no valid window exists
    if (arr.length < k) return -1;
    
    // maxSum: stores the maximum sum found so far
    int maxSum = 0;
    
    // windowSum: current sum of k elements in window
    int windowSum = 0;
    
    // ===== Build first window: [0, k-1] =====
    for (int i = 0; i < k; i++) {
        windowSum += arr[i];  // Sum first k elements
    }
    // First window's sum is our initial maximum
    maxSum = windowSum;
    
    // ===== Slide the window through remaining array =====
    // Start at index k, process until end of array
    for (int i = k; i < arr.length; i++) {
        // Slide window one position right:
        // OLD window: [i-k, i-k+1, ..., i-1]
        // NEW window: [i-k+1, i-k+2, ..., i]
        // 
        // Efficient calculation: instead of recalculating entire sum,
        // just remove arr[i-k] and add arr[i]
        windowSum = windowSum - arr[i - k] + arr[i];
        
        // Update maximum if current window has larger sum
        maxSum = Math.max(maxSum, windowSum);
    }
    
    // Return the maximum sum across all windows
    return maxSum;
}

/* 
 * TRACE EXAMPLE: arr = [2, 1, 5, 1, 3, 2], k = 3
 * 
 * Initial: window [2,1,5], sum=8, max=8
 * i=3: Remove 2, Add 1 → [1,5,1], sum=7, max=8
 * i=4: Remove 1, Add 3 → [5,1,3], sum=9, max=9 ✓
 * i=5: Remove 5, Add 2 → [1,3,2], sum=6, max=9
 * 
 * Answer: 9
 */

Example: Average of Subarrays of Size K (LeetCode 643)

/**
 * Problem: Find Averages of All Contiguous Subarrays of Size K
 * Example: arr = [1, 3, 2, 6, -1, 4, 1, 8, 2], k = 5
 * Answer: [2.2, 2.8, 2.4, 3.6, 2.8]
 * 
 * Approach: Fixed Sliding Window
 * - Calculate average of each window of size k
 * - Return array of all averages
 * 
 * Number of windows = arr.length - k + 1
 * For arr.length = 9, k = 5 → 9 - 5 + 1 = 5 windows
 */

public double[] findAverages(int[] arr, int k) {
    // === Initialize result array ===
    // We'll have (arr.length - k + 1) windows total
    // Each window produces one average value
    double[] result = new double[arr.length - k + 1];
    
    // windowSum: running sum of current k elements
    int windowSum = 0;
    
    // windowStart: tracks which window we're on (for result array index)
    // Also represents the left boundary of the window
    int windowStart = 0;
    
    // === Process each element ===
    // windowEnd expands from 0 to arr.length - 1
    for (int windowEnd = 0; windowEnd < arr.length; windowEnd++) {
        // === ADD: Include current element in window ===
        // Add element at windowEnd to our running sum
        windowSum += arr[windowEnd];
        
        // === CHECK: Do we have a complete window? ===
        // Window is complete when we have k elements
        // First window completes when windowEnd = k - 1
        // After that, every iteration completes a new window
        if (windowEnd >= k - 1) {
            // === CALCULATE: Average for current window ===
            // Window contains arr[windowStart] to arr[windowEnd]
            // That's exactly k elements
            // Cast to double to get decimal average
            result[windowStart] = (double) windowSum / k;
            
            // === SLIDE: Remove leftmost element for next window ===
            // Subtract the element at windowStart from sum
            // This element will be outside the next window
            windowSum -= arr[windowStart];
            
            // === ADVANCE: Move to next window ===
            // Increment windowStart to prepare for next window
            // Also serves as index for next result value
            windowStart++;
        }
        
        // If windowEnd < k-1, we're still building first window
        // Loop continues to add more elements
        // Once windowEnd = k-1, we start producing results
    }
    
    // Return array of all window averages
    return result;
}

/*
 * DETAILED TRACE: arr = [1, 3, 2, 6, -1], k = 3
 * 
 * windowEnd=0: sum=1, windowEnd < 2, skip if-block (building window)
 * windowEnd=1: sum=4, windowEnd < 2, skip if-block (building window)
 * 
 * windowEnd=2: sum=6, windowEnd ≥ 2 ✓
 *              Window [0,2] = [1,3,2]
 *              result[0] = 6/3 = 2.0
 *              sum -= arr[0] → sum=5
 *              windowStart++ → windowStart=1
 * 
 * windowEnd=3: sum=11, windowEnd ≥ 2 ✓
 *              Window [1,3] = [3,2,6]
 *              result[1] = 11/3 = 3.67
 *              sum -= arr[1] → sum=8
 *              windowStart++ → windowStart=2
 * 
 * windowEnd=4: sum=7, windowEnd ≥ 2 ✓
 *              Window [2,4] = [2,6,-1]
 *              result[2] = 7/3 = 2.33
 *              sum -= arr[2] → sum=5
 *              windowStart++ → windowStart=3
 * 
 * Final result = [2.0, 3.67, 2.33]
 * 
 * WHY THIS PATTERN WORKS:
 * 1. We only calculate one average per iteration (after first k elements)
 * 2. windowEnd controls when we're adding elements
 * 3. windowStart controls:
 *    - Which element to remove (arr[windowStart])
 *    - Where to store result (result[windowStart])
 * 4. The if-check (windowEnd >= k-1) ensures we have k elements
 * 
 * ALTERNATIVE STYLE (Two-Pointer):
 * - Build first window with separate loop
 * - Then use for loop from k to n
 * Both styles work - this one is more compact
 */

🔄 Variable Window Template

Use When: Window size changes dynamically

Examples: "longest substring with condition", "smallest subarray with sum ≥ S"

Time Complexity: O(n)

Space Complexity: O(1) to O(k) depending on tracking needs

// Variable Window Template - Generic
// Use this when: window size is NOT fixed, need to find longest/shortest
// Common patterns: "at most k distinct", "longest substring", "shortest subarray"
// Time Complexity: O(n) - each element added once, removed once
// Space Complexity: O(1) to O(k) depending on tracking needs

public ReturnType variableWindow(int[] arr) {
    // ========== STEP 1: Initialize Pointers & Variables ==========
    // windowStart (left pointer): marks beginning of current window
    int windowStart = 0;
    
    // result: stores best answer found
    // For LONGEST: initialize to 0, use Math.max
    // For SHORTEST: initialize to Integer.MAX_VALUE, use Math.min
    int result = 0;
    
    // windowState: tracks window's current state
    // Could be: sum, count, HashMap of frequencies, Set of elements, etc.
    // Depends on problem requirements
    int windowState = 0;
    
    // ========== STEP 2: Expand Window with Right Pointer ==========
    // windowEnd (right pointer): expands window by moving right
    // This loop ALWAYS moves forward - we process each element once
    for (int windowEnd = 0; windowEnd < arr.length; windowEnd++) {
        
        // === ADD: Include new element in window ===
        // Window is now [windowStart, windowEnd]
        // Update windowState to reflect new element
        windowState += arr[windowEnd];
        
        // === SHRINK: Contract window if condition violated ===
        // Keep removing from left while window is INVALID
        // This ensures window is always valid after this loop
        while (/* condition is violated */) {
            // Remove element from left side of window
            windowState -= arr[windowStart];
            
            // Move left pointer forward (shrink window)
            windowStart++;
        }
        
        // === UPDATE: Record result for current valid window ===
        // At this point, window [windowStart, windowEnd] is VALID
        // Window length = windowEnd - windowStart + 1
        result = Math.max(result, windowEnd - windowStart + 1);
    }
    
    // Return best result found across all valid windows
    return result;
}

/*
 * KEY INSIGHTS:
 * 
 * 1. TWO POINTERS MOVE INDEPENDENTLY:
 *    - windowEnd ALWAYS moves right (outer loop)
 *    - windowStart moves right ONLY when needed (inner while)
 * 
 * 2. WINDOW EXPANSION & CONTRACTION:
 *    - Expand: Add arr[windowEnd] and move windowEnd++
 *    - Shrink: Remove arr[windowStart] and move windowStart++
 * 
 * 3. TIME COMPLEXITY IS O(N):
 *    - Each element enters window once (windowEnd loop)
 *    - Each element leaves window at most once (windowStart++)
 *    - Total operations: 2n → O(n)
 * 
 * 4. UPDATE RESULT WHEN WINDOW IS VALID:
 *    - After while loop, window is guaranteed valid
 *    - For "longest": update with current window size
 *    - For "shortest": update when condition first satisfied
 * 
 * 5. COMMON CONDITIONS FOR WHILE LOOP:
 *    - Sum too large: (windowSum > target)
 *    - Too many distinct: (map.size() > k)
 *    - Duplicate found: (set.contains(arr[windowEnd]))
 */

Example: Longest Substring with At Most K Distinct Characters (LeetCode 340)

/**
 * Problem: Find length of longest substring with at most k distinct characters
 * Example: s = "eceba", k = 2
 * Answer: 3 (substring "ece" has 2 distinct chars: 'e' and 'c')
 * 
 * Approach: Variable Window + HashMap
 * - Expand window by adding characters
 * - Shrink when distinct count > k
 * - Track maximum length seen
 */

public int lengthOfLongestSubstringKDistinct(String s, int k) {
    // Edge case: if k=0, no valid substring exists
    if (k == 0) return 0;
    
    // HashMap: tracks frequency of each character in current window
    // Key = character, Value = count of that character
    Map<Character, Integer> charFrequency = new HashMap<>();
    
    // windowStart: left boundary of current window
    int windowStart = 0;
    
    // maxLength: stores the longest valid window found
    int maxLength = 0;
    
    // ===== Expand window with right pointer =====
    for (int windowEnd = 0; windowEnd < s.length(); windowEnd++) {
        // === ADD character to window ===
        char rightChar = s.charAt(windowEnd);
        
        // Increment frequency of this character
        // getOrDefault returns 0 if character not in map yet
        charFrequency.put(rightChar, charFrequency.getOrDefault(rightChar, 0) + 1);
        
        // === SHRINK window if too many distinct characters ===
        // Window is INVALID when we have MORE than k distinct chars
        // Keep shrinking until we're back to at most k distinct
        while (charFrequency.size() > k) {
            // Remove leftmost character from window
            char leftChar = s.charAt(windowStart);
            
            // Decrement its frequency
            charFrequency.put(leftChar, charFrequency.get(leftChar) - 1);
            
            // If frequency reaches 0, remove character from map entirely
            // This reduces distinct count
            if (charFrequency.get(leftChar) == 0) {
                charFrequency.remove(leftChar);
            }
            
            // Move left pointer right (shrink window)
            windowStart++;
        }
        
        // === UPDATE maximum length ===
        // After while loop, window [windowStart, windowEnd] is VALID
        // Window has at most k distinct characters
        // Check if this is the longest valid window we've seen
        maxLength = Math.max(maxLength, windowEnd - windowStart + 1);
    }
    
    // Return the maximum length found
    return maxLength;
}

/*
 * TRACE EXAMPLE: s = "eceba", k = 2
 * 
 * windowEnd=0: Add 'e' → {e:1}, size=1 ≤ 2 ✓, len=1, max=1
 * windowEnd=1: Add 'c' → {e:1,c:1}, size=2 ≤ 2 ✓, len=2, max=2
 * windowEnd=2: Add 'e' → {e:2,c:1}, size=2 ≤ 2 ✓, len=3, max=3 ← BEST
 * windowEnd=3: Add 'b' → {e:2,c:1,b:1}, size=3 > 2 ✗ INVALID!
 *              Shrink: Remove 'e' → {e:1,c:1,b:1}, size=3 > 2 ✗
 *              Shrink: Remove 'c' → {e:1,b:1}, size=2 ≤ 2 ✓
 *              Window = "eb", len=2
 * windowEnd=4: Add 'a' → {e:1,b:1,a:1}, size=3 > 2 ✗
 *              Shrink until valid...
 * 
 * Answer: 3 (window "ece")
 * 
 * WHY IT WORKS:
 * - We try to maximize window size by expanding right
 * - When invalid (too many distinct), we shrink from left
 * - HashMap.size() gives us distinct character count in O(1)
 * - Each character added once, removed at most once → O(n)
 */

Example: Minimum Size Subarray Sum (LeetCode 209)

/**
 * Problem: Find minimal length of subarray whose sum ≥ target
 * Example: target = 7, nums = [2,3,1,2,4,3]
 * Answer: 2 (subarray [4,3] has sum=7 with length 2)
 * 
 * Approach: Variable Window
 * - Expand window until sum ≥ target (satisfies condition)
 * - Shrink window to minimize length WHILE keeping sum ≥ target
 * - Key insight: we want SHORTEST window, so shrink greedily
 * 
 * Why "at least" → shortest:
 * - We can be "greedy" about shrinking
 * - Once sum ≥ target, we try to make window smaller
 * - Unlike "at most" where we expand greedily
 */

public int minSubArrayLen(int target, int[] nums) {
    // windowStart: left boundary of current window
    int windowStart = 0;
    
    // minLength: stores shortest valid window found
    // Initialize to MAX_VALUE (infinity) since we want minimum
    // If no valid window exists, we'll return 0 at the end
    int minLength = Integer.MAX_VALUE;
    
    // windowSum: current sum of elements in window [windowStart, windowEnd]
    int windowSum = 0;
    
    // ===== Expand window with right pointer =====
    for (int windowEnd = 0; windowEnd < nums.length; windowEnd++) {
        // === ADD: Include new element at windowEnd ===
        // Expand window to the right by adding nums[windowEnd]
        windowSum += nums[windowEnd];
        
        // === SHRINK: Make window smaller while still valid ===
        // Window is VALID when sum ≥ target
        // Keep shrinking while we can STILL satisfy the condition
        // This finds the MINIMUM length for current windowEnd
        //
        // Why while and not if?
        // - We might be able to remove multiple elements from left
        // - We want to shrink as much as possible
        while (windowSum >= target) {
            // === UPDATE: Record current window length ===
            // Current window [windowStart, windowEnd] is VALID
            // Check if it's shorter than previous minimum
            minLength = Math.min(minLength, windowEnd - windowStart + 1);
            
            // === SHRINK: Remove leftmost element ===
            // Try to make window even smaller
            // Subtract element leaving the window
            windowSum -= nums[windowStart];
            
            // Move left pointer right (reduce window size)
            windowStart++;
            
            // Loop continues: check if window STILL valid after shrinking
            // If yes, we can shrink more! If no, exit while loop
        }
        
        // After while loop:
        // - Either window is now INVALID (sum < target)
        // - Or we've found the minimum for this windowEnd
        // Continue to next windowEnd to explore more possibilities
    }
    
    // === Return result ===
    // If minLength is still MAX_VALUE, no valid window was found → return 0
    // Otherwise, return the minimum length we found
    return minLength == Integer.MAX_VALUE ? 0 : minLength;
}

/*
 * DETAILED TRACE: target = 7, nums = [2,3,1,2,4,3]
 * 
 * windowEnd=0: sum=2 < 7 ✗ (skip while, window too small)
 * windowEnd=1: sum=5 < 7 ✗ (skip while)
 * windowEnd=2: sum=6 < 7 ✗ (skip while)
 * windowEnd=3: sum=8 ≥ 7 ✓ VALID!
 *              Window [0,3] = [2,3,1,2], len=4, min=4
 *              Shrink: sum-=2 → sum=6 < 7 ✗ (exit while)
 * windowEnd=4: sum=10 ≥ 7 ✓ VALID!
 *              Window [1,4] = [3,1,2,4], len=4, min=4
 *              Shrink: sum-=3 → sum=7 ≥ 7 ✓ still valid!
 *              Window [2,4] = [1,2,4], len=3, min=3 ← BETTER
 *              Shrink: sum-=1 → sum=6 < 7 ✗ (exit while)
 * windowEnd=5: sum=9 ≥ 7 ✓ VALID!
 *              Window [3,5] = [2,4,3], len=3, min=3
 *              Shrink: sum-=2 → sum=7 ≥ 7 ✓ still valid!
 *              Window [4,5] = [4,3], len=2, min=2 ← BEST!
 *              Shrink: sum-=4 → sum=3 < 7 ✗ (exit while)
 * 
 * Final Answer: 2
 * 
 * KEY INSIGHTS:
 * 1. "At least" (≥) → Find SHORTEST window
 * 2. Shrink WHILE condition is still satisfied
 * 3. Update result INSIDE while loop (before shrinking)
 * 4. Each element added once, removed once → O(n) time
 * 
 * CONTRAST WITH "AT MOST":
 * - "At most k": Expand greedily, shrink when violated → LONGEST
 * - "At least k": Expand to satisfy, shrink to minimize → SHORTEST
 */

Example: Longest Substring Without Repeating Characters (LeetCode 3)

/**
 * Problem: Find length of longest substring without repeating characters
 * Example: s = "abcabcbb"
 * Answer: 3 (substring "abc")
 * 
 * Approach: Variable Window + HashMap
 * - HashMap stores: character → its most recent index
 * - When we find a duplicate, jump windowStart past the previous occurrence
 * - This is an OPTIMIZED sliding window (can skip elements)
 * 
 * Key Insight:
 * - If char at index i was seen at index j
 * - We can't include both in same window
 * - Jump windowStart to j+1 to exclude the duplicate
 */

public int lengthOfLongestSubstring(String s) {
    // charIndex: HashMap to store last seen index of each character
    // Key = character, Value = most recent index where we saw it
    // This lets us quickly find duplicates and know where to jump
    Map<Character, Integer> charIndex = new HashMap<>();
    
    // windowStart: left boundary of current window
    // Marks the start of our current substring without duplicates
    int windowStart = 0;
    
    // maxLength: longest valid substring length found so far
    int maxLength = 0;
    
    // ===== Expand window with right pointer =====
    // windowEnd represents the character we're currently processing
    for (int windowEnd = 0; windowEnd < s.length(); windowEnd++) {
        char rightChar = s.charAt(windowEnd);
        
        // === CHECK: Is this character a duplicate? ===
        // If rightChar exists in map AND its index is within current window
        // then we have a duplicate in our window
        if (charIndex.containsKey(rightChar)) {
            // === SHRINK: Jump windowStart to exclude duplicate ===
            // Move windowStart to position AFTER the previous occurrence
            // 
            // Why Math.max?
            // - charIndex.get(rightChar) + 1: position after duplicate
            // - windowStart: current left boundary
            // - We take max because windowStart should NEVER move backward
            // 
            // Example: s = "abba"
            //   At index 3 (second 'a'), charIndex has {a:0, b:2}
            //   charIndex.get('a') + 1 = 1, but windowStart is already 2
            //   We keep windowStart = 2 (don't move backward)
            windowStart = Math.max(windowStart, charIndex.get(rightChar) + 1);
        }
        
        // === UPDATE: Store/update current character's index ===
        // Always update the map with the current index
        // This keeps track of most recent occurrence
        charIndex.put(rightChar, windowEnd);
        
        // === UPDATE: Check if current window is longest ===
        // Window [windowStart, windowEnd] has NO duplicates now
        // Calculate its length and compare with maxLength
        maxLength = Math.max(maxLength, windowEnd - windowStart + 1);
    }
    
    // Return the maximum length found
    return maxLength;
}

/*
 * DETAILED TRACE: s = "abcabcbb"
 * 
 * i=0, char='a': map={a:0}, window=[0,0]="a", len=1, max=1
 * i=1, char='b': map={a:0,b:1}, window=[0,1]="ab", len=2, max=2
 * i=2, char='c': map={a:0,b:1,c:2}, window=[0,2]="abc", len=3, max=3
 * 
 * i=3, char='a': DUPLICATE! (was at index 0)
 *                windowStart = max(0, 0+1) = 1
 *                map={a:3,b:1,c:2}, window=[1,3]="bca", len=3, max=3
 * 
 * i=4, char='b': DUPLICATE! (was at index 1, but windowStart=1)
 *                windowStart = max(1, 1+1) = 2
 *                map={a:3,b:4,c:2}, window=[2,4]="cab", len=3, max=3
 * 
 * i=5, char='c': DUPLICATE! (was at index 2, but windowStart=2)
 *                windowStart = max(2, 2+1) = 3
 *                map={a:3,b:4,c:5}, window=[3,5]="abc", len=3, max=3
 * 
 * i=6, char='b': DUPLICATE! (was at index 4)
 *                windowStart = max(3, 4+1) = 5
 *                map={a:3,b:6,c:5}, window=[5,6]="cb", len=2, max=3
 * 
 * i=7, char='b': DUPLICATE! (was at index 6)
 *                windowStart = max(5, 6+1) = 7
 *                map={a:3,b:7,c:5}, window=[7,7]="b", len=1, max=3
 * 
 * Final Answer: 3
 * 
 * WHY THIS WORKS:
 * 1. HashMap gives O(1) lookup to check for duplicates
 * 2. We store INDEX, not just presence, so we know where to jump
 * 3. Math.max prevents windowStart from moving backward
 * 4. Each character processed exactly once → O(n) time
 * 
 * ALTERNATIVE APPROACH (Standard Shrink):
 * while (hasDuplicate) {
 *     remove leftmost character
 *     windowStart++
 * }
 * This works but is less efficient - could be O(n²) worst case
 * Our approach jumps directly → always O(n)
 */

🔄 Hybrid Patterns - Visual Guide with LeetCode

Combining Sliding Window with other data structures creates powerful patterns for complex problems.

🎯 How to Recognize Hybrid Patterns

The Recognition Framework

Hybrid patterns are needed when basic sliding window alone isn't enough. Look for these signals in the problem statement:

🗂️ Pattern 1: Sliding Window + Heap

🔍 Recognition Keywords:

  • ✓ "maximum in window" or "minimum in window"
  • ✓ "kth largest/smallest" in sliding window
  • ✓ "median" in sliding window
  • ✓ "top k elements" as window moves
  • ✓ Need to track ordering within the window

💡 Why Heap?

Basic sliding window can track sum/count, but can't efficiently find max/min when elements leave the window. Heap gives us O(log k) insertion/removal while maintaining the maximum or minimum at the top.

📝 Example Problem Indicators:

"Find the maximum value in each window of size k"

→ Need heap because naive approach would be O(nk) to find max in each window

"Find the median of each window"

→ Need two heaps (max heap for left half, min heap for right half)

Common LeetCode Problems: LC 239 (Sliding Window Maximum), LC 480 (Sliding Window Median)

🗺️ Pattern 2: Sliding Window + HashMap

🔍 Recognition Keywords:

  • ✓ "distinct characters/elements" in window
  • ✓ "frequency" or "count" of characters
  • ✓ "anagram" or "permutation" in string
  • ✓ "at most k distinct" characters
  • ✓ Need to track occurrences as window moves
  • ✓ "character set" or "pattern matching"

💡 Why HashMap?

Need to track what's in the window and how many times each element appears. HashMap gives O(1) lookup, insert, and delete to track frequencies as we slide.

📝 Example Problem Indicators:

"Find longest substring with at most 2 distinct characters"

→ Need HashMap to count distinct characters and their frequencies

"Find all anagrams of a pattern in a string"

→ Need HashMap to match character frequencies between pattern and window

"Minimum window that contains all characters of pattern"

→ Need HashMap to track which pattern characters are satisfied

Common LeetCode Problems: LC 76 (Min Window Substring), LC 438 (Find All Anagrams), LC 567 (Permutation in String)

📊 Pattern 3: Sliding Window + Monotonic Deque

🔍 Recognition Keywords:

  • ✓ "maximum in sliding window" (optimized version)
  • ✓ "minimum in sliding window" (optimized version)
  • ✓ Need O(n) time not O(n log k) like heap
  • ✓ "next greater/smaller element" pattern in window
  • ✓ Want to maintain order but remove from both ends

💡 Why Deque?

Deque (double-ended queue) lets us remove from both front and back in O(1). We maintain a monotonic deque (decreasing for max, increasing for min) to always have the best candidate at the front.

🎯 Heap vs Deque Decision:

Criteria Use Heap Use Deque
Time Complexity O(n log k) O(n) ⚡
Difficulty Easier to code Trickier implementation
Use Case Kth element, Median Max/Min in window only

📝 Example Problem Indicators:

"Find maximum in each window (optimal solution)"

→ Deque gives O(n) vs Heap's O(n log k)

"Shortest subarray with sum at least K"

→ Monotonic deque to maintain increasing prefix sums

Common LeetCode Problems: LC 239 (Sliding Window Maximum - optimal), LC 862 (Shortest Subarray with Sum at Least K)

🎯 The "AT MOST" vs "AT LEAST" vs "EXACTLY" Pattern

🔵 "AT MOST"

Common Phrasings:

  • "at most k distinct"
  • "with at most k"
  • "≤ k distinct"
  • "no more than k"

Goal: Find LONGEST valid window
Strategy: Expand greedily, shrink when violated
Example:
"Longest substring with at most 2 distinct chars"
→ Expand until distinct > 2, then shrink
🟠 "AT LEAST"

Common Phrasings:

  • "at least k sum"
  • "sum ≥ target"
  • "minimum k elements"
  • "no less than k"
Pattern: Variable Window
Goal: Find SHORTEST valid window
Strategy: Expand to satisfy, shrink while still valid
Example:
"Shortest subarray with sum ≥ target"
→ Shrink while sum still ≥ target
🟣 "EXACTLY"

Common Phrasings:

  • "exactly k distinct"
  • "window of size k"
  • "sum equals target"
  • "= k elements"
Pattern: Trick! Use subtraction
Formula: exactly(k) = atMost(k) - atMost(k-1)
OR: Fixed window if size given
Example:
"Subarrays with exactly k distinct"
→ Count(atMost k) - Count(atMost k-1)

📏 "LONGEST" vs "SHORTEST" vs "MAXIMUM" vs "MINIMUM"

Keyword What It Means Window Type Example
LONGEST Find maximum length/size that satisfies condition Variable "Longest substring with at most k distinct chars"
SHORTEST Find minimum length/size that satisfies condition Variable "Shortest subarray with sum ≥ S"
MAXIMUM Find max VALUE within windows (not window size) Fixed or Variable "Maximum sum of subarray size k" (value not size)
MINIMUM Find min VALUE within windows (not window size) Fixed or Variable "Minimum in each window of size k" (value not size)
🎯 Key Distinction:
  • LONGEST/SHORTEST → Looking for window SIZE (result is a length)
  • MAXIMUM/MINIMUM → Looking for a VALUE in the window (result is sum/element/etc)

// LeetCode 239: Sliding Window Maximum
// Using MaxHeap to track maximum in window
public int[] maxSlidingWindow(int[] nums, int k) {
    // Result array to store maximums
    int[] result = new int[nums.length - k + 1];
    
    // MaxHeap: stores [value, index] pairs
    // Prioritizes by value (descending), so peek gives maximum
    PriorityQueue<int[]> maxHeap = new PriorityQueue<>(
        (a, b) -> b[0] - a[0]  // Compare by value (descending)
    );
    
    // Process each element
    for (int i = 0; i < nums.length; i++) {
        // Add current element with its index to heap
        maxHeap.offer(new int[]{nums[i], i});
        
        // Remove elements outside current window from heap top
        // Heap may contain old elements, but top must be in window
        while (!maxHeap.isEmpty() && maxHeap.peek()[1] <= i - k) {
            maxHeap.poll();  // Remove element outside window
        }
        
        // Once we have a full window, record the maximum
        if (i >= k - 1) {
            result[i - k + 1] = maxHeap.peek()[0];  // Top of maxHeap is maximum
        }
    }
    
    return result;
}

// Time Complexity: O(n log k) - Each insert/remove is O(log k)
// Space Complexity: O(k) - Heap stores at most k elements

// LeetCode 3: Longest Substring Without Repeating Characters
// Using HashMap to track character frequencies
public int lengthOfLongestSubstring(String s) {
    // HashMap to store character frequencies in current window
    Map<Character, Integer> charCount = new HashMap<>();
    
    int left = 0;           // Left pointer of window
    int maxLength = 0;      // Maximum length found so far
    
    // Right pointer expands the window
    for (int right = 0; right < s.length(); right++) {
        char rightChar = s.charAt(right);
        
        // Add right character to window (increment its frequency)
        charCount.put(rightChar, charCount.getOrDefault(rightChar, 0) + 1);
        
        // If we have a duplicate (frequency > 1), shrink window from left
        while (charCount.get(rightChar) > 1) {
            char leftChar = s.charAt(left);
            
            // Remove left character from window
            charCount.put(leftChar, charCount.get(leftChar) - 1);
            
            // If frequency becomes 0, remove from map to save space
            if (charCount.get(leftChar) == 0) {
                charCount.remove(leftChar);
            }
            
            left++;  // Move left pointer right (shrink window)
        }
        
        // Update maximum length with current valid window size
        maxLength = Math.max(maxLength, right - left + 1);
    }
    
    return maxLength;
}

// Time Complexity: O(n) - Each character visited at most twice (by left and right)
// Space Complexity: O(min(n, m)) - m is charset size (at most 26 for lowercase)

// LeetCode 239: Sliding Window Maximum (Optimal with Deque)
// Using Monotonic Deque - O(n) solution!
public int[] maxSlidingWindow(int[] nums, int k) {
    // Result array
    int[] result = new int[nums.length - k + 1];
    
    // Deque stores INDICES (not values!) in decreasing order of VALUES
    // Front of deque = index of maximum element in current window
    Deque<Integer> deque = new LinkedList<>();
    
    for (int i = 0; i < nums.length; i++) {
        // 1. Remove indices outside current window from front
        //    If front index is too far left (outside window), remove it
        while (!deque.isEmpty() && deque.peekFirst() <= i - k) {
            deque.pollFirst();
        }
        
        // 2. Remove indices from back whose values are smaller than current
        //    They can never be maximum, so we don't need them
        while (!deque.isEmpty() && nums[deque.peekLast()] <= nums[i]) {
            deque.pollLast();
        }
        
        // 3. Add current index to back of deque
        deque.offerLast(i);
        
        // 4. Once we have a full window, record the maximum
        //    Front of deque always has index of maximum value
        if (i >= k - 1) {
            result[i - k + 1] = nums[deque.peekFirst()];
        }
    }
    
    return result;
}

// Time Complexity: O(n) - Each element added/removed from deque at most once!
// Space Complexity: O(k) - Deque stores at most k elements
// This is FASTER than the heap solution O(n log k)!

Need Use Time
Track frequencies/counts HashMap O(n)
Find max/min (don't need O(1)) Heap O(n log k)
Find max/min (need O(1)) Monotonic Deque O(n)
