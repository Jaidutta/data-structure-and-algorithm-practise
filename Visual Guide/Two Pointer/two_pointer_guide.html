👉👈 Two Pointer Algorithm - Complete Visual Guide

1️⃣ What is the Two Pointer Technique?

Two Pointer is an algorithmic pattern where you use two pointers (indices) to traverse a data structure, typically an array or string. Instead of using nested loops, you move pointers intelligently to solve the problem in linear time.
💡 Key Idea: Use TWO pointers instead of TWO nested loops
Reduces time complexity from O(n²) to O(n)!

2️⃣ How to Recognize Two Pointer Problems

🎯 Recognition Patterns

The input is a sorted array or string
You need to find a pair, triplet, or subarray
Problem asks for "in-place" modification (constant space)
You need to compare elements from both ends
Problem involves removing duplicates
Problem involves finding a window or subarray with certain properties
You're checking if something is a palindrome
Keywords: "sorted", "pair", "remove", "in-place", "palindrome", "merge"
⚠️ NOT a Two Pointer Problem When:
  • Array is unsorted AND you can't sort it
  • You need to track multiple elements simultaneously (might need hash map)
  • Problem requires finding all combinations (might need backtracking)
  • Need to maintain order of appearance (might need different approach)

// Two Sum - Sorted Array public static int[] twoSum(int[] arr, int target) { int left = 0; int right = arr.length - 1; while (left < right) { int sum = arr[left] + arr[right]; if (sum == target) { return new int[] {left, right}; // Found! } else if (sum < target) { left++; // Need larger sum } else { right--; // Need smaller sum } } return new int[] {-1, -1}; // Not found } // Time: O(n), Space: O(1)
✅ Why Two Pointers Works Here:
  • Array is sorted - critical requirement!
  • If sum is too large, we need smaller numbers → move right pointer left
  • If sum is too small, we need larger numbers → move left pointer right
  • We eliminate possibilities at each step - never miss the answer

// Remove Duplicates from Sorted Array public static int removeDuplicates(int[] arr) { if (arr.length == 0) return 0; int slow = 0; // Points to last unique element for (int fast = 1; fast < arr.length; fast++) { if (arr[fast] != arr[slow]) { slow++; arr[slow] = arr[fast]; // Copy unique element } } return slow + 1; // Length of unique elements } // Time: O(n), Space: O(1)

// Valid Palindrome public static boolean isPalindrome(String s) { int left = 0; int right = s.length() - 1; while (left < right) { // Skip non-alphanumeric characters while (left < right && !Character.isLetterOrDigit(s.charAt(left))) { left++; } while (left < right && !Character.isLetterOrDigit(s.charAt(right))) { right--; } // Compare characters (case-insensitive) if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) { return false; } left++; right--; } return true; } // Time: O(n), Space: O(1)

7️⃣ Example 4: Container With Most Water

Problem: Given heights of vertical lines, find two lines that together with x-axis form a container with maximum water.

Maximizing Area

Heights: [1, 8, 6, 2, 5, 4, 8, 3, 7]

Strategy:

  • Area = min(height[L], height[R]) × (R - L)
  • Start with widest container (L=0, R=end)
  • Move the pointer with smaller height (only way to potentially get larger area)
// Container With Most Water public static int maxArea(int[] height) { int left = 0; int right = height.length - 1; int maxArea = 0; while (left < right) { // Calculate area with current pointers int width = right - left; int minHeight = Math.min(height[left], height[right]); int area = width * minHeight; maxArea = Math.max(maxArea, area); // Move pointer with smaller height if (height[left] < height[right]) { left++; } else { right--; } } return maxArea; } // Time: O(n), Space: O(1)
💡 Why move the shorter line?
Moving the taller line can only decrease area (width decreases, height stays same or gets smaller).
Moving the shorter line gives a chance to find a taller line and increase area!
