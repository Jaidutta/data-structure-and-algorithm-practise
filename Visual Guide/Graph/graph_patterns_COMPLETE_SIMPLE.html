🎯 PATTERN 1: Finding Loops (Cycle Detection)

🔍 WHEN TO USE THIS PATTERN

Look for these keywords in the problem:

Keyword/Phrase Example Problem
"Detect cycle" "Is there a cycle in this graph?"
"Circular dependency" "Check for circular dependencies in tasks"
"Can complete all tasks" "Can you finish all courses?" (if cycle → NO!)
"Valid tree" "Is this a valid tree?" (trees have NO cycles)
"Return to start" "Can you return to where you started?"

🧒 Explain It Like I'm 10

The Breadcrumb Story:

You're walking through your neighborhood.
You drop a breadcrumb at every house you visit.

If you ever step on a breadcrumb you ALREADY dropped → You walked in a CIRCLE!
That's a loop!

That's ALL cycle detection is!
Mark where you've been. If you come back to a marked place → LOOP FOUND!

🔗 Connecting Algorithm to Analogy

Algorithm Step Breadcrumb Analogy
visited[] array = Your breadcrumbs on the ground
DFS(node) = Walking to a new house
visited[node] = true = Dropping a breadcrumb at this house
if (visited[neighbor] && neighbor != parent) = "Oh no! I already dropped a breadcrumb here!" → LOOP!
for (neighbor : neighbors) = Looking at all connected houses

📊 Visual Step-by-Step Execution

Step 1: Start at house 0, drop breadcrumb

Breadcrumbs: {0} | Current: 0

Step 2: Walk to house 1, drop breadcrumb

Breadcrumbs: {0, 1} | Path so far: 0→1

Step 3: Continue to 2, then 3, then 4...

Breadcrumbs: {0, 1, 2, 3, 4} | Path: 0→1→2→3→4

Step 4: House 4 connects back to house 0... I ALREADY HAVE A BREADCRUMB THERE!

❌ CYCLE DETECTED! I stepped on my own breadcrumb! Path: 0→1→2→3→4→0

// Pattern 1: Cycle Detection - The Breadcrumb Method! boolean[] breadcrumbs = new boolean[graph.size()]; // Track where we've been public boolean hasCycle(int house, int cameFrom) { breadcrumbs[house] = true; // Drop breadcrumb at this house // Look at all neighboring houses for (int neighbor : graph.get(house)) { // Don't go back to the house we just came from if (neighbor == cameFrom) continue; // OH NO! We already dropped a breadcrumb here! That's a LOOP! if (breadcrumbs[neighbor]) { return true; // CYCLE FOUND! } // Walk to this neighbor if (hasCycle(neighbor, house)) { return true; } } return false; // No cycle found }

🎯 PATTERN 2: Finding Quickest Way (BFS Shortest Path)

🔍 WHEN TO USE THIS PATTERN

Look for these keywords in the problem:

Keyword/Phrase Example Problem
"Shortest path" "What's the shortest path from A to B?"
"Minimum steps" "Minimum steps to reach the target"
"Fewest moves" "Fewest moves to solve puzzle"
"Level by level" "Process layer by layer"
IMPORTANT All edges have SAME cost (unweighted)

🧒 Explain It Like I'm 10

The Grocery Store Story:

You're looking for milk in a huge grocery store.

You check aisles closest to you FIRST (distance 1).
Then aisles a bit further away (distance 2).
Then even further aisles (distance 3).

Like ripples when you drop a stone in water - expanding circles!

The FIRST time you find milk = shortest distance!
Why? Because you checked all closer aisles first!

🔗 Connecting Algorithm to Analogy

Algorithm Step Grocery Store Analogy
Queue = List of aisles to check (waiting in line)
queue.add(start) = Start where you are now
node = queue.poll() = Check the next aisle in line
for (neighbor : neighbors) = Look at all aisles connected to this one
queue.add(neighbor) = Add them to the end of the line to check later
distance++ = One more step away from start

📊 Visual Step-by-Step Execution

Step 1: Start at aisle 0 (distance = 0)

Queue: [0] | Distance: 0 | Level 0

Step 2: Check aisles 1 step away (distance = 1)

Queue: [1, 2] | Checked all Level 1 aisles

Step 3: Check aisles 2 steps away (distance = 2)

Queue: [3] | Level 2 reached

Step 4: Check aisle 3's neighbor... FOUND MILK!

✅ FOUND MILK! Shortest path: 0→1→3→4 (3 steps)

// Pattern 2: BFS Shortest Path - The Grocery Store Method! Queue waitingLine = new LinkedList<>(); // Aisles to check int distance = 0; waitingLine.add(start); // Start where you are visited[start] = true; while (!waitingLine.isEmpty()) { int levelSize = waitingLine.size(); // How many aisles at this distance? // Check ALL aisles at this distance for (int i = 0; i < levelSize; i++) { int currentAisle = waitingLine.poll(); // Check next aisle in line if (currentAisle == milk) return distance; // FOUND IT! // Look at connected aisles for (int neighbor : graph.get(currentAisle)) { if (!visited[neighbor]) { visited[neighbor] = true; waitingLine.add(neighbor); // Add to end of line } } } distance++; // Move one step further away } return -1; // Milk not found

🎯 PATTERN 3: Finding Right Order (Topological Sort)

🔍 WHEN TO USE THIS PATTERN

Look for these keywords in the problem:

Keyword/Phrase Example Problem
"Order of tasks" "In what order should I do these?"
"Prerequisites" "Task A needs Task B done first"
"Dependencies" "X depends on Y"
"Must do X before Y" "Course A before Course B"
"Schedule" / "Sequence" "Find valid sequence"

🧒 Explain It Like I'm 10

The Getting Dressed Story:

You MUST put on underwear before pants.
You MUST put on socks before shoes.
But you can put shirt on anytime!

Valid order: underwear → socks → pants → shirt → shoes ✅
Invalid order: shoes → socks (WRONG! Socks first!) ❌

Topological sort finds ANY valid order!
Rule: If A must come before B, then A appears first!

🔗 Connecting Algorithm to Analogy

Algorithm Step Getting Dressed Analogy
indegree[i] = How many things must go on BEFORE this?
indegree[shirt] = 0 = Shirt needs nothing first! Can put it on anytime!
indegree[pants] = 1 = Pants needs 1 thing first (underwear)
queue (indegree = 0) = Things you CAN do right now!
indegree-- = One less thing to wait for!
result.add() = Put this on! Add to your outfit order!

📊 Visual Step-by-Step: Course Prerequisites

Start: Count prerequisites (indegrees)

Can do now (needs 0): Course 0!
Queue: [0]

Step 1: Take Course 0, update neighbors

Order so far: [0]
Can do now: Courses 1 and 2! Queue: [1, 2]

Step 2-3: Take courses 1 and 2

Order so far: [0, 1, 2]
Can do now: Course 3! Queue: [3]

Final: Take course 3 - ALL DONE!

✅ VALID ORDER: [0, 1, 2, 3] - All prerequisites satisfied!

// Pattern 3: Topological Sort - The Getting Dressed Method! int[] needsFirst = new int[numCourses]; // How many prerequisites? // Count prerequisites for each course for (int course : allCourses) { for (int nextCourse : requires[course]) { needsFirst[nextCourse]++; // This needs one more thing first } } Queue canDoNow = new LinkedList<>(); // Find what we can do right now (needs nothing) for (int i = 0; i < numCourses; i++) { if (needsFirst[i] == 0) { canDoNow.add(i); // Can put this on now! } } List order = new ArrayList<>(); while (!canDoNow.isEmpty()) { int course = canDoNow.poll(); order.add(course); // Add to our outfit order! // What can we do next? for (int next : requires[course]) { needsFirst[next]--; // One less thing to wait for! if (needsFirst[next] == 0) { canDoNow.add(next); // Can do this now! } } } return order; // The order to get dressed!

🎯 PATTERN 4: Grouping Things Together (Union Find)

🔍 WHEN TO USE THIS PATTERN

Look for these keywords in the problem:

Keyword/Phrase Example Problem
"Connected components" "How many separate groups?"
"Are X and Y connected?" "Are these two in the same group?"
"Merge/Union groups" "Combine these two groups"
"Dynamic connectivity" "Groups change over time"
"Redundant connection" "Which edge creates a loop?"

🧒 Explain It Like I'm 10

The Party Friends Story:

You're organizing a party. Everyone starts alone (their own group).

"Alice is friends with Bob" → Put them in same group! (Alice-Bob)
"Bob is friends with Charlie" → Now Alice-Bob-Charlie are ALL in one group!
"David is friends with Eve" → They make a DIFFERENT group! (David-Eve)

Quick Questions:
"Are Alice and Charlie in same group?" → YES! (through Bob)
"Are Alice and David in same group?" → NO! (separate groups)

Union Find keeps track of who's in which friend group!

🔗 Connecting Algorithm to Analogy

Algorithm Step Party Friends Analogy
parent[i] = i = Everyone is their own boss initially
find(x) = "Who's the boss of your friend group?"
union(x, y) = "Make X and Y's groups become friends!"
if (find(x) == find(y)) = "Do X and Y have the same boss?" (same group!)
parent[rootX] = rootY = "Make one boss report to the other"

📊 Visual Step-by-Step: Making Friend Groups

Start: Everyone is alone (their own boss)

Groups: 5 separate people

Step 1: Union(0, 1) - "0 and 1 are friends!"

Groups: {0,1}, {2}, {3}, {4} → 4 groups now

Step 2: Union(2, 3) - "2 and 3 are friends!"

Groups: {0,1}, {2,3}, {4} → 3 groups

Step 3: Union(1, 3) - "Merge the two big groups!"

✅ Groups: {0,1,2,3}, {4} → 2 groups! Everyone points to boss 0!

// Pattern 4: Union Find - The Party Friends Method! int[] boss; // Who's the boss of each person's group? // Start: Everyone is their own boss public UnionFind(int n) { boss = new int[n]; for (int i = 0; i < n; i++) { boss[i] = i; // I'm my own boss! } } // Find: Who's the BIG BOSS of your group? public int find(int person) { if (boss[person] != person) { // Follow the chain to the BIG BOSS boss[person] = find(boss[person]); // Shortcut for next time! } return boss[person]; } // Union: Make two groups become friends! public void union(int person1, int person2) { int boss1 = find(person1); // Find boss of group 1 int boss2 = find(person2); // Find boss of group 2 if (boss1 != boss2) { boss[boss1] = boss2; // One boss reports to the other! } } // Are they in the same friend group? public boolean connected(int person1, int person2) { return find(person1) == find(person2); // Same boss = same group! }

🎯 PATTERN 5: Finding Cheapest Route (Dijkstra's Algorithm)

🔍 WHEN TO USE THIS PATTERN

Look for these keywords in the problem:

Keyword/Phrase Example Problem
"Shortest path" with costs "Cheapest route from A to B"
"Minimum cost" "Minimum cost to connect all"
"Weighted graph" "Edges have different costs/times"
"Network delay" "Minimum time for signal to reach all"
IMPORTANT NO negative weights! (use Bellman-Ford if negative)

🧒 Explain It Like I'm 10

The Candy Store Story:

You want candy! Different stores cost different amounts to reach:
• Store A: $5 to get there
• Store B: $2 to get there, but Store C costs $1 from Store B (total $3)
• Store C: $10 to get there directly

Which is cheaper? Go to B first ($2), then to C ($1) = $3 total!
Better than going straight to C ($10)!

Dijkstra's always picks the cheapest unvisited place next!
Like a greedy person always going to closest/cheapest candy first!

🔗 Connecting Algorithm to Analogy

Algorithm Step Candy Store Analogy
dist[] array = "How much to get to each store?"
PriorityQueue = "Sorted list: cheapest stores first!"
dist[start] = 0 = "You're at home, costs $0!"
pq.poll() = "Pick cheapest unvisited store"
if (newCost < dist[neighbor]) = "Found a cheaper way to get there!"
dist[neighbor] = newCost = "Update the price tag!"

📊 Visual Step-by-Step: Finding Cheapest Candy

Start: You're at store 0, costs to others unknown

Costs: [0, ∞, ∞, ∞] | Next cheapest: Store 0 ($0)

Step 1: From store 0, update neighbor costs

Costs: [0, 4, 2, ∞] | Next cheapest: Store 2 ($2)

Step 2: From store 2 ($2), can reach store 3 for $2+$3=$5!

✅ Cheapest path to store 3: 0→2→3 costs $5!

// Pattern 5: Dijkstra's - The Candy Store Method! int[] cost = new int[n]; // Cost to reach each store Arrays.fill(cost, Integer.MAX_VALUE); // All stores cost ∞ initially cost[start] = 0; // You're at home, costs $0! // Priority Queue: Always pick cheapest unvisited store! PriorityQueue cheapestFirst = new PriorityQueue<>((a,b) -> a[0] - b[0]); cheapestFirst.add(new int[]{0, start}); // [cost, store] while (!cheapestFirst.isEmpty()) { int[] current = cheapestFirst.poll(); int currentCost = current[0]; int store = current[1]; if (currentCost > cost[store]) continue; // Already found cheaper way // Check all neighbor stores for (int[] edge : graph.get(store)) { int nextStore = edge[0]; int travelCost = edge[1]; int newTotalCost = currentCost + travelCost; // Cost to go there through here // Found a cheaper way! if (newTotalCost < cost[nextStore]) { cost[nextStore] = newTotalCost; // Update price tag! cheapestFirst.add(new int[]{newTotalCost, nextStore}); } } } return cost; // Cheapest way to reach each store!

🎯 PATTERN 7: Two Teams (Bipartite Check)

🔍 WHEN TO USE THIS PATTERN

Keyword/PhraseExample Problem
"Divide into two groups""Can split into 2 teams?"
"Bipartite""Is graph bipartite?"
"Two colors""Color with 2 colors only"
"Possible partition""Can we partition into 2?"

🧒 Explain It Like I'm 10

The Team Selection Story:

You're dividing class into Team RED and Team BLUE.
Rule: Friends CANNOT be on the same team!

If Alice is RED → her friends MUST be BLUE
If Bob is BLUE → his friends MUST be RED

If you find "Alice is RED but her friend is also RED" → IMPOSSIBLE!
Otherwise → Possible! ✅

🔗 Algorithm ↔ Analogy

AlgorithmTeam Analogy
color[i] = 0= Not on any team yet
color[i] = 1= Team RED
color[i] = -1= Team BLUE
color[neighbor] = -color[node]= Friends go on opposite team!
if (same color)= Friends on same team → IMPOSSIBLE!

📊 Visual Step-by-Step: Can We Split Into 2 Teams?

Initial: Nobody on a team yet

Teams: [], [] | Rule: Friends must be on opposite teams!

Step 1: Put person 0 on Team RED

Team RED: {0} | Team BLUE: {}

Step 2: Person 0's friends (1, 2) must go on BLUE team

Team RED: {0} | Team BLUE: {1, 2}

Step 3: Person 1 and 2's friend (3) must go on RED team

✅ SUCCESS! Can split into 2 teams! RED: {0, 3} | BLUE: {1, 2}

Example of FAILURE: Triangle (odd cycle)

❌ IMPOSSIBLE! Node 2 should be RED (opposite of 0) but already BLUE (opposite of 1)

// Pattern 7: Two Teams (Bipartite)! int[] team = new int[n]; // 0=none, 1=RED, -1=BLUE public boolean canSplitIntoTwoTeams(int start) { Queue queue = new LinkedList<>(); queue.add(start); team[start] = 1; // Start on Team RED while (!queue.isEmpty()) { int person = queue.poll(); for (int friend : friends.get(person)) { if (team[friend] == 0) { team[friend] = -team[person]; // Opposite team! queue.add(friend); } else if (team[friend] == team[person]) { return false; // Friends on same team! Impossible! } } } return true; // Successfully split into 2 teams! }

🎯 PATTERN 8: Cheapest Network (Minimum Spanning Tree - Kruskal's)

🔍 WHEN TO USE THIS PATTERN

Keyword/PhraseExample Problem
"Connect all" + "minimum cost""Min cost to connect all cities"
"Minimum spanning tree""MST problem"
"Connect points with min cable""Cheapest way to wire everything"

🧒 Explain It Like I'm 10

The Cable Connection Story:

You need to connect all houses with internet cable.
Different connections cost different amounts.

Strategy: Always pick the CHEAPEST cable that doesn't create a loop!

Why no loops? If A connects to B and B connects to C, you don't need A→C directly!
Sort cables by price, add cheapest ones that don't make loops!

🔗 Algorithm ↔ Analogy

AlgorithmCable Analogy
Sort edges by weight= Sort cables by price
Union Find= Check if creates loop
if (!creates cycle)= If doesn't make loop
add edge= Install this cable!

📊 Visual Step-by-Step: Building Cheapest Network

Start: 4 houses, need to connect all. Cables sorted by price: 1, 2, 3, 4, 5

MST Edges: [] | Total Cost: $0

Step 1: Add cheapest cable (2-3, cost=$1)

MST Edges: [2-3] | Total Cost: $1

Step 2: Add next cheapest (0-2, cost=$2)

MST Edges: [2-3, 0-2] | Total Cost: $1 + $2 = $3

Step 3: Add 1-2 (cost=$3). Skip 0-1 (creates loop!)

✅ MST Complete! Cables: [2-3, 0-2, 1-2] | Total Cost: $1 + $2 + $3 = $6

// Pattern 8: Cheapest Network (MST)! Arrays.sort(cables, (a,b) -> a[2] - b[2]); // Sort by price! UnionFind houses = new UnionFind(n); int totalCost = 0; for (int[] cable : cables) { int house1 = cable[0], house2 = cable[1], price = cable[2]; // Will this create a loop? if (houses.find(house1) != houses.find(house2)) { houses.union(house1, house2); // Install cable! totalCost += price; } } return totalCost; // Total cost to connect all houses!
