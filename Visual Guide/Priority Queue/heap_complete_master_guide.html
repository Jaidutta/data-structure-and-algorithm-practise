1. What is a Heap? 🤔

Simple Definition:

A heap is a complete binary tree where every parent node has a specific ordering relationship with its children. It's NOT sorted, but has a partial ordering that makes it perfect for priority queues and finding min/max elements quickly.

Key Characteristics:

  • ✓ Complete Binary Tree: Filled level by level, left to right
  • ✓ Heap Property: Parent-child ordering (max or min)
  • ✓ NOT Sorted: Only partial ordering exists
  • ✓ Array-based: Usually stored in array, not with pointers
  • ✓ Efficient: O(log n) insertions and deletions

2. Heap Property & Types 🎯

Two Types of Heaps:

Max-Heap (Parent ≥ Children)

Property: Every parent is greater than or equal to its children

Root contains: Maximum element in the heap

Use when: Need to quickly access the largest element

MAX-HEAP Example:

            100
           /    \
        90          80
       /  \     /  \
     70   60    75  50

✓ 100 ≥ 90, 80
✓ 90 ≥ 70, 60
✓ 80 ≥ 75, 50
All parent ≥ children relationships satisfied!

Min-Heap (Parent ≤ Children)

Property: Every parent is less than or equal to its children

Root contains: Minimum element in the heap

Use when: Need to quickly access the smallest element

MIN-HEAP Example:

            10
           /    \
        20          30
       /  \     /  \
     40   50    60  70

✓ 10 ≤ 20, 30
✓ 20 ≤ 40, 50
✓ 30 ≤ 60, 70
All parent ≤ children relationships satisfied!
Important Note: Heaps are NOT sorted! Siblings have no ordering relationship. For example, in the max-heap above, 90 and 80 have no required order between them.

3. Array Representation 📊

Why Use Arrays?

Heaps are stored in arrays (not with left/right pointers) because:

  • ✓ Space efficient: No pointer overhead
  • ✓ Cache friendly: Better memory locality
  • ✓ Simple indexing: Easy parent/child calculation

Index Formulas (0-based indexing):

// For any node at index i:
Parent index      = (i - 1) / 2
Left child index  = 2 * i + 1
Right child index = 2 * i + 2

4. What is Heapify? ⚡

Definition:

Heapify is the process of restoring the heap property when it has been violated. It "fixes" the heap by moving elements up or down until the parent-child relationships are correct again.

Two Types of Heapify:

🔼 Heapify Up (Sift Up / Bubble Up)

When to use:

  • ✓ After inserting a new element at the bottom
  • ✓ The new element might be too large (max-heap) or too small (min-heap)

What it does:

  • → Compares node with its parent
  • → Swaps if heap property violated
  • → Moves ⬆ UP toward root until property satisfied

🔽 Heapify Down (Sift Down / Bubble Down)

When to use:

  • ✓ After removing the root (extract max/min)
  • ✓ When building a heap from an array
  • ✓ The replacement element might be too small (max-heap) or too large (min-heap)

What it does:

  • → Compares node with its children
  • → Swaps with larger child (max-heap) or smaller child (min-heap)
  • → Moves ⬇ DOWN toward leaves until property satisfied

Quick Visual Summary:

4.5. When is Heapify Done? 🛑

Quick Answer:

Heapify is done when the heap property is satisfied - meaning every parent is correctly ordered with respect to its children, and we can't move the element any further.

Heapify Up Stops When:

  1. Reached the root: No parent to compare with → DONE ✅
  2. Parent is valid:
    • Max-heap: parent ≥ current node → DONE ✅
    • Min-heap: parent ≤ current node → DONE ✅

Example: Heapify Up Stops

Max-Heap: Insert 95

Initial:    110
           /   \
         85     100

After insert at end:
            110
           /   \
         85     100
        /
      95

Check: 95 > parent(85)? YES → Swap

After swap:
            110
           /   \
         95     100    ← 95 moved up
        /
      85              ← 85 moved down

Check: 95 > parent(110)? NO ✅
STOP! Heapify up is DONE
Parent (110) ≥ current (95), heap property satisfied!

Heapify Down Stops When:

  1. Reached a leaf: No children to compare with → DONE ✅
  2. Current node is valid:
    • Max-heap: current ≥ both children → DONE ✅
    • Min-heap: current ≤ both children → DONE ✅
  3. Only one child and valid: Check only that child → DONE if valid ✅

Example: Heapify Down Stops

Max-Heap: Delete root, replace with 85

After replacement:
             85         ← Too small!
           /   \
         140    130

Check: 85 < max(140, 130) = 140? YES → Swap with 140

After swap:
            140         ← 140 up
           /   \
         85     130     ← 85 down
        /  \
      120  110

Check: 85 < max(120, 110) = 120? YES → Swap with 120

After swap:
            140
           /   \
         120    130     ← 120 up
        /  \
      85   110         ← 85 down

Check: 85 has no children? YES ✅
STOP! Heapify down is DONE
Reached a leaf node!

Common Stopping Scenarios:

Scenario Heapify Up Heapify Down
Best Case Element already in correct position (0 swaps) Replacement already larger than children (0 swaps)
Worst Case Bubble all the way to root (log n swaps) Sink all the way to leaf (log n swaps)
Partial Case Stop somewhere in middle when parent ≥ child Stop somewhere in middle when current ≥ children

Pseudocode with Stop Conditions:

// HEAPIFY UP (Max-Heap)
void heapifyUp(int index) {
    while (index > 0) {  // Not at root yet
        int parentIdx = (index - 1) / 2;
        
        // STOP CONDITION: Parent is valid (parent ≥ current)
        if (heap[parentIdx] >= heap[index]) {
            break;  // ✅ DONE! Heap property satisfied
        }
        
        // Swap and continue
        swap(heap[parentIdx], heap[index]);
        index = parentIdx;
    }
    // Also stops when index = 0 (reached root)
}

// HEAPIFY DOWN (Max-Heap)
void heapifyDown(int index) {
    int size = heap.length;
    
    while (true) {
        int leftChild = 2 * index + 1;
        int rightChild = 2 * index + 2;
        int largest = index;
        
        // STOP CONDITION 1: No left child (reached leaf)
        if (leftChild >= size) {
            break;  // ✅ DONE! No children to compare
        }
        
        // Find largest among current, left child, right child
        if (heap[leftChild] > heap[largest]) {
            largest = leftChild;
        }
        if (rightChild < size && heap[rightChild] > heap[largest]) {
            largest = rightChild;
        }
        
        // STOP CONDITION 2: Current is already largest
        if (largest == index) {
            break;  // ✅ DONE! Heap property satisfied
        }
        
        // Swap and continue
        swap(heap[index], heap[largest]);
        index = largest;
    }
}

⚠️ Important Notes:

  • Always check bounds: Make sure child indices are within array bounds
  • Single child edge case: Last level might have only left child, handle carefully
  • Equal values: If parent = child, heap property is satisfied (stop)
  • No infinite loops: Each iteration moves closer to stopping condition

✅ Summary - Heapify is Done When:

Operation Stop Condition
Heapify Up Reached root OR parent ≥ current (max) / parent ≤ current (min)
Heapify Down Reached leaf OR current ≥ both children (max) / current ≤ both children (min)

5. Where & When Heaps Are Used 🎯

Heaps are the foundation for many important algorithms and data structures!

Use Case Why Heap? Examples
Priority Queue Always get highest/lowest priority in O(1) Task scheduling, event simulation, Dijkstra's algorithm
Heap Sort O(n log n) sorting with O(1) space In-place sorting algorithm
Top K Problems Efficiently maintain k largest/smallest K largest elements, K closest points, median of stream
Graph Algorithms Efficiently get next best edge/node Dijkstra's shortest path, Prim's MST, A* search
Merge K Sorted Lists Always get smallest element across K lists External sorting, merge sorted streams
Median Finding Two heaps give O(log n) insert, O(1) median Running median, sliding window median

When to Choose Heap vs Other Structures:

Priority queue ✓ Standard choice -
Sorted order ✗ Heap is NOT sorted Use BST or sort array
Search by value ✗ O(n) search Use HashMap or BST
Frequent updates ✓ O(log n) insert/delete -

8. Five Complete Deletion Examples 🔽

Deletion Pattern (Extract Max from Max-Heap):

  1. Step 1: Save root value (this is what we're returning)
  2. Step 2: Replace root with LAST element
  3. Step 3: Remove last element (shrink heap size)
  4. Step 4: Heapify Down ⬇ from root
  5. Step 5: Compare with children, swap with LARGER child (max-heap)
  6. Step 6: Repeat until parent ≥ both children or reach leaf

9. Building a Heap from Array 🏗️

Build Heap Algorithm:

Start from the last non-leaf node and heapify down to the root. This is surprisingly O(n), not O(n log n)!

Why Start from n/2 - 1?

Key Insight: Nodes from index n/2 to n-1 are all leaves (have no children).

Heapify-ing a leaf does nothing, so we skip them and start from the last non-leaf node at index (n/2 - 1).

Example: Build Max-Heap from [10, 20, 15, 30, 40]

Initial Array: [10, 20, 15, 30, 40]
Tree View (before heapify):

        10
       /    \
     20          15
    /  \
  30  40

Start from index (5/2 - 1) = 1

Step 1: Heapify at index 1 (20)
- Children: 30, 40
- Max child = 40
- 20 < 40 → Swap

        10
       /    \
     40          15
    /  \
  30  20

Step 2: Heapify at index 0 (10)
- Children: 40, 15
- Max child = 40
- 10 < 40 → Swap

        40
       /    \
     10          15
    /  \
  30  20

- Continue heapify down 10:
- Children: 30, 20
- Max child = 30
- 10 < 30 → Swap

Final Max-Heap:

        40
       /    \
     30          15
    /  \
  10  20

Final Array: [40, 30, 15, 10, 20]

10. Time & Space Complexity ⏱️

Operation Time Complexity Space Complexity Explanation
Peek (Get Max/Min) O(1) O(1) Just return arr[0]
Insert O(log n) O(1) Bubble up max log n levels
Extract Max/Min O(log n) O(1) Sink down max log n levels
Build Heap O(n) O(1) Tighter analysis shows O(n), not O(n log n)!
Heap Sort O(n log n) O(1) Build heap O(n) + n extractions O(log n)
Search for element O(n) O(1) Must check all elements (not optimized for search)

Why Build Heap is O(n), not O(n log n):

While each heapify-down operation is O(log n), most nodes are near the bottom of the tree and only need to sink a few levels. The mathematical analysis (summing over all levels) proves the total work is O(n).

11. Code Templates 💻

Max-Heap Implementation in Java:

class MaxHeap {
    private int[] heap;
    private int size;
    private int capacity;
    
    // Constructor
    public MaxHeap(int capacity) {
        this.capacity = capacity;
        this.size = 0;
        this.heap = new int[capacity];
    }
    
    // Parent/child index calculations
    private int parent(int i) { return (i - 1) / 2; }
    private int leftChild(int i) { return 2 * i + 1; }
    private int rightChild(int i) { return 2 * i + 2; }
    
    // Swap helper
    private void swap(int i, int j) {
        int temp = heap[i];
        heap[i] = heap[j];
        heap[j] = temp;
    }
    
    // HEAPIFY UP (after insertion)
    private void heapifyUp(int i) {
        while (i > 0 && heap[parent(i)] < heap[i]) {
            swap(i, parent(i));
            i = parent(i);
        }
    }
    
    // HEAPIFY DOWN (after deletion)
    private void heapifyDown(int i) {
        int maxIndex = i;
        int left = leftChild(i);
        int right = rightChild(i);
        
        // Find largest among root, left, right
        if (left < size && heap[left] > heap[maxIndex]) {
            maxIndex = left;
        }
        if (right < size && heap[right] > heap[maxIndex]) {
            maxIndex = right;
        }
        
        // If root is not largest, swap and continue
        if (i != maxIndex) {
            swap(i, maxIndex);
            heapifyDown(maxIndex);
        }
    }
    
    // INSERT
    public void insert(int value) {
        if (size == capacity) {
            throw new RuntimeException("Heap is full");
        }
        
        // Add at end
        heap[size] = value;
        size++;
        
        // Heapify up from last position
        heapifyUp(size - 1);
    }
    
    // EXTRACT MAX (delete root)
    public int extractMax() {
        if (size == 0) {
            throw new RuntimeException("Heap is empty");
        }
        
        int max = heap[0];
        
        // Replace root with last element
        heap[0] = heap[size - 1];
        size--;
        
        // Heapify down from root
        if (size > 0) {
            heapifyDown(0);
        }
        
        return max;
    }
    
    // PEEK (get max without removing)
    public int peek() {
        if (size == 0) {
            throw new RuntimeException("Heap is empty");
        }
        return heap[0];
    }
    
    // BUILD HEAP from array
    public static void buildHeap(int[] arr) {
        int n = arr.length;
        
        // Start from last non-leaf node
        for (int i = n / 2 - 1; i >= 0; i--) {
            // Heapify down each node
            heapifyDownArray(arr, n, i);
        }
    }
    
    private static void heapifyDownArray(int[] arr, int n, int i) {
        int largest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;
        
        if (left < n && arr[left] > arr[largest]) largest = left;
        if (right < n && arr[right] > arr[largest]) largest = right;
        
        if (largest != i) {
            int temp = arr[i];
            arr[i] = arr[largest];
            arr[largest] = temp;
            heapifyDownArray(arr, n, largest);
        }
    }
}

import java.util.PriorityQueue;
import java.util.Collections;

// Min-Heap (default)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();

// Max-Heap (use reverse comparator)
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());

// Operations
maxHeap.offer(10);     // Insert
maxHeap.poll();       // Extract max
maxHeap.peek();       // Get max without removing
maxHeap.size();       // Get size

🎯 Quick Reference Summary

Concept Key Points
What is Heap? Complete binary tree with heap property (parent ≥ children for max-heap)
Heapify Up After insert: compare with parent, swap if needed, move UP ⬆
Heapify Down After delete: compare with children, swap with larger, move DOWN ⬇
Insert Add at end → Heapify Up → O(log n)
Delete Max Replace root with last → Heapify Down → O(log n)
When to Use Priority queues, top K problems, graph algorithms, scheduling
