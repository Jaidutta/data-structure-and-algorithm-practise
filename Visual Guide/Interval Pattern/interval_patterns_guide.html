📖 What are Intervals?

Definition

Interval: A pair of numbers representing a range with a start and end point.

Interval = [start, end]
where start ≤ end

Common Representations:

// Java representations

// 1. Array of two elements
int[] interval = {1, 3};  // [1, 3]

// 2. Class (most common in LeetCode)
class Interval {
    int start;
    int end;
    Interval(int s, int e) { start = s; end = e; }
}

// 3. 2D array (array of intervals)
int[][] intervals = {{1,3}, {2,6}, {8,10}};

Key Concepts

1 Overlap: Two intervals overlap if they share at least one point

• [1, 4] and [3, 6] overlap (share points 3, 4)

• [1, 3] and [5, 7] do NOT overlap

2 Merge: Combine overlapping intervals into one

• [1, 4] + [3, 6] = [1, 6]

3 Intersection: The common part of overlapping intervals

• [1, 4] ∩ [3, 6] = [3, 4]

4 Insert: Add a new interval and handle overlaps

• Insert [2, 5] into [[1, 2], [6, 9]] → [[1, 5], [6, 9]]

Pattern 1: Merge Overlapping Intervals

📝 Pattern 1: Merge Overlapping Intervals

Problem: Merge all overlapping intervals into non-overlapping intervals

Strategy: Sort by start time, then merge overlapping ones

Time: O(N log N) for sorting, Space: O(N)

🎯 Instant Recognition

You KNOW it's this pattern when:

  • ✓ Problem explicitly says "merge overlapping intervals"
  • ✓ "Combine intervals", "consolidate ranges"
  • ✓ Input: List of intervals, Output: Merged list
  • ✓ Example: [[1,3],[2,6],[8,10]] → [[1,6],[8,10]]

🎬 Visual Execution: Merge [[1,3],[2,6],[8,10],[15,18]]

Step 1: Original Intervals

[1, 3]
[2, 6]
[8, 10]
[15, 18]

Step 2: Already Sorted by Start Time ✓

Intervals: [1,3], [2,6], [8,10], [15,18]

✓ 1 ≤ 2 ≤ 8 ≤ 15 (sorted)

Step 3: Merge Process

Iteration 1: Start with [1, 3]

• current = [1, 3]

• result = [[1, 3]]

Iteration 2: Check [2, 6]

• Does [2, 6] overlap with [1, 3]?

• Check: 2 ≤ 3? YES! (overlaps)

• Merge: [1, 3] + [2, 6] = [1, max(3, 6)] = [1, 6]

• current = [1, 6]

Iteration 3: Check [8, 10]

• Does [8, 10] overlap with [1, 6]?

• Check: 8 ≤ 6? NO! (no overlap)

• Add [1, 6] to result, start new interval [8, 10]

• current = [8, 10]

• result = [[1, 6], [8, 10]]

Iteration 4: Check [15, 18]

• Does [15, 18] overlap with [8, 10]?

• Check: 15 ≤ 10? NO! (no overlap)

• Add [8, 10] to result, start new interval [15, 18]

• current = [15, 18]

Step 4: Final Result

[1, 6]
[8, 10]
[15, 18]

✅ Merged from 4 intervals to 3 intervals!

import java.util.*;

public class Solution {
    public int[][] merge(int[][] intervals) {
        // Edge case: empty or single interval
        if (intervals.length <= 1) {
            return intervals;
        }
        
        // Step 1: Sort intervals by start time
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
        
        // Step 2: Merge overlapping intervals
        List<int[]> result = new ArrayList<>();
        int[] currentInterval = intervals[0];
        result.add(currentInterval);
        
        for (int[] interval : intervals) {
            int currentEnd = currentInterval[1];
            int nextStart = interval[0];
            int nextEnd = interval[1];
            
            // Check if overlaps: next start ≤ current end
            if (nextStart <= currentEnd) {
                // Merge: extend current interval's end
                currentInterval[1] = Math.max(currentEnd, nextEnd);
            } else {
                // No overlap: start new interval
                currentInterval = interval;
                result.add(currentInterval);
            }
        }
        
        // Convert list to array
        return result.toArray(new int[result.size()][]);
    }
}

💡 Intuition: Why This Pattern Works

The Key Insight:

  1. Sorting is crucial! Once sorted by start time, we only need to check if the current interval overlaps with the next one. We don't need to check all pairs!
  2. Overlap condition: Two intervals [a, b] and [c, d] overlap if c ≤ b (next start ≤ current end)
  3. Merge formula: [a, b] + [c, d] = [a, max(b, d)]

Real-World Analogy:

Think of merging calendar events:

• Meeting 1: 9am-11am

• Meeting 2: 10am-12pm

• They overlap! → Block out 9am-12pm

Overlap Check:
intervals[i][0] ≤ merged.end

Merge Formula:
merged.end = Math.max(merged.end, intervals[i][1])

Pattern 2: Insert Interval

📝 Pattern 2: Insert Interval

Problem: Insert a new interval into a sorted list of non-overlapping intervals

Strategy: Three phases: before, merge, after

Time: O(N), Space: O(N)

🎯 Instant Recognition

  • ✓ "Insert a new interval"
  • ✓ Given: SORTED non-overlapping intervals
  • ✓ Task: Insert one interval and merge if needed
  • ✓ Example: Insert [2,5] into [[1,2],[6,9]]

🎬 Visual Execution: Insert [4,8] into [[1,2],[3,5],[6,7],[8,10],[12,16]]

Original Intervals:

[1, 2]
[3, 5]
[6, 7]
[8, 10]
[12, 16]

New interval to insert: [4, 8]

Three Phases:

Phase 1: Add intervals BEFORE new interval

• [1, 2]: end (2) < new start (4) → Add to result

• Result: [[1, 2]]

Phase 2: MERGE overlapping intervals

• [3, 5]: overlaps with [4, 8]? 5 ≥ 4 → YES! Merge to [3, 8]

• [6, 7]: overlaps with [3, 8]? 7 ≥ 3 and 6 ≤ 8 → YES! Merge to [3, 8]

• [8, 10]: overlaps with [3, 8]? 10 ≥ 3 and 8 ≤ 8 → YES! Merge to [3, 10]

• Result after merge: [[1, 2], [3, 10]]

Phase 3: Add intervals AFTER merged interval

• [12, 16]: start (12) > merged end (10) → Add to result

• Final result: [[1, 2], [3, 10], [12, 16]]

Final Result:

[1, 2]
[3, 10]
[12, 16]

✅ Merged [3,5], [4,8], [6,7], [8,10] into [3,10]!

public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0;
    int n = intervals.length;
    
    // Phase 1: Add all intervals that come BEFORE newInterval
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i]);
        i++;
    }
    
    // Phase 2: Merge all overlapping intervals with newInterval
    while (i < n && intervals[i][0] <= newInterval[1]) {
        // Expand newInterval to include current interval
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);
    
    // Phase 3: Add all intervals that come AFTER newInterval
    while (i < n) {
        result.add(intervals[i]);
        i++;
    }
    
    return result.toArray(new int[result.size()][]);
}

💡 Intuition

Why three phases?

Since intervals are already SORTED and NON-OVERLAPPING:

  1. Everything that ends before new interval starts → Safe to add
  2. Everything that overlaps with new interval → Merge
  3. Everything that starts after new interval ends → Safe to add

Overlap Detection:

Interval [a, b] overlaps with [c, d] if:

a ≤ d AND c ≤ b

Pattern 3: Meeting Rooms (Overlap Detection)

📝 Pattern 3: Can Attend All Meetings?

Problem: Check if a person can attend all meetings (no overlaps)

Strategy: Sort and check consecutive intervals for overlap

Time: O(N log N), Space: O(1)

🎯 Instant Recognition

  • ✓ "Can attend all meetings?"
  • ✓ "Detect if any intervals overlap"
  • ✓ "Check for conflicts"
  • ✓ Return: boolean (yes/no)
public boolean canAttendMeetings(int[][] intervals) {
    if (intervals.length <= 1) return true;
    
    // Sort by start time
    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
    
    // Check each consecutive pair for overlap
    for (int i = 1; i < intervals.length; i++) {
        // If current meeting starts before previous ends → overlap!
        if (intervals[i][0] < intervals[i-1][1]) {
            return false;  // Conflict found
        }
    }
    
    return true;  // No conflicts
}

Example: [[0,30], [5,10], [15,20]]

After sorting: [[0,30], [5,10], [15,20]]

• Check [0,30] vs [5,10]: 5 < 30 → OVERLAP!

• Return FALSE (cannot attend all)

Pattern 4: Minimum Meeting Rooms (Interval Overlaps Count)

📝 Pattern 4: Minimum Meeting Rooms

Problem: Find minimum number of rooms needed for all meetings

Strategy: Track concurrent meetings using start/end times

Time: O(N log N), Space: O(N)

🎯 Instant Recognition

  • ✓ "Minimum meeting rooms needed"
  • ✓ "Maximum concurrent intervals"
  • ✓ "Peak number of overlaps"
  • ✓ "Minimum platforms needed" (train station)

🎬 Visual: [[0,30], [5,10], [15,20]]

Timeline View:

Meeting 1: [0, 30]
M2: [5,10]
M3: [15,20]

At time 5-10:

• Meeting 1 is running

• Meeting 2 is running

→ Need 2 rooms!

Maximum concurrent: 2 rooms needed

// Solution 1: Min Heap (Priority Queue)
public int minMeetingRooms(int[][] intervals) {
    if (intervals.length == 0) return 0;
    
    // Sort by start time
    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
    
    // Min heap to track end times of meetings
    PriorityQueue<Integer> endTimes = new PriorityQueue<>();
    endTimes.add(intervals[0][1]);
    
    for (int i = 1; i < intervals.length; i++) {
        // If earliest ending meeting finished, reuse that room
        if (intervals[i][0] >= endTimes.peek()) {
            endTimes.poll();
        }
        
        // Add current meeting's end time
        endTimes.add(intervals[i][1]);
    }
    
    // Size of heap = number of rooms needed
    return endTimes.size();
}

// Solution 2: Chronological Ordering (Sweep Line)
public int minMeetingRooms2(int[][] intervals) {
    int[] starts = new int[intervals.length];
    int[] ends = new int[intervals.length];
    
    for (int i = 0; i < intervals.length; i++) {
        starts[i] = intervals[i][0];
        ends[i] = intervals[i][1];
    }
    
    // Sort both arrays
    Arrays.sort(starts);
    Arrays.sort(ends);
    
    int rooms = 0;
    int maxRooms = 0;
    int startPtr = 0, endPtr = 0;
    
    while (startPtr < intervals.length) {
        if (starts[startPtr] < ends[endPtr]) {
            // Meeting starting, need a room
            rooms++;
            startPtr++;
        } else {
            // Meeting ending, free a room
            rooms--;
            endPtr++;
        }
        maxRooms = Math.max(maxRooms, rooms);
    }
    
    return maxRooms;
}

💡 Intuition

Two Approaches:

Approach 1: Min Heap

Track when rooms become available. If a room is free (earliest end time ≤ current start), reuse it. Otherwise, allocate new room.

Approach 2: Sweep Line

Treat starts and ends separately. Each start adds +1 room, each end adds -1 room. Maximum concurrent rooms = answer.

Pattern 5: Interval Intersection

📝 Pattern 5: Interval Intersection

Problem: Find intersection (overlap) of two interval lists

Strategy: Two pointers, find overlap region

Time: O(M + N), Space: O(min(M, N))

🎯 Instant Recognition

  • ✓ "Intersection of two interval lists"
  • ✓ "Common time slots"
  • ✓ "Overlapping periods"
  • ✓ Given: TWO sorted interval lists
public int[][] intervalIntersection(int[][] A, int[][] B) {
    List<int[]> result = new ArrayList<>();
    int i = 0, j = 0;
    
    while (i < A.length && j < B.length) {
        // Check if A[i] and B[j] overlap
        int start = Math.max(A[i][0], B[j][0]);
        int end = Math.min(A[i][1], B[j][1]);
        
        if (start <= end) {
            // They overlap, add intersection
            result.add(new int[]{start, end});
        }
        
        // Move pointer of interval that ends first
        if (A[i][1] < B[j][1]) {
            i++;
        } else {
            j++;
        }
    }
    
    return result.toArray(new int[result.size()][]);
}
Intersection of [a, b] and [c, d]:

start = max(a, c)
end = min(b, d)

If start ≤ end → Intersection = [start, end]

Pattern 6: Remove Overlapping Intervals

📝 Pattern 6: Remove Minimum Intervals

Problem: Remove minimum intervals to make rest non-overlapping

Strategy: Greedy - keep intervals with earliest end time

Time: O(N log N), Space: O(1)

🎯 Instant Recognition

  • ✓ "Minimum intervals to remove"
  • ✓ "Make non-overlapping"
  • ✓ "Fewest removals"
  • ✓ Related: Activity selection problem
public int eraseOverlapIntervals(int[][] intervals) {
    if (intervals.length <= 1) return 0;
    
    // Sort by END time (greedy choice)
    Arrays.sort(intervals, (a, b) -> Integer.compare(a[1], b[1]));
    
    int removed = 0;
    int prevEnd = intervals[0][1];
    
    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] < prevEnd) {
            // Overlap found - remove this interval
            removed++;
        } else {
            // No overlap - update end time
            prevEnd = intervals[i][1];
        }
    }
    
    return removed;
}

💡 Intuition

Why sort by END time?

Greedy strategy: Always keep the interval that ends earliest. This leaves more room for future intervals!

Example: [[1,4], [2,3], [3,5]]

After sorting by end: [[2,3], [1,4], [3,5]]

• Keep [2,3] (ends earliest)

• Keep [3,5] (starts at 3 ≥ 3, no overlap)

• Remove [1,4] (overlaps with [2,3])

Answer: 1 removal

🔑 Common Techniques Reference

Sorting Strategies

// 1. Sort by START time (most common)
Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

// 2. Sort by END time (greedy problems)
Arrays.sort(intervals, (a, b) -> Integer.compare(a[1], b[1]));

// 3. Custom comparator (start, then by end)
Arrays.sort(intervals, (a, b) -> {
    if (a[0] != b[0]) return a[0] - b[0];
    return a[1] - b[1];
});

Overlap Detection Formulas

Two intervals [a, b] and [c, d] overlap if:

max(a, c) ≤ min(b, d)

OR equivalently:
c ≤ b AND a ≤ d
Merge two overlapping intervals [a, b] and [c, d]:

Result = [min(a, c), max(b, d)]
Intersection of [a, b] and [c, d]:

If they overlap:
Intersection = [max(a, c), min(b, d)]

💡 Pro Tips

Essential Tips

  1. Always sort first! 99% of interval problems need sorting
  2. Sort by start by default unless it's a greedy problem (then sort by end)
  3. Watch for edge cases: Empty array, single interval, all overlapping
  4. Overlap check: nextStart ≤ currentEnd (remember this!)
  5. Use Math.max/min for finding overlaps and merges
  6. Two pointers for intersection problems with two lists
  7. Heap/PriorityQueue for tracking active intervals

⚠️ Common Mistakes

  1. Forgetting to sort - Most interval algorithms assume sorted input!
  2. Wrong overlap check: Using < instead of ≤
  3. Not handling edge cases: What if intervals.length == 0?
  4. Off-by-one errors: Is [1,3] and [3,5] overlapping? (YES, at point 3)
  5. Modifying while iterating: Use result list, don't modify input in-place

⚠️ How NOT to Confuse with Similar Patterns

🔥 CRITICAL: Interval vs. Other Patterns

Interval problems can look similar to other patterns. Here's how to distinguish them!

🚨 Confusion #1: Intervals vs. Sliding Window

✅ It's INTERVALS when:

  • Input is array of [start, end] pairs
  • Deals with DISCRETE ranges
  • Keywords: "meetings", "merge", "overlap"
  • Multiple separate ranges to process
  • Example: [[1,3], [2,6], [8,10]]

Example Problem:

"Merge overlapping intervals"

Input: [[1,3],[2,6],[8,10]]

→ Interval Pattern!

❌ It's SLIDING WINDOW when:

  • Input is ONE continuous array/string
  • Find a SUBARRAY within that array
  • Keywords: "substring", "subarray", "contiguous"
  • Use two pointers (left, right) moving
  • Example: [1,2,3,4,5,6,7]

Example Problem:

"Longest substring without repeating chars"

Input: "abcabcbb"

→ Sliding Window!

🔑 Key Difference:

Intervals: Multiple disconnected ranges → Process/merge them

Sliding Window: ONE continuous sequence → Find optimal window inside it

🚨 Confusion #2: Intervals vs. Array Two Pointers

✅ It's INTERVALS when:

  • Each element is a PAIR [start, end]
  • Represents time ranges/periods
  • Need to MERGE or find OVERLAPS
  • Problem about scheduling/time

Example:

"Merge [[1,3],[2,6],[8,10]]"

→ About time ranges!

❌ It's TWO POINTERS when:

  • Each element is a SINGLE value
  • Array of integers/characters
  • Find pair/triplet that meets condition
  • Often involves target sum

Example:

"Two sum in [2,7,11,15], target=9"

→ About finding pairs!

🔑 Visual Test:

If input looks like: [[a,b], [c,d], [e,f]] → INTERVALS

If input looks like: [a, b, c, d, e, f] → TWO POINTERS

🚨 Confusion #3: Pattern 1 (Merge) vs Pattern 2 (Insert)

Aspect Pattern 1: Merge Intervals Pattern 2: Insert Interval
Input List of intervals (may overlap) SORTED non-overlapping list + ONE new interval
Key Signal "Merge all overlapping intervals" "Insert a new interval"
Pre-sorted? ❌ NO - need to sort! ✅ YES - already sorted!
Strategy Sort, then merge consecutive overlaps Three phases: before, merge, after
Example [[1,3],[2,6],[8,10]] → [[1,6],[8,10]] Insert [2,5] into [[1,2],[6,9]]

⚡ Quick Check:

If problem says "insert" or gives you a SORTED list → Pattern 2

If problem just says "merge" with unsorted input → Pattern 1

🚨 Confusion #5: Intervals vs. Dynamic Programming

✅ It's INTERVALS when:

  • Input is list of [start, end] pairs
  • About time ranges/scheduling
  • Main operation: SORT and MERGE
  • No complex decision making
  • Greedy or simple iteration works

Example:

"Non-overlapping Intervals (LC 435)"

→ Sort by end, greedily keep earliest ending

❌ It's DP when:

  • Input is array of single values
  • About optimization (max/min)
  • Has OVERLAPPING SUBPROBLEMS
  • Need to make CHOICES affecting future
  • Requires memoization/table

Example:

"House Robber (LC 198)"

→ Can't rob adjacent houses, maximize sum

🔑 Tricky Case:

"Maximum Non-overlapping Intervals" sounds like DP but...

If input is [[1,2], [2,3], [3,4]] → INTERVAL pattern (greedy works!)

If input is [1, 2, 3, 4, 5] with complex rules → DP
