📊 What is Kadane's Algorithm?

Definition: Kadane's Algorithm is a dynamic programming technique to find the maximum sum of a contiguous subarray in O(N) time.

The Problem:

Given an array, find the contiguous subarray with the largest sum.

Example:

Array: [-2, 1, -3, 4, -1, 2, 1, -5, 4]

Answer: [4, -1, 2, 1] with sum = 6

🎯 When to Use Kadane's Algorithm?

✅ Look for these keywords:

  • "Maximum sum subarray" - Direct indication
  • "Largest contiguous sum" - Same as above
  • "Best consecutive sequence" - Finding optimal range
  • "Maximum product subarray" - Modified Kadane's
  • "Maximum sum circular subarray" - Variant pattern
  • "Maximum alternating subarray" - State-based variant

⚠️ Don't confuse with:

  • Prefix Sum: When you need sum = K (specific value), not maximum
  • Sliding Window: When you need EXACTLY k elements or min/max of fixed window
  • Two Pointers: When finding pairs/triplets with specific sum
  • DP (general): When problem has more complex state transitions

💡 The Core Intuition

The Key Question at Each Position:

"Should I extend the current subarray, or start fresh from here?"

1 If current sum is POSITIVE: It helps! Keep adding to it.
2 If current sum is NEGATIVE: It hurts! Better to start fresh.

Example walkthrough:

Array: [5, -3, 5]

  • At index 0: current = 5
  • At index 1: current = 5 + (-3) = 2 (positive, so keep it!)
  • At index 2: current = 2 + 5 = 7 (maximum!)

If we had started fresh at index 2: max would only be 5 (worse!)

Pattern 1: Classic Maximum Subarray Sum

Problem: Find the contiguous subarray with maximum sum

Key Insight: At each position, decide: extend current or start new

Time: O(N), Space: O(1)

🤔 Why Use Kadane's Algorithm?

The Problem Without Kadane's:

Finding maximum subarray sum naively requires checking all subarrays:

int maxSum = Integer.MIN_VALUE;
for (int i = 0; i < n; i++) {
    int sum = 0;
    for (int j = i; j < n; j++) {
        sum += arr[j];
        maxSum = Math.max(maxSum, sum);
    }
}
// O(N²) - checks all N*(N+1)/2 subarrays

For an array of 10,000 elements:

  • ❌ Naive: ~50,000,000 operations (N²)
  • ✅ Kadane's: ~10,000 operations (N)
  • 🚀 Speed up: ~5000x faster!

The Brilliant Insight:

We don't need to remember ALL previous subarrays. We only need to know:

  • maxEndingHere: Maximum sum ending at current position
  • maxSoFar: Maximum sum seen anywhere so far

The Decision at Each Step:

maxEndingHere = max(arr[i], maxEndingHere + arr[i])

This means: "Is it better to start fresh or extend current subarray?"

  • If arr[i] alone is better → Start fresh from here
  • If maxEndingHere + arr[i] is better → Extend current subarray
public int maxSubArray(int[] arr) {
    int maxSoFar = arr[0];        // Global maximum
    int maxEndingHere = arr[0];   // Maximum ending at current position
    
    for (int i = 1; i < arr.length; i++) {
        // Decide: extend current subarray or start new?
        maxEndingHere = Math.max(arr[i], maxEndingHere + arr[i]);
        
        // Update global maximum
        maxSoFar = Math.max(maxSoFar, maxEndingHere);
    }
    
    return maxSoFar;
}

🚫 Common Mistakes

Mistake 1: Wrong Initialization

❌ Wrong

int maxSoFar = 0;
int maxEndingHere = 0;

Problem: If all elements are negative, this returns 0 (wrong!)

Example: [-5, -2, -3] → returns 0, should return -2

✅ Correct

int maxSoFar = arr[0];
int maxEndingHere = arr[0];

Why: Initialize with first element to handle all-negative arrays

Example: [-5, -2, -3] → correctly returns -2

Mistake 2: Using Integer.MIN_VALUE for maxEndingHere

Problem: Only maxSoFar should start at Integer.MIN_VALUE or arr[0]

Why: maxEndingHere represents current subarray, must start with actual element

Mistake 3: Not Updating maxSoFar Inside Loop

// Wrong: Update maxSoFar after loop
for (int i = 1; i < n; i++) {
    maxEndingHere = Math.max(arr[i], maxEndingHere + arr[i]);
}
maxSoFar = maxEndingHere;  // ❌ Too late!

// Correct: Update inside loop
for (int i = 1; i < n; i++) {
    maxEndingHere = Math.max(arr[i], maxEndingHere + arr[i]);
    maxSoFar = Math.max(maxSoFar, maxEndingHere);  // ✅ Track best
}

Mistake 4: Empty Subarray Confusion

Problem: Some problems allow empty subarrays (sum = 0), most don't

Default Kadane's: Does NOT allow empty subarrays

If empty allowed: Return max(0, kadaneResult)

Pattern 2: Maximum Product Subarray

Problem: Find contiguous subarray with maximum product

Key Insight: Track both max AND min (negative × negative = positive!)

Time: O(N), Space: O(1)

🤔 Why Different from Sum?

The Critical Difference:

With multiplication, a negative number can flip everything!

  • A large negative × current min (also negative) = huge positive!
  • We need to track BOTH max and min products

Example showing why we need min:

Array: [2, 3, -2, 4, -1]

  • At index 2 (-2): max = 2×3 = 6, min = 6×(-2) = -12
  • At index 3 (4): max = 4, min = -12×4 = -48
  • At index 4 (-1): max = -48×(-1) = 48 🎯 (min becomes max!)

Key Insight:

maxProduct = max(arr[i], maxProd × arr[i], minProd × arr[i])
minProduct = min(arr[i], maxProd × arr[i], minProd × arr[i]) 
public int maxProduct(int[] arr) {
    int maxSoFar = arr[0];
    int maxEndingHere = arr[0];
    int minEndingHere = arr[0];  // Track minimum too!
    
    for (int i = 1; i < arr.length; i++) {
        int temp = maxEndingHere;  // Save before updating
        
        // Max can come from: current, max×current, or min×current
        maxEndingHere = Math.max(arr[i], 
                        Math.max(maxEndingHere * arr[i], 
                                 minEndingHere * arr[i]));
        
        // Min can come from same three options
        minEndingHere = Math.min(arr[i], 
                        Math.min(temp * arr[i], 
                                 minEndingHere * arr[i]));
        
        maxSoFar = Math.max(maxSoFar, maxEndingHere);
    }
    
    return maxSoFar;
}

Pattern 3: Maximum Circular Subarray

Problem: Array is circular, maximum subarray can wrap around

Key Insight: Max circular = max(normal max, total - min subarray)

Time: O(N), Space: O(1)

🤔 Why This Pattern?

The Circular Challenge:

In a circular array, the maximum subarray can wrap around from end to beginning.

Example: [5, -3, 5]

Linear max: 7 ([5, -3, 5])

Circular max: 10 ([5] + [5], wrapping around)

The Brilliant Trick:

If max wraps around, it means we're taking:

  • Some prefix from start
  • Some suffix from end
  • Skipping some middle part

Key Insight:

Max Circular = Total Sum - Min Subarray Sum

Why? If we remove the minimum subarray, we keep the maximum wrapped part!

1 Find max subarray (normal Kadane's)
2 Find min subarray (inverted Kadane's)
3 Answer = max(maxSubarray, totalSum - minSubarray)

Edge Case: If all numbers are negative, totalSum - minSubarray = 0 (empty array). Return maxSubarray instead.

public int maxCircularSubarray(int[] arr) {
    int n = arr.length;
    
    // Case 1: Maximum subarray (normal Kadane's)
    int normalMax = kadaneMax(arr);
    
    // Edge case: all negative
    if (normalMax < 0) return normalMax;
    
    // Case 2: Maximum wrap-around
    int totalSum = 0;
    for (int i = 0; i < n; i++) {
        totalSum += arr[i];
        arr[i] = -arr[i];  // Invert for finding min
    }
    
    // Find minimum subarray (= maximum of inverted)
    int circularMax = totalSum + kadaneMax(arr);
    
    return Math.max(normalMax, circularMax);
}

private int kadaneMax(int[] arr) {
    int maxSoFar = arr[0];
    int maxEndingHere = arr[0];
    
    for (int i = 1; i < arr.length; i++) {
        maxEndingHere = Math.max(arr[i], maxEndingHere + arr[i]);
        maxSoFar = Math.max(maxSoFar, maxEndingHere);
    }
    
    return maxSoFar;
}

Pattern 4: Maximum Subarray with Constraints

Problem: Find max subarray with length ≤ K or other constraints

Key Insight: Use prefix sum + sliding window with Kadane's logic

Time: O(N), Space: O(N) or O(K)

🤔 When Constraints Matter?

Common Constraints:

  • Subarray length must be ≤ K
  • Subarray length must be ≥ K
  • Subarray length must be exactly K
  • Cannot use more than M negative numbers

Example: Max sum with length ≤ K

Array: [1, -2, 3, 4], K = 2

Without constraint: [3, 4] = 7

With K=2: Still [3, 4] = 7 (length 2, within limit)

But if K=1: Best is [4] = 4

// Maximum subarray with length at most K
public int maxSubarrayLengthK(int[] arr, int k) {
    int n = arr.length;
    int maxSum = Integer.MIN_VALUE;
    int currentSum = 0;
    
    for (int i = 0; i < n; i++) {
        currentSum += arr[i];
        
        // Update max for subarrays ending here (up to length k)
        maxSum = Math.max(maxSum, currentSum);
        
        // If window exceeds k, remove leftmost elements
        if (i >= k) {
            currentSum -= arr[i - k];
        }
        
        // Apply Kadane's logic: reset if negative
        if (currentSum < 0) {
            currentSum = 0;
        }
    }
    
    return maxSum;
}

Pattern 5: Maximum Alternating Subarray

Problem: Subarray with alternating positive/negative (or other states)

Key Insight: Track state transitions, use multiple Kadane's

Time: O(N), Space: O(1)

🤔 State-Based Kadane's?

When Elements Have States:

Sometimes we need to track different states at each position:

  • Ending with positive vs negative number
  • Ending with odd vs even number
  • Buy vs sell state (stock problems)

Example: Wiggle Subsequence

Find longest subsequence where elements alternate between increasing/decreasing

[1, 7, 4, 9, 2, 5] → length 6 (all elements qualify)

Solution: Track two states:

  • up[i]: Length ending with an increase
  • down[i]: Length ending with a decrease
// Wiggle Subsequence - alternating increases/decreases
public int wiggleMaxLength(int[] arr) {
    if (arr.length < 2) return arr.length;
    
    int up = 1;    // Length ending with increase
    int down = 1;  // Length ending with decrease
    
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] > arr[i - 1]) {
            // Current is higher: extend "down" sequence
            up = down + 1;
        } else if (arr[i] < arr[i - 1]) {
            // Current is lower: extend "up" sequence
            down = up + 1;
        }
        // If equal, no change
    }
    
    return Math.max(up, down);
}

Pattern 6: Buy and Sell Stock (Kadane Variant)

Problem: Maximize profit from buying and selling stocks

Key Insight: Track min price seen, max profit = current - min

Time: O(N), Space: O(1)

🤔 Connection to Kadane's?

Stock Problem as Kadane's:

If we convert prices to daily changes (differences):

diff[i] = prices[i] - prices[i-1]

Then finding max profit = finding max sum subarray of differences!

Direct Approach (Better):

Track minimum price seen so far, calculate profit at each step

// Best Time to Buy and Sell Stock
public int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE;
    int maxProfit = 0;
    
    for (int price : prices) {
        // Update minimum price seen
        minPrice = Math.min(minPrice, price);
        
        // Calculate profit if we sell today
        maxProfit = Math.max(maxProfit, price - minPrice);
    }
    
    return maxProfit;
}

// Alternative: Convert to Kadane's
public int maxProfitKadane(int[] prices) {
    int maxEndingHere = 0;
    int maxSoFar = 0;
    
    for (int i = 1; i < prices.length; i++) {
        // Daily change (profit/loss)
        int change = prices[i] - prices[i - 1];
        
        // Apply Kadane's to changes
        maxEndingHere = Math.max(0, maxEndingHere + change);
        maxSoFar = Math.max(maxSoFar, maxEndingHere);
    }
    
    return maxSoFar;
}

Pattern 7: 2D Kadane (Maximum Sum Rectangle)

Problem: Find rectangle in 2D matrix with maximum sum

Key Insight: Fix top/bottom rows, compress to 1D, apply Kadane's

Time: O(N² × M), Space: O(M)

🤔 Extending to 2D?

The Challenge:

A brute force check of all rectangles is O(N² × M²) - too slow!

The Clever Trick:

1 Fix top row and bottom row (O(N²) combinations)
2 Compress all rows between them into a 1D array by summing columns
3 Apply Kadane's algorithm on this 1D array (O(M))

Total Time: O(N²) × O(M) = O(N² × M)

public int maxSumRectangle(int[][] matrix) {
    int rows = matrix.length;
    int cols = matrix[0].length;
    int maxSum = Integer.MIN_VALUE;
    
    // Try all possible top row positions
    for (int top = 0; top < rows; top++) {
        int[] temp = new int[cols];  // Compressed array
        
        // Try all possible bottom row positions
        for (int bottom = top; bottom < rows; bottom++) {
            // Add current row to compressed array
            for (int col = 0; col < cols; col++) {
                temp[col] += matrix[bottom][col];
            }
            
            // Apply Kadane's on compressed 1D array
            int currentMax = kadane(temp);
            maxSum = Math.max(maxSum, currentMax);
        }
    }
    
    return maxSum;
}

private int kadane(int[] arr) {
    int maxSoFar = arr[0];
    int maxEndingHere = arr[0];
    
    for (int i = 1; i < arr.length; i++) {
        maxEndingHere = Math.max(arr[i], maxEndingHere + arr[i]);
        maxSoFar = Math.max(maxSoFar, maxEndingHere);
    }
    
    return maxSoFar;
}

📊 Example: 3×3 matrix

2 1 -3
-1 4 -1
-1 -1 5

Fix top=0, bottom=1 (first two rows):

Compress columns: [2-1, 1+4, -3-1] = [1, 5, -4]

Kadane's on [1, 5, -4]: max = 6 (take [1, 5])

This represents the green rectangle: sum = 2+1-1+4 = 6 ✓

Pattern 8: Minimum Subarray Sum (Inverted Kadane's)

Problem: Find contiguous subarray with MINIMUM sum

Key Insight: Same as Kadane's but track minimum instead of maximum

Time: O(N), Space: O(1)

🤔 Why Minimum Subarray?

When Do We Need Minimum?

  • Circular Maximum: We use min to find max circular subarray
  • Optimization Problems: Finding worst-case scenarios
  • Inverted Problems: Negate array, find max = find min of original

The Modification:

Simply flip max to min in Kadane's algorithm:

minEndingHere = min(arr[i], minEndingHere + arr[i])
minSoFar = min(minSoFar, minEndingHere) 
public int minSubArray(int[] arr) {
    int minSoFar = arr[0];
    int minEndingHere = arr[0];
    
    for (int i = 1; i < arr.length; i++) {
        // Keep extending if it makes sum smaller, else start fresh
        minEndingHere = Math.min(arr[i], minEndingHere + arr[i]);
        minSoFar = Math.min(minSoFar, minEndingHere);
    }
    
    return minSoFar;
}

// Alternative: Use max Kadane's on negated array
public int minSubArrayNegation(int[] arr) {
    int[] negated = new int[arr.length];
    for (int i = 0; i < arr.length; i++) {
        negated[i] = -arr[i];
    }
    return -maxSubArray(negated);  // Negate result back
}

Pattern 9: Maximum Subarray with At Most K Deletions

Problem: Find max sum allowing deletion of at most K elements

Key Insight: Track state: how many deletions used so far

Time: O(N × K), Space: O(K) or O(1) with optimization

🤔 Why Allow Deletions?

The Power of Skipping:

Sometimes we want to skip bad elements (like a large negative) while keeping the subarray "contiguous" in a looser sense.

Example: arr = [8, -1, -1, -1, 9], K = 2

Without deletions: max = 9

With K=2 deletions: [8, skip(-1), skip(-1), -1, 9] = 8 + (-1) + 9 = 16

State Tracking:

We need to track: dp[i][j] = max sum ending at i with j deletions used

1 Include current element: dp[i][j] = dp[i-1][j] + arr[i]
2 Delete current element: dp[i][j] = dp[i-1][j-1] (if j > 0)
3 Start fresh: dp[i][j] = arr[i] 
public int maxSumWithDeletions(int[] arr, int k) {
    int n = arr.length;
    int[][] dp = new int[n][k + 1];
    int maxSum = Integer.MIN_VALUE;
    
    // Base case: first element
    for (int j = 0; j <= k; j++) {
        dp[0][j] = arr[0];
        maxSum = Math.max(maxSum, arr[0]);
    }
    
    for (int i = 1; i < n; i++) {
        for (int j = 0; j <= k; j++) {
            // Option 1: Include current element
            dp[i][j] = dp[i - 1][j] + arr[i];
            
            // Option 2: Start fresh from current
            dp[i][j] = Math.max(dp[i][j], arr[i]);
            
            // Option 3: Delete current (use one deletion)
            if (j > 0) {
                dp[i][j] = Math.max(dp[i][j], dp[i - 1][j - 1]);
            }
            
            maxSum = Math.max(maxSum, dp[i][j]);
        }
    }
    
    return maxSum;
}

// Space-optimized: O(K) space
public int maxSumWithDeletionsOptimized(int[] arr, int k) {
    int[] prev = new int[k + 1];
    int[] curr = new int[k + 1];
    int maxSum = Integer.MIN_VALUE;
    
    Arrays.fill(prev, arr[0]);
    maxSum = arr[0];
    
    for (int i = 1; i < arr.length; i++) {
        for (int j = 0; j <= k; j++) {
            curr[j] = Math.max(arr[i], prev[j] + arr[i]);
            if (j > 0) {
                curr[j] = Math.max(curr[j], prev[j - 1]);
            }
            maxSum = Math.max(maxSum, curr[j]);
        }
        int[] temp = prev;
        prev = curr;
        curr = temp;
    }
    
    return maxSum;
}

Pattern 10: Maximum Sum with No Adjacent Elements (House Robber)

Problem: Find max sum where no two selected elements are adjacent

Key Insight: At each position: take current + skip previous, or skip current

Time: O(N), Space: O(1)

🤔 Why No Adjacent?

The House Robber Analogy:

You're a robber robbing houses in a row. Adjacent houses have alarms that trigger if both are robbed. Maximize stolen money!

Example: [2, 7, 9, 3, 1]

Can't take 7 AND 9 (adjacent)

Best: 2 + 9 + 1 = 12

The DP Transition:

dp[i] = max(dp[i-1], arr[i] + dp[i-2])
1 Skip current: dp[i-1] (best so far without current)
2 Take current: arr[i] + dp[i-2] (current + best from 2 back)

Space Optimization:

We only need last 2 values, so use two variables instead of array!

// House Robber - O(1) space
public int rob(int[] arr) {
    if (arr.length == 0) return 0;
    if (arr.length == 1) return arr[0];
    
    int prev2 = arr[0];           // dp[i-2]
    int prev1 = Math.max(arr[0], arr[1]);  // dp[i-1]
    
    for (int i = 2; i < arr.length; i++) {
        int current = Math.max(prev1, arr[i] + prev2);
        prev2 = prev1;
        prev1 = current;
    }
    
    return prev1;
}

// With DP array (for clarity)
public int robDP(int[] arr) {
    if (arr.length == 0) return 0;
    if (arr.length == 1) return arr[0];
    
    int[] dp = new int[arr.length];
    dp[0] = arr[0];
    dp[1] = Math.max(arr[0], arr[1]);
    
    for (int i = 2; i < arr.length; i++) {
        dp[i] = Math.max(dp[i - 1], arr[i] + dp[i - 2]);
    }
    
    return dp[arr.length - 1];
}

🔄 Kadane's vs Other Techniques

Technique When to Use Time Key Difference
Kadane's MAXIMUM sum subarray O(N) Greedy DP approach
Prefix Sum Sum = K (specific value) O(N) Uses HashMap
Sliding Window Fixed size K or min/max O(N) Maintains window
Two Pointers Sorted, find pairs O(N) Bidirectional scan

💡 Pro Tips

Tip 1: Watch for Integer Overflow

When dealing with very large numbers, use long instead of int:

long maxSoFar = arr[0];
long maxEndingHere = arr[0];

Tip 2: Track Indices

To return the actual subarray (not just the sum):

int start = 0, end = 0, tempStart = 0;

for (int i = 1; i < n; i++) {
    if (arr[i] > maxEndingHere + arr[i]) {
        maxEndingHere = arr[i];
        tempStart = i;  // New start
    } else {
        maxEndingHere += arr[i];
    }
    
    if (maxEndingHere > maxSoFar) {
        maxSoFar = maxEndingHere;
        start = tempStart;
        end = i;
    }
}

Tip 3: All Negative Numbers

Always initialize with arr[0], not 0 or Integer.MIN_VALUE

This handles the all-negative case correctly
