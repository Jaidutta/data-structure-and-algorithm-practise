Understanding Binary Trees & BST

What is a Binary Tree?

A Binary Tree is a hierarchical structure where each node has at most two children. It's fundamental to many algorithms and real-world applications. 

        1
       / \
      2   3
     / \   \
    4   5   6

Components:
- Root: 1
- Internal Nodes: 2, 3
- Leaf Nodes: 4, 5, 6
- Height: 2

What is a Binary Search Tree (BST)?

A BST maintains ordering: Left < Root < Right for all nodes. 

        8
       / \
      3   10
     / \    \
    1   6    14

BST Property:
✓ Left subtree (1,3,6) < 8
✓ Right subtree (10,14) > 8
✓ Property holds recursively

Binary Tree vs BST

Aspect Binary Tree BST
Ordering No specific order Left < Node < Right
Search O(n) O(log n) average
Insert O(1) if position known O(log n) average
Use Case Hierarchical data Fast search/insert

Pattern 1: Depth-First Search (DFS)

DFS Traversal Patterns

🔍 What is it?

DFS explores depth-first by going deep along branches before backtracking. Three types:

  • Preorder: Root → Left → Right
  • Inorder: Left → Root → Right
  • Postorder: Left → Right → Root

📊 Visual Example

       1
      / \
     2   3
    / \
   4   5

Preorder:  1 → 2 → 4 → 5 → 3
Inorder:   4 → 2 → 5 → 1 → 3
Postorder: 4 → 5 → 2 → 3 → 1

🎯 When to Use?

  • Explore all root-to-leaf paths
  • Tree serialization/copying
  • Inorder for BST: Get sorted order
  • Postorder: Delete tree, calculate sizes

💡 Pattern Intuition

DFS matches tree's recursive structure. Preorder processes parent first (copying). Inorder on BST gives sorted values. Postorder processes children first (deletion).

💻 Java Code

// Preorder: Root → Left → Right
public void preorder(TreeNode root) {
    if (root == null) return;
    System.out.print(root.val + " ");  // Process root
    preorder(root.left);
    preorder(root.right);
}

// Inorder: Left → Root → Right
public void inorder(TreeNode root) {
    if (root == null) return;
    inorder(root.left);
    System.out.print(root.val + " ");  // Process root
    inorder(root.right);
}

// Postorder: Left → Right → Root
public void postorder(TreeNode root) {
    if (root == null) return;
    postorder(root.left);
    postorder(root.right);
    System.out.print(root.val + " ");  // Process root
}

🔍 How to Identify This Pattern?

  • Need to visit ALL nodes
  • Order matters (preorder/inorder/postorder)
  • Problem mentions "traverse" or "visit all nodes"
  • BST problems often use inorder
⏱️ Time: O(n) 💾 Space: O(h)

Pattern 2: Breadth-First Search (BFS)

Level Order Traversal

🔍 What is it?

BFS visits nodes level by level using a queue. All nodes at depth d before depth d+1.

📊 Visual Example

       1           Level 0: [1]
      / \
     2   3         Level 1: [2, 3]
    / \   \
   4   5   6       Level 2: [4, 5, 6]

BFS Order: [[1], [2, 3], [4, 5, 6]]

🎯 When to Use?

  • Level-by-level processing
  • Finding shortest path/minimum depth
  • Right/left side view of tree
  • Zigzag level order

💡 Pattern Intuition

Queue ensures FIFO order, processing all level N nodes before level N+1. Perfect for level-wise information or finding shortest distances.

💻 Java Code

public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    
    while (!queue.isEmpty()) {
        int levelSize = queue.size();  // KEY: Capture level size
        List<Integer> level = new ArrayList<>();
        
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}

🔍 How to Identify This Pattern?

  • Problem mentions "level" or "layer"
  • Need to process nodes level by level
  • Finding shortest path in tree
  • Right/left view of tree
⏱️ Time: O(n) 💾 Space: O(w) - max width

Pattern 3: BST Search & Validation

Binary Search Tree Operations

🔍 What is it?

Leverage BST property (Left < Root < Right) to eliminate half the tree at each step.

📊 Visual Example

        8
       / \
      3   10      Search 6:
     / \    \     6 < 8 → Go LEFT
    1   6    14   6 > 3 → Go RIGHT
       / \   /    Found 6! ✓
      4   7 13

Only 3 comparisons!

🎯 When to Use?

  • Search/Insert/Delete in BST
  • Validate if tree is valid BST
  • Find kth smallest/largest element
  • Range sum queries

💡 Pattern Intuition

BST creates binary search opportunity - compare value to eliminate half the search space. Inorder traversal gives sorted order, useful for kth element problems.

💻 Java Code

// BST Search
public TreeNode searchBST(TreeNode root, int val) {
    if (root == null || root.val == val) return root;
    
    if (val < root.val) {
        return searchBST(root.left, val);
    }
    return searchBST(root.right, val);
}

// Validate BST - CRITICAL: Check entire subtree!
public boolean isValidBST(TreeNode root) {
    return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
}

private boolean validate(TreeNode node, long min, long max) {
    if (node == null) return true;
    if (node.val <= min || node.val >= max) return false;
    
    return validate(node.left, min, node.val) &&
           validate(node.right, node.val, max);
}

🔍 How to Identify This Pattern?

  • Problem explicitly mentions BST
  • Need to find/insert/delete specific value
  • Question about kth smallest/largest
  • Validate BST property
⏱️ Time: O(log n) avg, O(n) worst 💾 Space: O(h)

Pattern 4: Path Sum / Root-to-Leaf

Path Tracking Pattern

🔍 What is it?

Traverse from root to leaves, tracking cumulative values. Uses backtracking for all paths.

📊 Visual Example

        5
       / \
      4   8
     /   / \
    11  13  4
   /  \      \
  7    2      1

Target: 22
Path: 5 → 4 → 11 → 2 = 22 ✓

💻 Java Code

public boolean hasPathSum(TreeNode root, int sum) {
    if (root == null) return false;
    
    // Leaf node - check if sum matches
    if (root.left == null && root.right == null) {
        return root.val == sum;
    }
    
    int remaining = sum - root.val;
    return hasPathSum(root.left, remaining) || 
           hasPathSum(root.right, remaining);
}

🔍 How to Identify This Pattern?

  • Problem mentions "path" from root to leaf
  • Need to track sum/product along path
  • Finding all paths with property

Pattern 5: Two Trees Comparison

Parallel Traversal Pattern

🔍 What is it?

Simultaneously traverse two trees to compare structures or values.

💻 Java Code

public boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) return true;
    if (p == null || q == null) return false;
    if (p.val != q.val) return false;
    
    return isSameTree(p.left, q.left) && 
           isSameTree(p.right, q.right);
}

Pattern 6: Lowest Common Ancestor

Finding Split Point Pattern

💻 Java Code

public TreeNode lowestCommonAncestor(TreeNode root, 
                                        TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    
    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    
    if (left != null && right != null) return root;
    return (left != null) ? left : right;
}

Pattern 7: Tree Construction

Build from Traversals Pattern

💻 Java Code

private int preIndex = 0;
private Map<Integer, Integer> inorderMap;

public TreeNode buildTree(int[] preorder, int[] inorder) {
    inorderMap = new HashMap<>();
    for (int i = 0; i < inorder.length; i++) {
        inorderMap.put(inorder[i], i);
    }
    return build(preorder, 0, inorder.length - 1);
}

private TreeNode build(int[] preorder, int left, int right) {
    if (left > right) return null;
    
    int rootVal = preorder[preIndex++];
    TreeNode root = new TreeNode(rootVal);
    
    int index = inorderMap.get(rootVal);
    root.left = build(preorder, left, index - 1);
    root.right = build(preorder, index + 1, right);
    
    return root;
}

Pattern 8: Recursion with Return Values

Bottom-Up Computation Pattern

💻 Java Code

// Max Depth
public int maxDepth(TreeNode root) {
    if (root == null) return 0;
    return 1 + Math.max(maxDepth(root.left), maxDepth(root.right));
}

// Diameter
private int diameter = 0;
public int diameterOfBinaryTree(TreeNode root) {
    height(root);
    return diameter;
}

private int height(TreeNode node) {
    if (node == null) return 0;
    int left = height(node.left);
    int right = height(node.right);
    diameter = Math.max(diameter, left + right);
    return 1 + Math.max(left, right);
}

Pattern Recognition Guide

If You See... Use Pattern Example
"traverse" or "visit all" DFS Preorder/Inorder/Postorder
"level" or "layer" BFS Level order traversal
"BST" + search/insert BST Search Binary search in tree
"path" from root to leaf Path Sum Track cumulative sum
Compare two trees Two Trees Parallel traversal
"common ancestor" LCA Find split point

🎯 Problem-Solving Strategy

  1. Identify if it's a tree problem (structure, traversal, property)
  2. Check for keywords (level, path, BST, ancestor, etc.)
  3. Choose pattern based on keywords
  4. Implement using pattern template
  5. Test with edge cases (null, single node, skewed tree)
