What is a Tree?

Understanding Trees

A tree is a hierarchical data structure that consists of nodes connected by edges. Unlike linear data structures (arrays, linked lists), trees represent hierarchical relationships, similar to a family tree or organizational chart.

Real-World Analogy

Think of a tree like a family tree: Grandparent (Root) | _____|_____ | | Parent1 Parent2 | | Child1 Child2 Key Concepts: - Root: The topmost ancestor (Grandparent) - Parent: Has descendants (Parent1, Parent2) - Children: Descendants of a parent - Leaf: Has no descendants (Child1, Child2) - Edge: Connection between nodes

Why Use Trees?

🎯 Hierarchical Data

File systems, HTML DOM, organizational charts

⚡ Fast Operations

Search, insert, delete in O(log n) time

🔍 Sorted Data

BST maintains sorted order efficiently

📊 Efficient Storage

Balanced trees optimize space usage

Tree vs Other Data Structures

Operation Array Linked List Balanced BST
Search O(n) or O(log n) if sorted O(n) O(log n)
Insert O(n) - need to shift O(1) if at head O(log n)
Delete O(n) - need to shift O(1) if at head O(log n)
Maintain Order Requires sorting Requires sorting Automatic

Generic Tree Fundamentals

What is a Generic Tree?

A Generic Tree (or N-ary Tree) is a tree data structure where each node can have any number of children (0 to N), unlike a binary tree which is limited to 2 children per node.

Generic Tree vs Binary Tree

Generic Tree (N-ary): A / | \ / | \ B C D ← Node A has 3 children /|\ | /\ / | \ | / \ E F G H I J ← Node B has 3 children ← Node C has 1 child ← Node D has 2 children Any number of children allowed! ✓ Binary Tree: A / \ / \ B C ← Each node has max 2 children / \ / \ D E ← Strictly limited to 2 ✓

Generic Tree Structure

Node Representation Options

Option 1: Array/List of Children Node structure: ┌────────────────────┐ │ val: 'A' │ │ children: [B,C,D] │ ← List can hold any number of children └────────────────────┘ Visual tree: A / | \ B C D Each child (B, C, D) is also a TreeNode with its own children list Option 2: First Child, Next Sibling Converts N-ary tree to binary representation: Original N-ary tree: A / | \ B C D /| E F Converted to First-Child/Next-Sibling: A / B ──→ C ──→ D ← B's siblings are C and D / E ──→ F ← E's sibling is F Each node has: • firstChild: pointer to leftmost child • nextSibling: pointer to right sibling

Generic Tree Operations

1. Traversal

Unlike binary trees with specific traversal orders, generic trees typically use:

Preorder Traversal (Parent before Children)

Tree: A / | \ / | \ B C D /| | / | | E F G Preorder Visit Order: A → B → E → F → C → G → D Step-by-step execution: 1. Visit A (root) 2. Visit A's children left to right: - Visit B - Visit B's children: E, F - Visit C - Visit C's children: G - Visit D - D has no children Algorithm: At each node: 1. Process/print current node 2. Recursively visit each child (left to right)
// Preorder Traversal
public void preorder(GenericTreeNode root) {
    if (root == null) return;
    
    // Visit current node
    System.out.print(root.val + " ");
    
    // Visit all children
    for (GenericTreeNode child : root.children) {
        preorder(child);
    }
}

// Postorder Traversal (Children before Parent)
public void postorder(GenericTreeNode root) {
    if (root == null) return;
    
    // Visit all children first
    for (GenericTreeNode child : root.children) {
        postorder(child);
    }
    
    // Then visit current node
    System.out.print(root.val + " ");
}

// Level Order Traversal
public void levelOrder(GenericTreeNode root) {
    if (root == null) return;
    
    Queue<GenericTreeNode> queue = new LinkedList<>();
    queue.offer(root);
    
    while (!queue.isEmpty()) {
        GenericTreeNode node = queue.poll();
        System.out.print(node.val + " ");
        
        // Add all children to queue
        for (GenericTreeNode child : node.children) {
            queue.offer(child);
        }
    }
}

2. Calculate Height

Height Calculation Example: Tree: A ← height = 2 / | \ / | \ B C D ← height = 1 for B and C, height = 0 for D /| | / | | E F G ← height = 0 for all leaves How height is calculated: Node E: height = 0 (leaf node, no children) Node F: height = 0 (leaf node, no children) Node G: height = 0 (leaf node, no children) Node D: height = 0 (leaf node, no children) Node B: height = 1 + max(height(E), height(F)) = 1 + max(0, 0) = 1 Node C: height = 1 + max(height(G)) = 1 + 0 = 1 Node A: height = 1 + max(height(B), height(C), height(D)) = 1 + max(1, 1, 0) = 2 Formula: height = 1 + max(all children heights)
public int height(GenericTreeNode root) {
    if (root == null) return -1;
    
    int maxChildHeight = -1;
    
    // Find maximum height among all children
    for (GenericTreeNode child : root.children) {
        int childHeight = height(child);
        maxChildHeight = Math.max(maxChildHeight, childHeight);
    }
    
    return maxChildHeight + 1;
}

3. Count Nodes

public int countNodes(GenericTreeNode root) {
    if (root == null) return 0;
    
    int count = 1;  // Count current node
    
    // Add counts from all children
    for (GenericTreeNode child : root.children) {
        count += countNodes(child);
    }
    
    return count;
}

// Count leaf nodes (nodes with no children)
public int countLeaves(GenericTreeNode root) {
    if (root == null) return 0;
    
    // Leaf node: no children
    if (root.children.isEmpty()) {
        return 1;
    }
    
    int leafCount = 0;
    for (GenericTreeNode child : root.children) {
        leafCount += countLeaves(child);
    }
    
    return leafCount;
}

Generic Tree vs Binary Tree Comparison

Aspect Generic Tree Binary Tree
Children per Node 0 to N (unlimited) 0 to 2 (maximum)
Node Structure List/Array of children left and right pointers
Traversal Types Preorder, Postorder, Level-order Preorder, Inorder, Postorder, Level-order
Memory Usage More (dynamic list of children) Less (fixed 2 pointers)

💡 When to Use Generic Trees?

  • Variable Children: When nodes naturally have varying numbers of children (file systems)
  • Hierarchical Relationships: Organization structures, category trees
  • DOM Representation: HTML/XML document trees
  • Game Trees: Decision trees in games (chess, tic-tac-toe)
  • Syntax Trees: Representing programming language syntax

⚡ Key Differences from Binary Trees

  • Flexibility: Generic trees allow any number of children, making them more flexible
  • No Inorder: Inorder traversal doesn't make sense (no "between" when there are N children)
  • Iteration: Must iterate through children list/array instead of just left/right
  • Space: Slightly more memory overhead due to dynamic children storage

Types of Binary Trees

1. Full Binary Tree

Every node has either 0 or 2 children. No node has exactly 1 child.

✓ Full Binary Tree: 1 / \ / \ 2 3 Every node has 0 or 2 children / \ / \ 4 5 Nodes 4 and 5 have 0 children ✓ Nodes 2 and 3 have 0 or 2 children ✓ Node 1 has 2 children ✓ ✗ NOT Full: 1 / \ / \ 2 3 / Node 2 has only 1 child ✗ / 4

2. Complete Binary Tree

All levels are completely filled except possibly the last level, which is filled from left to right.

✓ Complete Binary Tree: 1 / \ / \ 2 3 All levels filled left to right / \ / / \ / 4 5 6 Level 0: [1] - Completely filled ✓ Level 1: [2, 3] - Completely filled ✓ Level 2: [4, 5, 6] - Filled from left ✓ ✗ NOT Complete: 1 / \ / \ 2 3 \ / Not filled left to right ✗ \ / 5 6 (Missing node 4)

3. Perfect Binary Tree

All internal nodes have exactly 2 children AND all leaf nodes are at the same level.

✓ Perfect Binary Tree: 1 / \ / \ 2 3 All levels completely filled / \ / \ / \ / \ 4 5 6 7 Properties: - All internal nodes have 2 children ✓ - All leaves at same level ✓ - Total nodes = 2^(h+1) - 1 = 7 - Height h = 2, Nodes = 2^3 - 1 = 7 ✗ NOT Perfect: 1 / \ / \ 2 3 Leaves not at same level ✗ / \ / \ 4 5

4. Balanced Binary Tree

The height difference between left and right subtrees is at most 1 for every node.

✓ Balanced Binary Tree: 1 height = 2 / \ / \ 2 3 height = 1 (both) / \ / \ 4 5 height = 0 (all) Check node 1: left height = 1, right height = 1 |1 - 1| = 0 ≤ 1 ✓ Check node 2: left height = 0, right height = 0 |0 - 0| = 0 ≤ 1 ✓ ✗ NOT Balanced: 1 / Height difference > 1 ✗ / 2 / / 3 Node 1: left height = 2, right height = -1 |2 - (-1)| = 3 > 1 ✗

5. Degenerate (Skewed) Tree

Each parent node has only one child. Essentially a linked list.

Left-Skewed Tree: 1 / / 2 Worst case performance / Search: O(n) 3 Like a linked list \ \ 4 Right-Skewed Tree: 1 \ \ 2 Also O(n) performance \ \ 3 No benefit of tree structure \ \ 4

6. Binary Search Tree (BST)

A special binary tree where for every node: left subtree values < node value < right subtree values

✓ Valid BST: 8 / \ 3 10 BST Property holds everywhere: / \ \ - All of 8's left (1,3,6) < 8 ✓ 1 6 14 - All of 8's right (10,14) > 8 ✓ / \ / 4 7 13 Inorder traversal: 1, 3, 4, 6, 7, 8, 10, 13, 14 (Sorted order! ✓) ✗ Invalid BST: 8 / \ 3 10 / \ 1 12 12 is in left subtree of 8 but 12 > 8 ✗ Violates BST property!

⚡ Key Differences Summary

Type Key Property Use Case
Full 0 or 2 children per node Expression trees
Complete Filled left to right Heaps, priority queues
Perfect All levels full Theoretical analysis
Balanced Height difference ≤ 1 AVL trees, Red-Black trees
BST Left < Root < Right Fast search, sorted data
Degenerate Only 1 child per node Worst case to avoid!

DFS Traversal Algorithms

What is DFS (Depth-First Search)?

DFS is a tree traversal algorithm that explores as far down a branch as possible before backtracking. Think of it like exploring a maze - you go deep down one path until you hit a dead end, then come back and try another path.

📖 The DFS Concept

Analogy: Imagine reading a book with chapters and subsections:

  • Preorder: Read chapter title, then all subsections (Table of Contents)
  • Inorder: Read left subsection, chapter title, right subsection (Reading left-to-right)
  • Postorder: Read all subsections first, then chapter title (Index at end)

How DFS Works: The Core Mechanism

DFS Uses Recursion (or Stack): 1. Start at root 2. Process node (or delay based on type) 3. Recursively visit left subtree 4. Recursively visit right subtree The Call Stack: visit(1) ├─ visit(2) │ ├─ visit(4) ← Goes deep first! │ │ └─ return │ ├─ visit(5) │ │ └─ return │ └─ return ├─ visit(3) │ └─ return └─ return Key Insight: Recursion naturally uses the call stack to remember where to go back to!

Three Types of DFS

Type Processing Order When to Use Real Example
Preorder Root → Left → Right Copy tree, Create prefix expression Backing up files (process folder then contents)
Inorder Left → Root → Right Get sorted values from BST Reading a sorted phone book
Postorder Left → Right → Root Delete tree, Calculate directory size Deleting files (delete contents before folder)

1. Preorder Traversal (Root → Left → Right)

How the Algorithm Works

Preorder processes the current node first, then recursively processes the left subtree, followed by the right subtree. The name "pre" means we process the root before its children.

💡 Algorithm Logic

  1. Base Case: If node is null, return (nothing to process)
  2. Process Current: Print/store the current node's value
  3. Recurse Left: Apply same algorithm to left child
  4. Recurse Right: Apply same algorithm to right child

Why it works: Each node is processed before diving deeper, ensuring parents come before children in output.

Preorder Algorithm Flow

Decision Tree for Each Node: At node N: ├─ Is N null? → YES → Return └─ Is N null? → NO ├─ Print N ← Process FIRST ├─ preorder(N.left) ← Then go left └─ preorder(N.right)← Then go right Example Execution Path: Tree: 1 / \ / \ 2 3 Execution sequence: 1. Visit 1 → Print 1 2. Call preorder(2) - Visit 2 → Print 2 - Call preorder(null) → return - Call preorder(null) → return 3. Call preorder(3) - Visit 3 → Print 3 - Call preorder(null) → return - Call preorder(null) → return Output: 1 2 3

🌳 Tree Structure for DFS Examples

Complete Tree (Used in Preorder, Inorder, Postorder): 1 ← Root (value: 1) / \ / \ 2 3 ← Level 1 / \ / \ 4 5 ← Level 2 (Leaf nodes) Node Details: • Node 1 (root): left=2, right=3 • Node 2: left=4, right=5 • Node 3: left=null, right=null (LEAF) • Node 4: left=null, right=null (LEAF) • Node 5: left=null, right=null (LEAF) Tree Properties: • Total Nodes: 5 • Leaf Nodes: 3 (nodes 3, 4, 5) • Internal Nodes: 2 (nodes 1, 2) • Height: 2 • Max Width: 2 (at level 2) Expected Outputs: • Preorder: 1 → 2 → 4 → 5 → 3 • Inorder: 4 → 2 → 5 → 1 → 3 • Postorder: 4 → 5 → 2 → 3 → 1

// Recursive Preorder Traversal
public void preorderTraversal(TreeNode root) {
    // Base case: if node is null, return
    if (root == null) {
        return;
    }
    
    // Step 1: Process current node (Root)
    System.out.print(root.val + " ");
    
    // Step 2: Recursively traverse left subtree
    preorderTraversal(root.left);
    
    // Step 3: Recursively traverse right subtree
    preorderTraversal(root.right);
}

// Iterative Preorder Traversal using Stack
public List<Integer> preorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;
    
    Stack<TreeNode> stack = new Stack<>();
    stack.push(root);
    
    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        result.add(node.val);  // Process node
        
        // Push right first (so left is processed first)
        if (node.right != null) {
            stack.push(node.right);
        }
        if (node.left != null) {
            stack.push(node.left);
        }
    }
    
    return result;
}

2. Inorder Traversal (Left → Root → Right)

How the Algorithm Works

Inorder processes the left subtree first, then the current node, then the right subtree. For BSTs, this produces values in sorted order!

💡 Algorithm Logic

  1. Base Case: If node is null, return
  2. Recurse Left: Process entire left subtree first
  3. Process Current: Print/store current node's value
  4. Recurse Right: Process entire right subtree

Why it works: Going leftmost first ensures smaller values (in BST) are processed first, maintaining sorted order.

Inorder Algorithm Flow

BST Example - See the Sorting Magic! Tree (BST): 5 / \ / \ 3 7 / \ / \ 1 4 At each node N: ├─ inorder(N.left) ← Go left first ├─ Print N ← Process in middle └─ inorder(N.right) ← Go right last Execution: 1. Start at 5 2. Go left to 3 3. Go left to 1 4. No more left → Print 1 ← First output! 5. Back to 3 → Print 3 6. Go right to 4 → Print 4 7. Back to 5 → Print 5 8. Go right to 7 → Print 7 Output: 1 3 4 5 7 ← SORTED! ✓ For BST: Inorder ALWAYS gives sorted values!

🌳 Tree Structure Reminder

Using the Same Tree for Inorder: 1 / \ / \ 2 3 / \ / \ 4 5 Inorder Expected Output: 4 → 2 → 5 → 1 → 3 (Left → Root → Right for each subtree)

Step-by-Step Execution

1 Go All the Way Left: Reach leftmost leaf
1 / \ / \ 2 3 / \ / \ [4] 5 ← Leftmost node, process it! Output: 4
2 Visit Parent: Process node 2
1 / \ / \ [2] 3 ← Process parent after left child / \ / \ 4 5 Output: 4 2
3 Go Right: Process right child of 2
1 / \ / \ 2 3 / \ / \ 4 [5] ← Process right child Output: 4 2 5
4 Visit Root: Now process root
[1] ← Process root after left subtree / \ / \ 2 3 / \ / \ 4 5 Output: 4 2 5 1
5 Go Right: Process right subtree
1 / \ / \ 2 [3] ← Process right subtree / \ / \ 4 5 Output: 4 2 5 1 3
Final Output: 4 → 2 → 5 → 1 → 3 Key Insight for BST: For a Binary Search Tree, inorder gives SORTED order! Example BST: 5 / \ / \ 3 7 / \ / \ 1 4 Inorder: 1 → 3 → 4 → 5 → 7 (Sorted!)
// Recursive Inorder Traversal
public void inorderTraversal(TreeNode root) {
    if (root == null) {
        return;
    }
    
    // Step 1: Recursively traverse left subtree
    inorderTraversal(root.left);
    
    // Step 2: Process current node (Root)
    System.out.print(root.val + " ");
    
    // Step 3: Recursively traverse right subtree
    inorderTraversal(root.right);
}

// Iterative Inorder Traversal using Stack
public List<Integer> inorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    Stack<TreeNode> stack = new Stack<>();
    TreeNode current = root;
    
    while (current != null || !stack.isEmpty()) {
        // Go to leftmost node
        while (current != null) {
            stack.push(current);
            current = current.left;
        }
        
        // Current is null, pop and process
        current = stack.pop();
        result.add(current.val);
        
        // Move to right subtree
        current = current.right;
    }
    
    return result;
}

3. Postorder Traversal (Left → Right → Root)

How the Algorithm Works

Postorder processes both subtrees before the current node. The name "post" means we process the root after its children. This is crucial for operations that require bottom-up computation.

💡 Algorithm Logic

  1. Base Case: If node is null, return
  2. Recurse Left: Completely process left subtree
  3. Recurse Right: Completely process right subtree
  4. Process Current: Finally process current node

Why it works: Children are processed before parent, perfect for deletion (delete files before folder) or calculating sizes (sum children's sizes first).

Postorder Algorithm Flow

Use Case: Deleting a Tree To delete a tree, we MUST delete children before parent! Tree: 1 / \ / \ 2 3 / / 4 Wrong order: Delete 1 first → Lose access to 2,3,4! Right order: Delete 4,2,3 first, then 1 ✓ At each node N: ├─ postorder(N.left) ← Delete left child ├─ postorder(N.right) ← Delete right child └─ Delete N ← Now safe to delete parent Execution: 1. Start at 1 2. Go left to 2 3. Go left to 4 4. No children → Delete 4 ← First deletion! 5. Back to 2 → No right child → Delete 2 6. Back to 1 → Go right to 3 7. No children → Delete 3 8. Back to 1 → Delete 1 ← Last deletion! Delete order: 4 2 3 1 ← Safe! ✓ Parent deleted AFTER all children!

// Recursive Postorder Traversal
public void postorderTraversal(TreeNode root) {
    if (root == null) {
        return;
    }
    
    // Step 1: Recursively traverse left subtree
    postorderTraversal(root.left);
    
    // Step 2: Recursively traverse right subtree
    postorderTraversal(root.right);
    
    // Step 3: Process current node (Root) - LAST
    System.out.print(root.val + " ");
}

// Iterative Postorder using Two Stacks
public List<Integer> postorderIterative(TreeNode root) {
    List<Integer> result = new ArrayList<>();
    if (root == null) return result;
    
    Stack<TreeNode> stack1 = new Stack<>();
    Stack<TreeNode> stack2 = new Stack<>();
    
    stack1.push(root);
    
    while (!stack1.isEmpty()) {
        TreeNode node = stack1.pop();
        stack2.push(node);
        
        // Push left and right (reverse of preorder)
        if (node.left != null) {
            stack1.push(node.left);
        }
        if (node.right != null) {
            stack1.push(node.right);
        }
    }
    
    // Pop from second stack for result
    while (!stack2.isEmpty()) {
        result.add(stack2.pop().val);
    }
    
    return result;
}

BFS (Level Order) Algorithm

What is BFS (Breadth-First Search)?

BFS explores the tree level by level, visiting all nodes at depth d before visiting any node at depth d+1. Unlike DFS which goes deep, BFS goes wide.

📖 The BFS Concept

Analogy: Imagine reading a family photo album:

  • DFS: Follow one family line all the way down (grandparent → parent → child → grandchild)
  • BFS: Look at all photos from each generation together (all grandparents, then all parents, then all children)

How BFS Works: The Core Mechanism

BFS uses a Queue (FIFO - First In First Out) data structure, unlike DFS which uses a stack (or recursion).

BFS Algorithm Steps: 1. Create an empty queue 2. Add root to queue 3. While queue is not empty: a. Count nodes at current level (levelSize) b. Process all nodes at this level: - Remove node from queue - Process node - Add its children to queue c. Move to next level Why Use a Queue? Queue ensures FIFO order: Add: [1] → [1,2] → [1,2,3] Remove: [2,3] ← Remove 1 first (oldest) This naturally processes nodes level by level! The Magic of levelSize: Before loop: queue = [2, 3] levelSize = 2 ← Captures CURRENT level count After loop: queue = [4, 5, 6] ← Next level But we only process 2 nodes (the original level)

BFS vs DFS Comparison

Aspect DFS BFS
Data Structure Stack (or recursion) Queue
Exploration Deep first Wide first (level by level)
Space O(h) - height O(w) - max width
Use Case All paths, recursion Shortest path, level processing
Implementation Simple (recursive) Requires queue

💡 When to Use BFS?

  • Finding shortest path: BFS finds closest nodes first
  • Level-wise processing: Need to process level 1, then level 2, etc.
  • Minimum depth: First leaf found is at minimum depth
  • Tree width: Finding maximum width of tree
  • Right/Left view: Last/first node at each level

public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    
    while (!queue.isEmpty()) {
        int levelSize = queue.size();  // ← KEY: Capture level size!
        List<Integer> level = new ArrayList<>();
        
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}

BST Operations Algorithms

What Makes BST Special?

A Binary Search Tree (BST) maintains a special property: for every node, all values in the left subtree are smaller, and all values in the right subtree are larger. This property enables O(log n) operations!

📖 The BST Property

For EVERY node N in the tree:

  • All values in N's left subtree < N.value
  • All values in N's right subtree > N.value
  • Both left and right subtrees are also BSTs (recursive property)

Why BST is Powerful

Binary Search in Action: Array (sorted): [1, 3, 6, 8, 10, 14] Search for 6: Compare with middle (8): 6 < 8 → Search left half Compare with middle (3): 6 > 3 → Search right half Found 6! ✓ BST does the same, but in tree structure: 8 6 < 8? Go LEFT / \ 3 10 6 > 3? Go RIGHT / \ 1 6 Found 6! ✓ Both eliminate half the data each step! Time: O(log n) for balanced tree

BST Search Algorithm

How the Algorithm Works

BST search uses the ordering property to decide which subtree to explore, eliminating half the remaining nodes at each step.

💡 Search Algorithm Logic

  1. Base Case: If node is null → value not found
  2. Found: If node.value == target → return node
  3. Go Left: If target < node.value → search left subtree
  4. Go Right: If target > node.value → search right subtree

Why it works: BST property guarantees target can only be in one subtree, so we safely ignore the other half!

Decision Making at Each Node: At node N, searching for value V: Is N null? YES → Return null (not found) NO ↓ Is V == N.value? YES → Return N (found!) NO ↓ Is V < N.value? YES → Search N.left (all values in left < N) NO → Search N.right (all values in right > N) Example: Search for 6 in: 8 ← V(6) < 8? YES → Go LEFT / \ 3 10 ← V(6) > 3? YES → Go RIGHT / \ 1 6 ← V(6) == 6? YES → FOUND! Comparisons: 3 (instead of 9 if we checked all nodes)

BST Insertion Algorithm

How Insertion Works

Insertion in BST follows the same search path to find where the new value should go, then creates a new leaf node at that position.

💡 Insertion Algorithm Logic

  1. Find Position: Use BST search to find where value should go
  2. Reached Null: When we reach null, we've found the insertion point
  3. Create Node: Create new node with value
  4. Maintain BST: Insert on correct side maintains BST property

Why it works: Following the BST search path ensures we insert in correct position to maintain ordering!

BST Validation Algorithm

How Validation Works

Validating a BST requires checking that every node satisfies the BST property with respect to ALL its ancestors, not just its immediate children.

⚠️ Common Mistake!

WRONG: Just checking if left.value < root.value < right.value

RIGHT: Check if ALL left subtree values < root < ALL right subtree values

Use min/max bounds that propagate down the tree!

Validation with Min/Max Bounds: At each node N: - N must be in range (min, max) - Left subtree: range (min, N.value) - Right subtree: range (N.value, max) 8 (min=-∞, max=+∞) / \ 3 10 / \ 1 6 (min=3, max=8) ← Range narrows! Check node 6: Is 3 < 6 < 8? YES ✓ If there was a node 12 under 3: Is 3 < 12 < 8? NO! ✗ 12 is NOT less than 8, violates BST!
